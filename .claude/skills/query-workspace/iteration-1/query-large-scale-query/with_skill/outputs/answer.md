# CAP定理和分布式共识之间有什么关系？

## 回答

根据 TestLarge 知识库 wiki 中的内容，**目前没有直接阐述 CAP 定理与分布式共识之间关系的详细页面**。wiki 中能找到的关联是间接且有限的：

### 已找到的间接关联

1. **概念页面互相引用**：[[CAP]] 和 [[Consensus]] 两个页面在"相关链接"中互相引用，但两个页面本身都只有极为简略的占位内容（仅标题和通用描述），未展开说明具体关系。

2. **分布式事务中的 CAP 引用**：[[分布式事务]] 页面提到"一致性代价：必须在强一致性和高可用性之间做出权衡（参见 [[CAP]]）"，说明在分布式事务场景下，CAP 定理指导了一致性与可用性的取舍。

3. **两阶段提交与共识的关联**：[[两阶段提交]] 页面将其归类为分布式事务协议，并在相关链接中指向 [[Consensus]]。其描述的"脑裂问题"（网络分区导致部分参与者提交、部分回滚）与 CAP 定理中的 Partition Tolerance（分区容错性）存在概念上的关联——网络分区是两者共同面对的核心挑战。

4. **分布式系统作为共同上下文**：[[DistributedSystems]] 页面同时链接到 [[CAP]] 和 [[Consensus]]，表明两者都属于分布式系统的核心概念范畴。
