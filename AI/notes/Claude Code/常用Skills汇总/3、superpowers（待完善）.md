
# 一、概述

`superpowers` 是一个面向`Claude Code`等编程 Agent 的**软件开发方法论**插件，它通过一组**可组合的技能（Skills）和初始引导指令**，专门解决 AI 编程中的一个核心问题：如何让 AI 写出“工程级”的代码，而不是“玩具级”的代码。

普通使用`Claude Code`时，经常会遇到这些问题：
- 一上来就写代码，没先弄清需求
- 没有先做设计，边做边改
- 不写测试，或者最后才补测试
- bug修复靠猜，改了很多次还是没找到根因
- 大任务拆分不清楚，做到一半偏离目标
- 代码写完后缺少系统性review
- Agent说自己完成了，但其实没有验证

它会把`Claude Code`的工作方式改造成更结构化的流程：先确认目标，再设计方案，再写计划，再执行，再测试，再review，再收尾。

核心特点：

- **零依赖设计**：不引入任何第三方依赖
- **自动触发**：技能在恰当的时机自动激活，无需手动调用
- **跨Agent**：支持`Claude Code`、`Codex`、`Cursor`等主流平台
- **强制纪律**：将TDD、系统化调试等最佳实践固化为不可跳过的流程
- **子代理驱动**：利用子代理执行独立任务，保持主会话上下文清晰


# 二、核心工作流

`superpowers` 定义了完整的软件开发工作流，由**7 个阶段**组成，每个阶段对应一个自动触发的技能。
```text
┌────────────────────────────────────────────────────────────
│ Superpowers 开发工作流
├────────────────────────────────────────────────────────────
│
│ 1. brainstorming
│    ← 用户提出想法 / 需求
│    └─ 探索需求、提出方案、获得设计批准
│
│        ↓
│
│ 2. using-git-worktrees
│    ← 创建隔离工作空间
│    └─ 新分支、依赖安装、基线测试验证
│
│        ↓
│
│ 3. writing-plans
│    ← 编写详细实施计划
│    └─ 分解为 2–5 分钟的小任务，每步含完整代码
│
│        ↓
│
│ 4. subagent-driven-development / executing-plans
│    ← 执行计划
│    └─ 每任务分派新子代理
│       规格合规审查 + 代码质量审查
│
│        ↓
│
│ 5. test-driven-development
│    ← 实施过程中强制执行
│    └─ RED（写失败测试）
│       → GREEN（最小实现）
│       → REFACTOR（重构）
│
│        ↓
│
│ 6. requesting-code-review
│    ← 任务间 / 完成后审查
│    └─ 规格符合性检查，按严重级别报告问题
│
│        ↓
│
│ 7. finishing-a-development-branch
│    ← 任务完成
│    └─ 验证测试、合并 / PR / 保留 / 丢弃选项、清理工作树
│
└────────────────────────────────────────────────────────────
```
