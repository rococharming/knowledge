# HTTP 报文

HTTP 通信通过**请求报文**和**响应报文**完成。客户端发送请求，服务端返回响应。无论底层传输格式是 HTTP/1.1 的文本报文还是 HTTP/2、HTTP/3 的二进制帧，核心语义不变：都有方法、路径、状态码、Header 和 Body。

## 请求报文

请求报文由四部分组成：请求行、请求头、空行、请求体。

- **请求行**：说明"要做什么"，包含请求方法、请求路径和 HTTP 版本
- **请求头**：补充说明"如何处理这个请求"，如 `Host`、`Content-Type`、`Authorization` 等
- **空行**：分隔 Header 和 Body
- **请求体**：携带提交给服务器的数据

常用请求方法包括 [[HTTP 请求|GET、POST、PUT、PATCH、DELETE、HEAD、OPTIONS]]，每种方法对应不同的语义操作。

## 响应报文

响应报文同样由四部分组成：状态行、响应头、空行、响应体。

- **状态行**：说明"处理结果"，包含 HTTP 版本、状态码和原因短语
- **响应头**：补充说明"如何理解这个响应"，如 `Content-Type`、`Set-Cookie`、`Cache-Control` 等
- **空行**：分隔 Header 和 Body
- **响应体**：服务器返回的实际内容

状态码按首位数字分为 5 类：1xx（信息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务端错误）。

## 调试工具

- **浏览器 Network 面板**：可视化查看请求列表、状态码、头部详情、耗时分析
- **curl 命令**：命令行 HTTP 客户端，可直接控制请求方法、头部、体、重定向等
