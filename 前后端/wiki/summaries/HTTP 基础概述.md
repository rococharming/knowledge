# HTTP 基础概述

## 核心观点

HTTP（Hyper Text Transfer Protocol）是现代 Web 开发的**基础网络通信协议**，运行在**应用层**，核心作用是**让客户端和服务器之间按照统一格式交换数据**。

## 内容范围

本文从五个维度系统介绍了 HTTP：

1. **HTTP 简介与核心作用** — 协议定位、资源类型、应用场景、核心交互模式
2. **HTTP 基本模型** — 客户端-服务器模型、请求-响应模型、面向资源设计、应用层定位、与 TCP/TLS/QUIC 的关系
3. **HTTP 系统组成** — 客户端、用户代理、服务器、Web 服务器与应用服务器、代理（正向/反向）
4. **HTTP 基本性质** — 简单易读、灵活（多类型数据传输）、可扩展（Header 机制）、无状态、依赖可靠传输
5. **一次完整的 HTTP 访问过程** — URL 解析 → DNS 解析 → 连接建立 → 请求发送 → 服务器处理 → 响应返回 → 客户端解析

## 关键要点

- HTTP 是**客户端-服务器**、**请求-响应**、**面向资源**的协议
- 资源通过 `URL` 标识，操作通过 **HTTP 方法**（GET/POST/PUT/PATCH/DELETE）表达
- HTTP 本身**无状态**，Web 应用通过 Cookie / Session / Token 建立有状态会话
- 传统 HTTP/1.1、HTTP/2 基于 **TCP**，HTTPS 增加 **TLS** 层，HTTP/3 基于 **QUIC**
- 打开一个网页通常需要**数十甚至上百个** HTTP 请求共同完成
