# 前后端 领域规则

## 领域概述

Web 前后端开发入门学习知识库。面向编程初学者，从零开始系统学习前端（HTML/CSS/JS → 框架）、后端（语言/框架/API/数据库）、HTTP 协议、系统设计基础及开发工具链。目标是建立扎实的 Web 开发通识基础，能够独立搭建完整的 Web 应用。

## 分类体系

wiki 页面按以下子目录组织：

- `summaries/` — 单个来源素材的提炼摘要
- `entities/` — 实体页面：框架、库、工具、平台（如 React、Express.js、MySQL）
- `concepts/` — 核心概念：编程范式、设计模式、技术原理（如 RESTful、响应式设计、懒加载）
- `comparisons/` — 对比分析：框架对比、方案选型、技术优劣
- `overviews/` — 广度概览：技术全景图、路线图、生态总览
- `syntheses/` — 综合判断：跨素材的深度结论、技术决策指南
- `recipes/` — 可操作配方：环境搭建步骤、部署流程、问题排查清单

## 标签体系

领域初始标签（统一使用英文，便于检索和 Dataview 查询）。标签是动态扩展的——以下只是种子标签，LLM 在 ingest 时会根据素材内容自动补充：

- `#frontend` — 前端相关：UI、浏览器渲染、CSS、DOM、响应式设计
- `#backend` — 后端相关：服务端逻辑、API 设计、中间件、认证授权
- `#api` — API 设计与交互：RESTful、GraphQL、接口规范、HTTP 语义
- `#database` — 数据库相关：SQL、ORM、数据建模、迁移、缓存
- `#architecture` — 系统架构：分层设计、微服务、前后端分离、部署架构
- `#http` — HTTP 协议基础：方法、状态码、缓存控制、CORS、TLS
- `#devops` — 开发运维：CI/CD、容器化、部署、监控、日志

**标签添加原则**：
- 初始标签作为 ingest 时的参考基准
- 当素材涉及新的子主题时，自动创建新标签
- 定期 review 标签使用情况，合并过于细分的标签

## 特殊约定

- **初学者导向**：所有概念的讲解应从最基础的层面开始，避免假设读者已有先验知识。ingest 时优先提炼核心原理和"为什么"，而非直接罗列 API 细节。
- **实战优先**：每个概念页面应包含实际可运行的代码示例，优先使用现代标准（ES6+、HTTP/2、TLS 1.3 等）。
- **技术选型基准**：涉及框架/工具对比时，以"适合个人/小团队快速开发"为首要筛选条件，兼顾学习曲线。
