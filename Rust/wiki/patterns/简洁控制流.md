# 简洁控制流

当 `match` 的完整形式显得冗余时，Rust 提供三种简洁的模式匹配控制流：`if let`、`let else` 和 `while let`。

## if let

只关心一种匹配情况时，`if let` 比完整的 `match` 更简洁。

### 基本形式

```rust
if let PATTERN = EXPR {
    // 匹配成功：可使用 PATTERN 绑定的变量
}
```

- `EXPR` 匹配 `PATTERN`：进入代码块，完成变量绑定
- `EXPR` 不匹配 `PATTERN`：什么也不做

示例：

```rust
let value = Some(10);

if let Some(n) = value {
    println!("{}", n);
}
```

等价于：

```rust
match value {
    Some(n) => println!("{}", n),
    None => {}
}
```

`if let` 可以理解为**只写一个关心的匹配分支，其他情况自动忽略**。

### 搭配 else

如果也想处理匹配失败的情况，可加 `else`：

```rust
let value = Some(10);

if let Some(n) = value {
    println!("{}", n);
} else {
    println!("no value");
}
```

`if let` 适合只关心某一种成功情况的代码，例如只处理 `Some`、只处理 `Ok`、只处理某个枚举变体。

## let else

`let else` 适合**匹配失败就提前退出**的场景，能显著减少嵌套。

### 基本语法

```rust
let PATTERN = EXPR else {
    // 匹配失败时执行
    // 必须提前退出：return / break / continue / panic!
};
```

- 若 `EXPR` 匹配 `PATTERN`：绑定写入变量，且该变量在**后面的外层作用域可用**
- 若 `EXPR` 不匹配 `PATTERN`：执行 `else` 块，且必须在块内提前退出（编译器强制检查）

示例：

```rust
fn process(value: Option<i32>) {
    let Some(n) = value else {
        return;
    };
    println!("{}", n);
}
```

如果 `value` 是 `Some(n)`，程序继续执行，后面可直接使用 `n`。如果不是，就执行 `else` 块并 `return`。

另一个例子：

```rust
fn parse(text: &str) -> i32 {
    let Ok(n) = text.parse::<i32>() else {
        return 0;
    };
    n
}
```

`let else` 让主流程更清楚，避免深层嵌套的 `match` 或 `if let`。

## while let

把循环和模式匹配结合：只要匹配成功就继续循环，匹配失败就退出。

### 基本语法

```rust
while let PATTERN = EXPR {
    // 循环体：可使用 PATTERN 绑定的变量
}
```

示例：

```rust
fn main() {
    let mut values = vec![1, 2, 3];

    while let Some(n) = values.pop() {
        println!("{}", n);
    }
}
```

输出：

```text
3
2
1
```

`Vec::pop()` 返回 `Option<T>`。还有元素时返回 `Some(n)`，循环继续；没有元素时返回 `None`，循环结束。

等价写法：

```rust
loop {
    match values.pop() {
        Some(n) => println!("{}", n),
        None => break,
    }
}
```

`while let` 适合不断获取下一个值、直到没有值为止的流程，如弹出栈顶、读取迭代器、接收通道消息等。

## 与基础控制流的关系

| 形式 | 场景 | 等价 match 形式 |
|---|---|---|
| `if let` | 只关心一种成功情况 | 单分支 + `_ => {}` |
| `let else` | 匹配失败时提前退出 | 成功分支后接主流程，失败分支 return/break |
| `while let` | 持续匹配直到失败 | `loop { match ... { Some(v) => ..., None => break } }` |

这三种形式都是 `match` 的语法糖，编译后没有运行时开销差异。选择哪种形式取决于可读性：代码意图越明确，维护成本越低。
