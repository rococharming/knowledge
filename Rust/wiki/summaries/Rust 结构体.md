# Rust 结构体

本文系统梳理 Rust 结构体的定义形式、实例化语法、可见性、方法系统与内存布局，是理解 Rust 自定义类型的基础。

## 核心主题

### 三种结构体形式

Rust 提供三种结构体：具名字段结构体（`struct User { name: String, age: u32 }`）、元组结构体（`struct Color(i32, i32, i32)`）和单元结构体（`struct Marker;`），分别适用于字段有命名需求、简单语义区分、零大小标记等场景。详见 [[结构体类型]]。

### 实例化与所有权

结构体实例化支持字段初始化简写（变量名与字段名相同时可省略）和结构体更新语法（`..old_instance` 补齐未显式指定的字段）。更新语法中非 `Copy` 字段会被移动，导致旧实例不可整体使用。这与 [[移动]] 和 [[Copy]] 机制直接关联。

### 可见性

结构体和字段默认私有，需 `pub` 逐层公开。即使结构体本身 `pub`，字段仍私有——这是标准库类型（如 `String`、`Vec`）隐藏内部实现、仅通过关联函数构造的设计基础。

### 方法与关联函数

方法定义在 `impl` 块中，首个参数为 `self`/`&self`/`&mut self` 决定所有权行为。调用时 Rust 自动借用。关联函数无 `self` 参数，常用于构造（如 `Queue::new()`），`Self` 是当前类型的别名。详见 [[方法与关联函数]]。

### 关联常量

`impl` 块中可定义关联常量（`const NAME: &str = "..."`），类型不必是当前结构体类型，只是挂在该类型命名空间下的常量。

### 内存布局

默认 `repr(Rust)` 不保证字段排列顺序。需要稳定布局（如 FFI 场景）时使用 `#[repr(C)]`。详见 [[结构体内存布局]]。
