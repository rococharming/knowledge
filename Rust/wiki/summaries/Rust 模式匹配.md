# Rust 模式匹配

模式匹配是 Rust 中核心的控制流与数据解构机制。它不仅能判断值属于哪种情况，还能在匹配成功时提取内部数据并绑定到变量上，实现"拆包"与"分支"的合二为一。

## 核心内容概览

### match 表达式

`match` 是 Rust 最完整的模式匹配形式，要求**穷尽性检查**（exhaustiveness checking）——必须覆盖所有可能情况。配合 `_` 通配符可作为兜底分支。

```rust
match value {
    pattern1 => expression1,
    pattern2 => expression2,
    _ => default_expression,
}
```

### 模式类型

Rust 支持多种模式形式，用于匹配不同形状的数据：

- **字面量、变量与通配符**：精确匹配、任意值捕获、忽略值
- **元组模式**：同时匹配多个位置 `(x, y)`
- **结构体模式**：按字段名匹配和绑定 `Point { x, y }`
- **枚举模式**：根据变体种类分别处理并取出数据
- **数组与切片模式**：按固定位置或可变长度匹配 `[first, .., last]`

详见 [[模式类型]]。

### 绑定与借用

模式匹配不只是判断，还会绑定变量并遵守所有权规则：

- 默认绑定可能发生**移动**
- `ref` / `ref mut` 将绑定改为**借用**
- `&pat` 引用模式可"解开一层"引用

详见 [[模式绑定与借用]]。

### 组合模式

通过组合机制表达更复杂的匹配条件：

- **或模式** `|`：一个分支匹配多种情况
- **范围模式** `..=`：匹配区间内的值
- **匹配守卫** `if`：模式匹配后附加条件判断
- **@ 绑定**：匹配模式的同时保留整个值

详见 [[组合模式]]。

### 简洁控制流

Rust 提供三种模式匹配的简洁形式：

- **if let**：只关心一种匹配情况
- **let else**：匹配失败就提前退出
- **while let**：循环中持续匹配

详见 [[简洁控制流]]。

## 模式出现的位置

模式不仅出现在 `match` 中，还广泛用于：

- 变量绑定：`let (x, y) = (10, 20)`
- 函数参数：`fn print_point((x, y): (i32, i32))`
- `for` 循环：`for (number, text) in pairs`
