本文是 Rust 所有权系统学习笔记的整体摘要，涵盖内存管理三条路线、所有权三大规则、作用域与 Drop、移动语义、Copy 类型以及函数与复合类型中的所有权转移。

## 核心内容概览

### 内存管理三条路线

| 路线 | 代表语言 | 机制 | 特点 |
|---|---|---|---|
| 垃圾回收（GC） | Java、Python | 运行时追踪回收 | 开发轻松，但有运行时开销与暂停 |
| 手动管理 | C/C++ | `malloc/free`、`new/delete` | 控制强，但易出内存安全问题 |
| 编译期所有权系统 | Rust | **所有权 + 借用 + 生命周期** | 编译期静态检查，不依赖 GC |

### 所有权三大规则

1. 每一个值都有一个所有者
2. 同一时刻，一个值只有一个所有者
3. 当所有者离开作用域，值被丢弃

### 核心机制

- **作用域与 Drop**：变量进入 `{}` 块后可用，离开作用域自动触发 `drop` 释放资源。遵循 RAII 原则。
- **移动（Move）**：非 `Copy` 类型赋值/传参时所有权转移，原绑定失效。详见 [[移动]]
- **Copy 类型**：实现了 `Copy` trait 的类型按位复制，原变量仍可用。详见 [[Copy]]
- **借用（Borrow）**：通过 `&T` / `&mut T` 临时访问值而不取得所有权。详见 [[引用与借用]]
- **共享所有权**：`Rc` / `Arc` 通过引用计数实现多所有者共享

### String 内存模型

`String` 在栈上保存 `ptr`、`len`、`cap` 三个字段，堆上存放 UTF-8 字节序列。离开作用域时自动释放堆内存。

### 函数中的所有权转移

- 非 `Copy` 类型作为实参传递 → 所有权移入函数
- 函数返回非 `Copy` 类型 → 所有权移回调用者
- `Copy` 类型传参 → 复制一份，原变量仍可用

### 移动与控制流

- `if/else` 中只要任一分支可能移动，移动后变量即失效
- 循环中不允许反复移动同一个非 `Copy` 值

### 从复合类型中移动

- 禁止直接从 `Vec` 索引中移动非 `Copy` 值（会留下空洞）
- 常用取出方式：`pop()`、`swap_remove(i)`、`std::mem::replace()`
- `for s in v` 会消耗 `Vec` 的所有权
