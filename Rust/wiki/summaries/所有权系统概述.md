# 所有权系统概述

本文梳理了 Rust 所有权系统的核心机制，涵盖从基础规则到高级共享策略的完整脉络。

## 核心观点

Rust 通过**编译期所有权系统**实现内存安全，既不依赖 GC 也不依赖程序员自律。系统由三大支柱构成：

1. **所有权规则** — 每个值有唯一所有者，所有者离开作用域时自动释放资源（RAII）
2. **移动语义** — 值的所有权可在绑定间转移，避免重复释放和悬垂访问
3. **共享所有权** — `Rc<T>` / `Arc<T>` 在严格限制下实现多所有者引用计数
