Rust 中的字符串处理围绕 UTF-8 编码展开，核心类型为 `String`（拥有所有权）和 `&str`（借用切片），辅以多种场景专用类型。

## 核心内容

### 字符串字面量

Rust 支持多种字符串字面量形式：普通字符串（支持转义和多行）、原始字符串 `r"..."`（免转义）、带 `#` 边界的原始字符串 `r#"..."#`、字节串 `b"..."` 和原始字节串 `br"..."`。详见 [[字符串类型]]。

### 内存中的字符串

Rust 字符串始终使用 UTF-8 编码。`String` 在栈上保存 ptr/len/cap 三元组，数据存放于堆；`&str` 是一个胖指针，包含数据起始地址和字节长度。字符串字面量的类型为 `&'static str`，数据硬编码在程序二进制中。详见 [[字符串类型]]。

### 创建与修改

`String` 可通过 `String::from`、`to_string`、`to_owned`、`format!`、`concat`/`join` 等方式创建。`String` 是可变的，支持 `push`、`push_str`、`clear` 等方法；`&str` 不可直接修改。详见 [[字符串类型]]。

### 索引与遍历

Rust 字符串不能直接使用下标访问字符（因为 UTF-8 可变长度）。应按字符遍历时使用 `chars()`，按字节遍历时使用 `bytes()`。字符串切片必须落在合法的 UTF-8 字符边界上，否则会 panic。详见 [[字符串类型]]。

### 其他文本类型

除 `String` 和 `&str` 外，Rust 还提供了针对不同场景的专用类型：`PathBuf`/`&Path`（文件路径）、`Vec<u8>`/`&[u8]`（二进制数据）、`OsString`/`&OsStr`（操作系统字符串）、`CString`/`&CStr`（C 接口）。详见 [[字符串相关类型]]。
