# 共享所有权

当单一所有者难以满足值的寿命需求时，Rust 提供**引用计数指针**实现多所有者共享。当最后一个所有者离开作用域时，值才被释放。

## Rc 与 Arc 的对比

| 类型 | 全称 | 线程安全 | 计数方式 | 使用场景 |
|------|------|---------|---------|---------|
| `Rc<T>` | Reference Counted | 否（单线程） | 非原子操作 | 单线程内共享数据 |
| `Arc<T>` | Atomic Reference Counted | 是（多线程） | 原子操作 | 跨线程共享数据 |

**优先使用 `Rc`**：不需要跨线程时避免 `Arc` 的原子操作开销。若误将 `Rc` 发送到其他线程，编译器会报错。

## 使用方式

`Rc<T>` 和 `Arc<T>` 的 API 完全一致。以 `Rc` 为例：

```rust
use std::rc::Rc;

let s: Rc<String> = Rc::new("hello".to_string());
let t: Rc<String> = Rc::clone(&s); // 只增加引用计数，不克隆数据
let u: Rc<String> = Rc::clone(&s);
```

此时 `s`、`t`、`u` 三个指针指向同一块内存，堆上同时保存：
- 引用计数
- `String` 本体（`ptr`/`len`/`cap`）

当所有 `Rc` 指针离开作用域，引用计数归零，`String` 及其指向的字符数据依次被释放。

## Deref 体验

`Rc<T>` 实现了 `Deref`，可以像使用引用一样直接调用目标类型的方法：

```rust
assert!(s.contains("he"));
assert_eq!(s.find("he"), Some(0));
println!("{}", s);
```

## 共享所有权 ≠ 共享可变性

`Rc<T>` 提供共享所有权，但**默认不提供共享可变性**。不能通过 `Rc<T>` 直接获得 `&mut T`：

```rust
let s = Rc::new(String::from("hello"));
// s.push_str(" world"); // 编译错误：不能获得可变引用
```

若需"多所有者 + 可变"，通常配合：
- **单线程**：`Rc<RefCell<T>>`（[[内部可变性]]）
- **多线程**：`Arc<Mutex<T>>` 或 `Arc<RwLock<T>>`（同步原语）
