结构体（struct）是 Rust 中定义自定义数据类型的核心机制，用于将多个相关值组合为一个整体。Rust 提供三种结构体形式：具名字段结构体、元组结构体和单元结构体。

## 具名字段结构体

最常用的形式，每个字段有名称和类型：

```rust
struct User {
    name: String,
    age: u32,
}
```

定义时无需分号，字段间用逗号分隔。

### 实例化

使用结构体表达式创建实例，花括号中列出字段名与值：

```rust
let user = User {
    name: String::from("spf"),
    age: 25,
};
```

字段顺序不必与定义一致，但建议保持一致以利于阅读。

打印结构体需 `#[derive(Debug)]`，使用 `{:?}` 或 `{:#?}` 格式化。

### 访问与修改

用 `.` 访问字段（`user.name`）。修改字段要求整个变量声明为 `mut`——Rust 的可变性是变量级别的，不能只让单个字段可变。

### 字段初始化简写

当变量名与字段名相同时可省略赋值：

```rust
fn build_user(name: String, age: u32) -> User {
    User { name, age } // 等价于 User { name: name, age: age }
}
```

### 结构体更新语法

基于已有实例创建新实例，未指定的字段从旧实例补齐：

```rust
let user2 = User {
    name: String::from("John"),
    ..user1 // age 和其他字段来自 user1
};
```

更新语法的所有权行为取决于字段类型：

- [[Copy]] 类型字段：从旧实例**复制**，旧实例仍可用
- 非 Copy 类型字段：从旧实例**移动**，旧实例整体不可用

> [!warning] 移动后慎用旧实例
> `..user1` 移动了 `user1.name`（`String` 不是 Copy 类型），因此 `user1` 不能再整体使用。仅访问未被移动的 Copy 字段（如 `user1.age`）在某些情况下仍合法。

## 元组结构体

字段无名称，只有类型和位置索引：

```rust
struct Color(i32, i32, i32); // RGB
struct Point(i32, i32, i32); // 三维坐标
```

`Color` 和 `Point` 内部类型完全相同，但它们是**不同类型**，不会被混用。这是元组结构体与普通元组的关键区别：拥有独立类型名，语义更清晰。

### 实例化与访问

```rust
let black = Color(0, 0, 0); // 定义元组结构体时 Rust 自动生成同名构造函数
println!("{}", black.0);     // 用数字索引访问
```

### 新类型模式

单字段元组结构体常用于区分同底层类型的不同语义：

```rust
struct Millimeters(u32);
struct Meters(u32);
struct UserId(u64);
struct OrderId(u64);
```

虽然内部都是 `u32` 或 `u64`，但类型不同，函数签名可精确约束参数——`find_user(id: UserId)` 不会误接收 `OrderId`。

元组结构体的单个元素可以单独标注可见性。

## 单元结构体

不含任何字段的结构体：

```rust
struct Marker;
let m = Marker; // 直接写类型名即可创建值
```

不携带运行时数据，是零大小类型（ZST）。适合表达"类型本身有意义但不需要存储数据"的场景：

- **标记类型**：在泛型或 trait 系统中标识特定策略或行为
- **行为载体**：仅为挂载方法而存在

```rust
struct ConsoleLogger;
impl ConsoleLogger {
    fn log(&self, message: &str) { println!("{}", message); }
}
```

为单元结构体实现 trait 可表达策略模式，详见 [[方法与关联函数]]。

## 可见性

结构体和字段默认**私有**，仅当前模块及子模块可访问。`pub` 逐层公开：

| 声明 | 效果 |
|---|---|
| `pub struct User { ... }` | 类型对外可见，但字段仍私有 |
| `pub struct User { pub name: String, age: u32 }` | 类型与 `name` 字段公开，`age` 仍私有 |
| `pub struct User { pub name: String, pub age: u32 }` | 全部公开 |

私有字段 + 公开关联函数是标准库常见设计（`String::new()`、`Vec::new()`），隐藏内部实现只暴露安全接口。
