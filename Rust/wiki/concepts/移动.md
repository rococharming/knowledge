# 移动

Rust 中对于**大多数非 Copy 类型**，赋值、函数传参、返回值等操作默认发生的是**移动（Move）**而非复制。移动将值的所有权从源绑定转移到目标绑定，源绑定此后不可再用。

## 移动的基本行为

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // Move：s1 的所有权转移给 s2
    // println!("s1: {}", s1); // 编译错误！s1 已失效
    println!("s2: {}", s2);
}
```

移动后 `s1` 和 `s2` 的栈字段内容相同（`ptr`/`len`/`cap` 被按位拷贝），但 Rust 将 `s1` 标记为无效，避免两者离开作用域时的**二次释放**。

### 需要深拷贝的场景

若需保留原变量，使用 `clone`：

```rust
let s1 = String::from("Hello");
let s2 = s1.clone(); // 深拷贝：分配独立堆内存
println!("s1: {}", s1); // 合法
println!("s2: {}", s2);
```

## Copy 类型：移动的例外

实现 `Copy` trait 的类型赋值时发生按位复制，原变量仍可用。标准 `Copy` 类型包括：

- 标量类型：整型、浮点型、布尔型、字符型
- 元素均为 `Copy` 的数组和元组
- 不可变引用类型 `&T`

**不能是 `Copy` 的类型**：`String`（需释放堆内存）、`Vec<T>`、文件句柄、`MutexGuard` 等——任何丢弃时需要特殊操作的类型。

结构体和枚举默认不是 `Copy`，若所有字段均为 `Copy`，可通过 `#[derive(Copy, Clone)]` 启用：

```rust
#[derive(Copy, Clone)]
struct Point {
    x: i32,
    y: i32,
}
```

## 函数中的移动

### 传参时移动

```rust
fn take_ownership(some_string: String) {
    println!("{}", some_string);
} // some_string 离开作用域，drop

fn main() {
    let s = String::from("Hello");
    take_ownership(s); // s 的所有权移入函数
    // println!("{}", s); // 错误
}
```

### 返回值时移动

```rust
fn gives_ownership() -> String {
    let s = String::from("Hello");
    s // 所有权转移给调用者
}

fn takes_and_gives(s: String) -> String {
    s // 所有权再次转移给调用者
}
```

## 控制流中的移动

### if/else

```rust
let s = String::from("hello");
if flag {
    f(s); // 分支 A 移动 s
} else {
    g(s); // 分支 B 移动 s
}
// h(s); // 错误：s 已变为未初始化状态
```

只要**所有路径都移动了 s**，编译器接受；但只要有一条路径没有移动，s 在分支后仍可用。

### 循环

```rust
let s = String::from("hello");
while true {
    f(s); // 编译错误：第一次迭代后 s 已失效
}
```

Rust 禁止在循环中直接移动同一变量。

## 从复合类型中移动值

直接从索引或字段移动非 `Copy` 值会在原地留下空洞，Rust 禁止此行为：

```rust
let v = vec![String::from("hello"), String::from("world")];
// let first = v[0]; // 编译错误
```

### 安全的移出方法

| 方法 | 说明 | 向量变化 |
|------|------|---------|
| `pop()` | 弹出末尾元素 | 缩短 |
| `swap_remove(i)` | 与末尾交换后弹出 | 缩短，不保证顺序 |
| `std::mem::replace(&mut v[i], new_val)` | 替换后返回旧值 | 长度不变 |

```rust
let mut v = vec!["a".to_string(), "b".to_string(), "c".to_string()];
let last = v.pop().unwrap();           // "c"，v 变短
let second = v.swap_remove(0);         // "a"，与末尾交换
let first = std::mem::replace(&mut v[0], "x".to_string()); // "b"
```

### 遍历中移动

`for s in v` 会将向量所有权移入循环：

```rust
let v = vec!["a".to_string(), "b".to_string()];
for s in v {
    println!("{}", s);
}
// v 已失效
```

如需保留 `v`，使用引用：`for s in &v` 或 `for s in &mut v`。
