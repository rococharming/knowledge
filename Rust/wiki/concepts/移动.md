移动（Move）是 Rust 所有权系统的默认行为：对于非 `Copy` 类型，赋值、传参、返回时发生的是所有权转移，而非复制。

## 移动的基本概念

移动将值的所有权从源绑定转移到目标绑定。移动后源绑定在语义上失效，编译器禁止继续使用它。新目标绑定负责该值的生命周期，并在适当时机触发 `drop`。

以 `String` 为例：

```rust
let s1 = String::from("hello");
let s2 = s1; // Move
// println!("{}", s1); // 编译报错，s1 已经失效
println!("{}", s2);
```

Rust 的移动不是运行时把堆数据重新复制一份，而是转移栈上的 `String` 结构（`ptr`、`len`、`cap`）。堆上的字符串数据仍是同一份。让 `s1` 失效是为了避免两者都以为自己拥有同一块堆内存，从而导致二次释放。

## Clone（显式克隆）

如果需要保留原变量，应显式调用 `clone`：

```rust
let s1 = String::from("Hello");
let s2 = s1.clone(); // 克隆
println!("{}", s1); // 正确
```

![[Image 35.png|400]]

`clone` 会分配新的堆内存并复制内容，成本较高。Rust 要求显式写出 `clone`，使这种成本在代码中可见。

## 函数中的所有权转移

### 函数传参

非 `Copy` 类型作为实参传递给函数时，所有权发生转移：

```rust
fn main() {
    let s = String::from("Hello");
    take_ownership(s); // s 移动进函数，此后 s 失效
    // println!("{}", s); // 错误
}

fn take_ownership(some_string: String) {
    println!("{}", some_string);
} // some_string 离开作用域，String 被 drop
```

### 函数返回值

函数返回非 `Copy` 类型的值时，所有权转移给调用者：

```rust
fn gives_ownership() -> String {
    let s = String::from("Hello");
    s // 所有权转移给调用者
}
```

## 移动与控制流

### if/else 中的移动

编译器会分析所有分支，只要存在移动的可能性，移动后变量即失效：

```rust
let s = String::from("hello");
if true {
    consume(s);
} else {
    consume(s);
}
// println!("{}", s); // 错误，s 已经被移动
```

### 循环中的移动

不允许在可能执行多次的循环中反复移动同一个非 Copy 值：

```rust
let s = String::from("hello");
loop {
    consume(s); // 错误，第一次循环后 s 已经被移动
}
```

如需在循环中使用，应使用借用。

## 从复合类型中移动值

### 不能直接从 Vec 索引中移动

Rust 禁止从索引表达式中直接移动非 `Copy` 值，因为这会在原位置留下空洞，导致容器处于不完整状态：

```rust
let v = vec![String::from("hello"), String::from("world")];
// let first = v[0]; // 错误
```

若 `Vec` 中存放的是 `Copy` 类型（如 `i32`），则没有问题，因为实际发生的是复制而非移动。

### 从 Vec 取出元素的常见方式

| 方法 | 行为 | 向量变化 |
|---|---|---|
| `pop()` | 取出末尾元素 | 长度减 1 |
| `swap_remove(i)` | 将目标与末尾交换后取出 | 长度减 1，顺序改变 |
| `std::mem::replace(&mut v[i], new_val)` | 替换原槽位，取出原值 | 长度不变 |

### for 循环消耗 Vec

`for s in v` 会调用 `into_iter`，将 `v` 的所有权移入迭代器。循环结束后 `v` 不再可用：

```rust
let v = vec![String::from("hello"), String::from("world")];
for s in v {
    println!("{}", s);
}
// println!("{:?}", v); // 错误，v 已经被移动
```
