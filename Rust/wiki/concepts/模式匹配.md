# 模式匹配

模式匹配是 Rust 中用于**判断值的形状、检查条件并提取数据**的核心机制。表达式用来创建值，模式用来拆开值。`match` 是最完整的模式匹配形式。

## match 表达式

### 基本形式

```rust
match value {
    pattern1 => expression1,
    pattern2 => expression2,
    pattern3 => expression3,
}
```

每个分支由模式和表达式组成。`match` 本身也是表达式，各分支返回类型必须一致。

```rust
fn describe_number(n: i32) -> &'static str {
    match n {
        0 => "zero",
        1 => "one",
        _ => "other",
    }
}
```

分支中需要多条语句时使用代码块：

```rust
match n {
    0 => println!("zero"),
    x => {
        println!("number:");
        println!("{}", x);
    }
}
```

### 匹配顺序

`match` 从上到下依次尝试每个分支，**第一个匹配成功的分支被执行**，后续不再检查。因此更具体的模式要放在前面。

```rust
let x = 10;

match x {
    n => println!("{}", n),  // 变量模式匹配任何值
    1 => println!("one"),     // 永远不可达
}
```

编译器会发出 `unreachable pattern` 警告。

## 穷尽性检查

Rust 要求 `match` 必须覆盖所有可能情况，这称为**穷尽性检查**（exhaustiveness checking）。

```rust
// 错误：未覆盖所有 i32 值
match n {
    0 => "zero",
    1 => "one",
}
```

使用 `_` 通配符作为兜底分支即可满足检查：

```rust
match n {
    0 => "zero",
    1 => "one",
    _ => "other",
}
```

穷尽性检查是 Rust 模式匹配的重要安全机制，尤其适合处理枚举，能有效减少遗漏分支造成的逻辑错误。

## 模式出现的位置

模式不仅出现在 `match` 中，还渗透在 Rust 的许多语法里。

### 变量绑定

`let` 的左侧就是模式：

```rust
let (x, y) = (10, 20);
```

### 函数参数

函数参数中也可使用模式拆解元组：

```rust
fn print_point((x, y): (i32, i32)) {
    println!("({}, {})", x, y);
}
```

### for 循环

`for` 循环常用模式拆开元素：

```rust
let pairs = vec![(1, "one"), (2, "two")];

for (number, text) in pairs {
    println!("{} = {}", number, text);
}
```

这些写法本质上都是在用模式拆解值。`match` 是最完整的形式，其他语法则是在特定场景下使用模式。
