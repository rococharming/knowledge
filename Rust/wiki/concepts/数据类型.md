# 数据类型

Rust 是一种**静态强类型语言**，所有变量在编译期必须具有确定类型。类型分为标量类型（单个值）和复合类型（多个值的组合）。

## 标量类型

标量类型表示单个值，包括整型、浮点型、布尔型和字符型。

### 整型

Rust 的整型系统提供丰富的类型选择、灵活的字面量表达以及四种显式的溢出处理策略。

#### 整型类型表

| 长度 | 有符号 | 无符号 |
|---|---|---|
| 8-bit | `i8` | `u8` |
| 16-bit | `i16` | `u16` |
| 32-bit | `i32` | `u32` |
| 64-bit | `i64` | `u64` |
| 128-bit | `i128` | `u128` |
| 架构相关 | `isize` | `usize` |

- `isize` / `usize` 的位数与目标架构地址空间一致（32 位或 64 位）
- **Rust 要求数组索引必须是 `usize`**
- **默认整型字面量为 `i32`**

#### 字面量写法

```rust
let a = 1_000;       // 十进制，下划线分隔提高可读性
let b = 0xffff_ffff; // 十六进制前缀 0x
let c = 0o12;        // 八进制前缀 0o
let d = 0b1110_0011; // 二进制前缀 0b
let e = 10u32;       // 类型后缀
```

**字节字符字面量**：为 `u8` 提供的特殊字面量，以 `b` 为前缀：

```rust
let a = b'A'; // u8 类型，值为 65
```

注意：`b'A'` 是 `u8`，而 `'A'`（不加 `b`）是 `char`（4 字节 Unicode）。

#### 溢出处理

整型有固定范围（如 `u8` 为 0~255）。**编译期可确定的常量表达式超范围会直接报错**；运行期溢出的行为取决于构建模式：

| 构建模式 | 默认行为 |
|---|---|
| Debug | 检查溢出，触发 **panic** |
| Release | 二进制补码回绕（wrapping） |

为统一和显式控制溢出，Rust 提供四种方法（以 `add` 为例，其他运算类似）：

| 方法 | 返回类型 | 溢出行为 | 示例 `254u8 + 2` |
|---|---|---|---|
| `checked_add` | `Option<T>` | 溢出时为 `None` | `None` |
| `wrapping_add` | `T` | 按补码规则回绕 | `0` |
| `saturating_add` | `T` | 钳制到类型边界 | `255` |
| `overflowing_add` | `(T, bool)` | 返回结果和溢出标志 | `(0, true)` |

### 浮点型

Rust 提供 IEEE 754 标准的单精度 `f32` 和双精度 `f64` 两种浮点类型，分别对应 C/C++ 的 `float` 和 `double`。

| 类型 | 精度 | 对应 C/C++ |
|---|---|---|
| `f32` | 单精度（32-bit） | `float` |
| `f64` | 双精度（64-bit） | `double` |

**默认浮点字面量类型为 `f64`**：

```rust
let x = 2.0;       // f64
let y: f32 = 3.14; // f32（显式注解）
let z = 5f32;      // f32（类型后缀）
```

#### 特殊值与数学常量

`f32` 和 `f64` 提供 IEEE 754 要求的特殊值关联常量：

| 常量 | 含义 |
|---|---|
| `INFINITY` | 正无穷大 |
| `NEG_INFINITY` | 负无穷大 |
| `NAN` | 非数值（Not a Number） |
| `MIN` | 最小有限值（最负） |
| `MAX` | 最大有限值 |

**推荐写法**：使用原生类型关联常量，如 `f32::NAN`、`f64::INFINITY`。

常用数学常量位于 `std::f32::consts` / `std::f64::consts`：

```rust
let pi = std::f32::consts::PI;
```

#### 注意事项

- 浮点型与整型之间转换必须使用 `as`，如 `3.14 as i32`（截断为 3）
- 浮点数比较应避免直接使用 `==`，因为精度问题
- 默认 `f64` 在多数场景下更安全，仅在性能敏感或内存受限时选择 `f32`

### 布尔型

只有两个取值：`true` 和 `false`。

```rust
let x = 1 < 2;     // bool，值为 true
let y: bool = false;
```

#### 严格类型检查

与 C/C++ 不同，**Rust 不允许将其他类型隐式转换为 bool**：

```rust
// C/C++ 中可以写 if(x)，Rust 中必须显式比较：
if x != 0 { ... }  // OK
if x { ... }       // error：x 不是 bool 类型
```

`if`、`while` 等控制结构的条件**必须是 bool 表达式**。

#### 与整型的转换

- `bool` → 整数：允许使用 `as`，`true` → 1，`false` → 0
- 整数 → `bool`：**不允许**，`1 as bool` 是非法的

#### 内存布局

虽然信息量上 `bool` 只需 1 bit，但 Rust 中 `bool` 的大小为 **1 字节**，以便像其他标量一样寻址、创建引用和指针。

### 字符型

Rust 的 `char` 类型占 **4 个字节**，表示一个 **Unicode 标量值**（Unicode Scalar Value）。

```rust
let c = 'z';
let z: char = '中';
let smile = '😊';
```

#### 关键特性

- 支持任何 Unicode 字符：英文字母、汉字、表情符号等
- 使用**单引号** `' '` 括起来
- 占 4 字节，范围 U+0000 到 U+D7FF 和 U+E000 到 U+10FFFF

#### 与字符串的区别

**重要**：Rust 对**单个字符**使用 `char`，但对**字符串**（`String` / `&str`）内部使用 **UTF-8 字节序列**存储，而非 `char` 数组。

这意味着：

- **不能直接用索引访问字符串的第 n 个字符**（不是 O(1) 操作）
- 需要遍历字符时应使用 `.chars()` 迭代器

```rust
let s = "hello";
// s[0] // 错误！Rust 不允许字符串索引
for c in s.chars() { ... } // 正确方式
```

## 复合类型

复合类型可以将多个值组合成一个类型。Rust 的原生复合类型包括数组和元组。

### 数组

数组中每个元素类型必须相同，一旦定义长度固定不变（长度是类型的一部分）。

```rust
let arr1 = [1, 2, 3, 4, 5];      // [i32; 5]
let arr2: [f64; 3] = [1.0, 2.0, 3.0]; // [f64; 3]
let arr3 = [10; 3];              // [10, 10, 10]，快捷写法
```

#### 类型标注

数组类型格式为 `[元素类型; 元素个数]`，如 `[u32; 5]` 表示包含 5 个 `u32` 的数组。

#### 元素访问

通过索引访问，索引值可以是变量：

```rust
let arr = [1, 2, 3, 4, 5];
let first = arr[0];
let second = arr[1];
```

**越界检查**：

- 编译期可确定的常量索引越界 → **编译错误**
- 运行时确定的索引越界 → **运行时 panic**

#### 打印输出

数组未实现 `Display` trait，需用 `Debug` 格式打印：

```rust
println!("{:?}", arr);   // 单行输出
println!("{:#?}", arr);  // 美化多行输出
```

### 元组

元组中的每个元素可以是不同类型，长度同样固定。

```rust
let tup1 = (10, 3.14);           // (i32, f64)
let tup2: (bool, char) = (true, 'A');
```

#### 元素访问

通过 `.索引` 访问，索引必须是编译期常量：

```rust
let tup = (10, 3.14);
println!("{}, {}", tup.0, tup.1);
```

#### 模式匹配解构

```rust
let tup = (10, 3.14);
let (x, y) = tup; // x = 10, y = 3.14
```

#### 单元元组（Unit Tuple）

无内容的元组 `()` 称为单元元组，其类型和值都是 `()`：

```rust
let unit = (); // () 类型
```

单元元组表示"没有有意义的返回值"。函数体最后没有返回表达式（或以 `;` 结尾）时，默认返回 `()`。

### 数组 vs 元组对比

| 特性 | 数组 | 元组 |
|---|---|---|
| 元素类型 | 必须相同 | 可以不同 |
| 类型标注 | `[T; N]` | `(T1, T2, ...)` |
| 访问方式 | `[索引]`（运行时变量） | `.索引`（编译期常量） |
| 解构 | 不支持直接解构 | 支持模式匹配解构 |
| 长度 | 编译期确定 | 编译期确定 |
| 典型用途 | 同类型数据集合 | 异构数据组合、多返回值 |

## 类型转换

Rust **无隐式类型转换**，所有类型转换必须显式使用 `as`：

```rust
let a = 3.14 as i32; // a = 3，浮点截断
let b = true as u8;  // b = 1
```

注意：`bool` → 整数允许，但整数 → `bool` 不允许。
