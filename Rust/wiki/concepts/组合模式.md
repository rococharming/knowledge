# 组合模式

Rust 允许通过组合机制表达更复杂的匹配条件，包括或模式、范围模式、匹配守卫和 @ 绑定。

## 或模式

用 `|` 表示一个分支可以匹配多种情况，适合合并处理逻辑相同的分支。

```rust
fn is_vowel(c: char) -> bool {
    match c {
        'a' | 'e' | 'i' | 'o' | 'u' => true,
        _ => false,
    }
}
```

另一个例子：

```rust
fn at_line_end(ch: Option<&char>) -> bool {
    match ch {
        Some(&'\n') | Some(&'\r') | None => true,
        _ => false,
    }
}
```

## 范围模式

用于匹配某个区间内的值。

```rust
fn classify(c: char) -> &'static str {
    match c {
        '0'..='9' => "digit",
        'a'..='z' | 'A'..='Z' => "letter",
        ' ' | '\n' | '\t' | '\r' => "whitespace",
        _ => "other",
    }
}
```

范围语法：

- `start..=end`：包含 `end` 的闭区间
- `start..end`：不包含 `end` 的开区间
- `start..`：从 `start` 到该类型最大值的范围

```rust
fn level(score: u32) -> &'static str {
    match score {
        0..=59 => "fail",
        60..=89 => "pass",
        90.. => "excellent",
    }
}
```

范围模式常用于数字和字符分类。注意：模式中的范围是**匹配条件**，与表达式中的范围迭代器不是一回事。

## 匹配守卫

在模式后面加 `if` 条件，表达更细的判断。

```rust
fn classify(c: char) -> &'static str {
    match c {
        '0'..='9' => "digit",
        'a'..='z' | 'A'..='Z' => "letter",
        c if c.is_whitespace() => "whitespace",
        _ => "other",
    }
}
```

`c if c.is_whitespace()` 是带匹配守卫的模式：先把当前字符绑定到变量 `c`，再判断 `c.is_whitespace()`。只有条件成立时才进入该分支。

### 语法与匹配流程

```rust
PATTERN if CONDITION => EXPR
```

匹配流程：

1. 判断值是否匹配 `PATTERN`
2. 若匹配成功，再计算 `CONDITION`
3. 只有条件为真才执行该分支；否则视为不匹配，继续尝试下一个分支

### 与嵌套 if 的比较

以下代码：

```rust
match clicked {
    None => Err("not on a page"),
    Some(p) => {
        if p == current { Err("you clicked the current page") }
        else { Ok(p) }
    }
}
```

用匹配守卫可改写为更扁平的形式：

```rust
match clicked {
    None => Err("not on a page"),
    Some(p) if p == current => Err("you clicked the current page"),
    Some(p) => Ok(p),
}
```

匹配守卫适合处理**结构匹配成功后还要额外判断值**的场景。

## @ 绑定

在匹配某个模式的同时，把匹配到的整个值绑定到变量上。

```rust
x @ pattern
```

示例：

```rust
fn read_token(c: char) -> String {
    match c {
        digit @ '0'..='9' => format!("digit: {}", digit),
        _ => "other".to_string(),
    }
}
```

`digit @ '0'..='9'` 表示：先判断 `c` 是否是数字字符，如果是，就把这个字符整体绑定到 `digit`。

@ 绑定适合**既要检查模式，又要保留匹配的值**的场景：

```rust
#[derive(Debug)]
enum Msg {
    Text(String),
    Ping,
}

fn handle(msg: &Msg) {
    println!("{:?}", msg);
}

fn demo(msg: Msg) {
    match msg {
        whole @ Msg::Text(_) => handle(&whole),
        Msg::Ping => {}
    }
}
```

`whole @ Msg::Text(_)` 匹配 `Msg::Text(...)` 的同时，把整个值绑定到 `whole`，以便后续传给需要引用的函数。

若被绑定的值不是 `Copy` 类型，@ 绑定同样遵守所有权规则，可能发生移动。
