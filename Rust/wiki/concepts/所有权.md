# 所有权

Rust 所有权系统的核心规则：每个值有且仅有一个所有者，所有者离开作用域时值被自动丢弃。

## 内存管理范式对比

| 方式 | 代表语言 | 机制 | 优点 | 缺点 |
|------|---------|------|------|------|
| 垃圾回收 | Java、Python | 运行时追踪不可达对象 | 开发体验轻松 | 运行时开销、暂停/延迟 |
| 手动管理 | C/C++ | 程序员负责 malloc/free | 控制力强 | 易泄漏、悬垂、二次释放 |
| 编译期所有权 | **Rust** | 编译器静态检查 | 零运行时开销 + 内存安全 | 学习曲线陡峭 |

## 作用域与 RAII

变量作用域由 `{}` 块界定。Rust 遵循 **RAII** 思想：值离开作用域时自动调用析构逻辑（`drop`，由 `Drop` trait 参与）。

- **栈上纯数据**：离开作用域即弹栈
- **拥有堆内存/系统资源的值**：`drop` 时释放堆内存、关闭文件句柄、释放锁等

```rust
fn main() {
    let s = String::from("Hello");
} // s 离开作用域 → drop → 释放堆缓冲区
```

## 所有权三大规则

1. **每一个值都有一个所有者（owner）**
2. **同一时刻，一个值有且仅有一个所有者**（共享需特殊类型如 `Rc<T>`）
3. **当所有者离开作用域，值会被丢弃（drop）**

## String 的所有权示例

`String` 是理解所有权的典型类型。它在栈上保存三个字段，堆上存放 UTF-8 字节序列：

```rust
fn main() {
    let s = String::from("Hello"); // s 获得堆内存所有权
} // s 离开作用域 → drop 释放堆缓冲区
```

栈上结构：
- `ptr` — 指向堆缓冲区的指针
- `len` — 当前长度
- `cap` — 当前容量

## 所有权放宽方式

| 机制 | 说明 | 典型场景 |
|------|------|---------|
| **移动（Move）** | 所有权从源绑定转移到目标绑定，源绑定失效 | 变量赋值、函数传参、返回值 |
| **复制（Copy）** | 对实现 `Copy` 的类型按位复制，原变量仍可用 | 整型、浮点型、布尔型、字符型 |
| **借用（Borrow）** | 通过 `&T` / `&mut T` 临时访问而不取得所有权 | 只读/可变引用 |
| **共享所有权** | `Rc<T>` / `Arc<T>` 引用计数，多所有者共享 | 单线程/多线程共享数据 |
