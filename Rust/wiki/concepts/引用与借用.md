引用（Reference）是 Rust 中非拥有型指针，允许代码临时访问某个值而不取得其所有权。Rust 把创建引用的过程称为**借用（Borrow）**。

## 共享引用

共享引用用 `&T` 表示，只能读取数据，不能修改。同一数据可以同时存在多个共享引用。

```rust
let s = String::from("Hello");
let r1 = &s;
let r2 = &s;
println!("{}", r1); // 正确
println!("{}", s);  // 正确，所有权未转移
```

共享引用是 `Copy` 类型，因为多个只读引用不会导致数据竞争。

## 可变引用

可变引用用 `&mut T` 表示，允许修改指向的数据，也称为**独占引用**。被引用的变量本身必须声明为 `mut`。

```rust
let mut s = String::from("Hello");
let r = &mut s;
r.push_str(", World");
```

`mut s` 与 `&mut s` 含义不同：前者表示变量本身可变，后者表示创建一个可变引用。

## 借用规则

1. **同一时刻可有多个共享引用**。
2. **同一时刻只能有一个可变引用，且不能与共享引用共存**。

```rust
let mut s = String::from("Hello");
let r1 = &s;
let r2 = &mut s; // 错误：共享与可变不能同时存在
```

## 所有者也要遵守借用规则

当引用存在时，所有者本身也不能绕过借用规则访问数据。

```rust
let mut s = String::from("Hello");
let r = &mut s;
println!("{}", s); // 错误：r 独占期间 s 不可访问
```

## 非词法生命周期

Rust 根据引用的**最后一次使用位置**判断借用是否结束，而非变量声明的作用域边界。

```rust
let mut s = String::from("Hello");
let r1 = &s;
let r2 = &s;
println!("{} {}", r1, r2); // r1、r2 最后一次使用
let r3 = &mut s;            // 合法，共享借用已结束
r3.push_str(", World");
```

## 悬垂引用

Rust 编译期阻止引用指向已失效的数据：

```rust
fn dangle() -> &String {      // 错误
    let s = String::from("Hello");
    &s                        // s 在函数结束时释放
}
```

正确做法是返回值本身，转移所有权：

```rust
fn no_dangle() -> String {
    let s = String::from("Hello");
    s // 所有权移交给调用者
}
```

## 借用临时值

引用可以指向临时值，但仅在 `let` 初始化语句中生命周期会被延长：

```rust
let r = &String::from("hello"); // 合法，生命周期延长
println!("{}", r);
```

先声明再赋值则不适用：

```rust
let r;
r = &String::from("hello"); // 错误：临时值在此语句后释放
println!("{}", r);
```

## 可变的引用变量

引用变量本身也可声明为 `mut`，从而改变指向的目标：

```rust
let x = 10;
let y = 20;
let mut r = &x;
r = &y; // r 改为指向 y
```

这与 C++ 引用不同，Rust 的引用变量可以重新绑定。

## 在函数中使用引用

通过引用传参避免所有权转移，使函数既能读取也能修改外部数据而不获取所有权。

```rust
fn show(table: &Table) { /* 只读 */ }
fn sort(table: &mut Table) { /* 修改 */ }
```
