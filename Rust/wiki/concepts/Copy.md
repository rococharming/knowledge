`Copy` 是 Rust 中的一个标记 trait，表示类型在赋值或传参时按位复制，而非移动。实现了 `Copy` 的类型，原变量在赋值后仍然可用。

## Copy 是移动的例外

对于实现了 `Copy` 的类型，赋值不会产生所有权转移，而是复制一份独立的值：

```rust
let num1: i32 = 36;
let num2 = num1;
println!("{}", num1); // 正确，i32 是 Copy 类型
```

`i32` 等简单数值不拥有堆内存，也不需要特殊释放逻辑，因此复制后两者彼此独立。

## 常见 Copy 类型

标准库中常见的 `Copy` 类型：

- 各种整数类型（`i8`、`u32`、`isize` 等）
- 浮点型（`f32`、`f64`）
- 字符型（`char`）
- 布尔类型（`bool`）
- 不可变引用类型（`&T`）
- 元素全是 `Copy` 类型的数组（`[T; N]`）
- 字段全部是 `Copy` 类型的元组

```rust
let a = 1;
let b = a; // a 仍然可用

let x = true;
let y = x; // x 仍然可用

let p = (1, 2);
let q = p; // p 仍然可用
```

> 不可变引用 `&T` 是 `Copy` 类型，但可变引用 `&mut T` **不是** `Copy` 类型。因为可变引用要求独占访问，复制多个可变引用会破坏借用规则。

## 不能成为 Copy 的类型

以下类型不能实现 `Copy`：

- `String`：拥有堆内存，需要在 `drop` 时释放
- `Box<T>`：拥有堆上的 `T` 值，需要在 `drop` 时释放
- `File`：代表操作系统文件句柄，复制行为不简单
- `MutexGuard`：代表对互斥锁的持有，离开作用域时需要释放锁

**核心原则：任何实现了 `Drop` 的类型都不能实现 `Copy`。**

因为 `Copy` 意味着可以悄悄复制出多个值，而 `Drop` 意味着这些值离开作用域时需要执行释放逻辑。两者共存容易导致重复释放或资源状态混乱。

结构体和枚举默认也不是 `Copy` 类型。如果所有字段都是 `Copy` 类型，可以通过 `#[derive(Copy, Clone)]` 让它们成为 `Copy` 类型。
