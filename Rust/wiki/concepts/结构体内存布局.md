结构体字段在内存中的排列方式由 `repr` 属性决定。理解布局规则对 FFI 交互和底层编程至关重要。

## 默认布局：repr(Rust)

Rust 默认结构体布局为 `repr(Rust)`，在此模式下编译器**不承诺**字段的具体顺序、间距和填充方式。编译器会根据字段类型、对齐要求和优化需要决定实际布局。

因此，普通 Rust 代码不应依赖字段在内存中的排列顺序：

```rust
struct Foo {
    a: u8,
    b: u32,
}
// 不要假设 a 一定在 b 之前存储
// 编译器可能为满足 u32 的对齐要求在 a 后插入填充字节
```

## 稳定布局：repr(C)

需要确定性布局时使用 `#[repr(C)]`，让 Rust 按 C ABI 规则排列字段：

```rust
#[repr(C)]
struct Foo {
    a: u8,  // 先存储 a
    b: u32, // 再存储 b（a 后可能有 3 字节对齐填充）
}
```

`#[repr(C)]` 的主要用途是 **FFI**（Foreign Function Interface）——Rust 与 C 或其他语言进行底层接口交互时，双方必须对结构体布局达成一致。

普通业务代码一般不需要关心 `#[repr(C)]`。
