# 常量与静态变量

Rust 提供两种编译期可确定的值：`const` 常量与 `static` 静态变量。两者都必须显式标注类型，但内存语义和可变性规则截然不同。

## const 常量

使用 `const` 定义编译期常量，值必须是**编译时可计算的常量表达式**。

```rust
const SECONDS_IN_HOUR: usize = 3_600;
const SECONDS_IN_DAY: usize = 24 * SECONDS_IN_HOUR;
```

### 关键特性

| 特性 | 说明 |
|---|---|
| 类型必须显式标注 | 如 `const X: u32 = 1;` |
| 命名规范 | 全大写，下划线分隔 |
| 作用域 | 模块级、函数内、任意块级均可 |
| 内存语义 | **不保证固定内存地址**，编译器通常内联到使用处 |
| 不可遮蔽 | 同一作用域内不能用相同名字重复定义 |

`const` 更像是一个**编译期符号**，而不是运行时分配的可变变量。即使对同一个 `const` 取引用，不同使用点也不保证指向同一地址。

同一作用域中不能重复定义同名 `const`：

```rust
const NUM: i32 = 1;
const NUM: i32 = 2; // 错误：重复定义同名 const
```

## static 变量

使用 `static` 定义具有**固定内存地址**的静态存储，在程序整个运行期间存在。

```rust
static NUM: i32 = 100;
```

### 关键特性

| 特性 | 说明 |
|---|---|
| 内存语义 | **单一实例，固定地址**，所有引用指向同一位置 |
| 不会被内联 | 与 const 不同，static 保证有独立存储 |
| 默认不可变 | 和 let 一样，默认 immutable |

## static mut

`static` 可以声明为可变：

```rust
static mut NUM: i32 = 100;
```

由于 `static mut` 允许多个执行路径同时访问和修改，存在数据竞争风险，因此**所有读取和写入必须放在 `unsafe` 块中**：

```rust
unsafe {
    NUM += 1;
    println!("{}", NUM);
}
```

### Edition 2024 的变化

在 **Edition 2024** 下，编译器更严格地禁止从 `static mut` 派生出共享引用（即使只是为了打印），例如 `println!("{}", NUM)` 会在内部创建对 `NUM` 的共享引用，触发报错：

推荐做法：

- 使用 `std::ptr::addr_of!` / `addr_of_mut!` 获取原始指针再读写
- **更推荐**：使用线程安全的全局状态方案替代 `static mut`，如 `Atomic*`、`Mutex`、`RwLock`

## const vs static 对比

| | const | static |
|---|---|---|
| 内存地址 | 不保证固定 | 固定且唯一 |
| 内联 | 通常内联到使用处 | 不内联 |
| 可变性 | 不可变 | 默认不可变，可用 `mut` |
| 运行时开销 | 无（编译期确定） | 有（全局存储） |
| 多线程安全 | 安全（只读） | `static mut` 需 unsafe |
| 使用场景 | 数学常量、配置值 | 全局状态、FFI 接口 |

## 与变量系统的对比

- [[变量与可变性]]：let 绑定是运行时变量，const/static 是编译期/全局存储
- `const` 不能被遮蔽，而 `let` 变量可以遮蔽
