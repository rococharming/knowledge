# 注释

Rust 的注释系统不仅包含常规代码注释，还支持可被 **rustdoc** 解析生成 API 文档的**文档注释**，以及可作为**文档测试（doctest）**运行的代码示例。

## 注释类型总览

| 类型 | 语法 | 用途 |
|---|---|---|
| 行注释 | `//` | 简短说明，从 // 到行尾 |
| 块注释 | `/* ... */` | 多行说明，可嵌套 |
| 行文档注释 | `///` | 为下方 item 添加文档 |
| 块文档注释 | `/** ... */` | 等价于 `///`，较少使用 |
| 模块文档注释 | `//!` | 为当前模块/crate 添加文档 |
| 块模块文档注释 | `/*! ... */` | 等价于 `//!`，较少使用 |

## 行注释与块注释

### 行注释

```rust
// 这是一个行注释
let x = 5; // 行尾注释
```

### 块注释

```rust
/*
 * 这是一个块注释
 * 可以跨越多行
 */
let y = 10; /* 也可以写在同一行 */
```

**Rust 的块注释支持嵌套**，这在调试时非常有用：

```rust
/* 外层注释
   /* 内层注释 */
   仍在外层注释中
*/
```

## 文档注释

文档注释使用 Markdown 语法，由 **rustdoc** 解析生成 HTML 文档。执行 `cargo doc` 时会调用 rustdoc。crates.io 上各库的在线 API 文档（函数说明、结构体说明、示例代码等）大多就是由文档注释生成：

### 行文档注释（///）

为紧随其后的**项（item）**添加文档：

```rust
/// `add_one` 将指定值加 1
///
/// # Examples
///
/// ```
/// let arg = 5;
/// let answer = add_one(arg);
/// assert_eq!(6, answer);
/// ```
pub fn add_one(x: i32) -> i32 {
    x + 1
}
```

特点：

- 使用 Markdown 语法（`#` 标题、代码块等）
- 代码块可作为**文档测试（doctest）**运行
- 被注释的项不必是 `pub`，但非 `pub` 项在公开 API 文档中默认不显示

### 模块/crate 文档注释（//!）

注释的对象不是紧随其后的 item，而是**所在的模块或 crate 本身**：

```rust
//! This crate provides mathematical utility functions.

mod math {
    // ...
}
```

通常写在文件顶部或 `mod` 花括号内部。

## 文档惯例标题

文档中常见的 Markdown 标题（社区惯例，非语法要求）：

| 标题 | 用途 |
|---|---|
| `# Examples` | 示例用法（最常用） |
| `# Panics` | 可能 panic 的情况 |
| `# Errors` | 返回 `Result` 时的可能错误 |
| `# Safety` | 涉及 `unsafe` 代码的前置条件 |

## 生成与查看文档

```bash
# 生成文档
cargo doc

# 生成并自动在浏览器打开
cargo doc --open
```

生成的 HTML 位于 `target/doc/` 目录下。文档首页列出 crate 中定义的项：

## 与工具链的关系

- `cargo doc` 底层调用 [[rustdoc]] 生成文档
- 文档注释的质量直接影响 crates.io 上库的 API 文档体验
- 文档测试是 Rust 测试生态的重要组成部分
