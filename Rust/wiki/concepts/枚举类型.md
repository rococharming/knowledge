# 枚举类型

枚举（enum）是 Rust 中一种自定义数据类型，用于表示**一组有限的可能状态**。与 C 语言枚举不同，Rust 枚举的变体不仅可以表示状态，还可以携带不同类型和数量的数据。

## 两种基本形态

Rust 枚举可分为两类理解：

- **C 风格枚举**：变体不携带额外数据，只表示固定状态
- **带数据枚举**：不同变体可携带不同类型、不同数量的数据

## C 风格枚举

### 定义与使用

```rust
#[derive(Debug)]
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    let c = Color::Red;
    println!("{:?}", c);  // Red
}
```

变体通过 `枚举名::变体名` 形式访问。标准库中的 `std::cmp::Ordering` 也是典型 C 风格枚举：

```rust
enum Ordering {
    Less = -1,
    Equal = 0,
    Greater = 1,
}
```

未显式指定判别值时，默认从 `0` 开始递增。可通过 `as` 转换为整数，但**不允许直接把任意整数转换为枚举值**——因为整数可能不对应合法变体。需要时通过 `match` 手动检查并返回 `Option`。

### 变体导入

```rust
use std::cmp::Ordering::{self, *};

fn compare(n: i32, m: i32) -> Ordering {
    if n < m { Less } else if n > m { Greater } else { Equal }
}
```

直接导入变体可读性较差，通常建议保留 `Ordering::Less` 等完整形式。

### 所有权规则

结构体和枚举默认不是 [[Copy]] 类型，直接赋值会发生所有权移动。若所有变体均不含非 `Copy` 数据，可通过 `#[derive(Copy, Clone)]` 使其支持按位复制。

## 带数据枚举

### 三种变体形式

| 形式 | 语法 | 说明 |
|---|---|---|
| 单元型 | `Quit` | 不携带数据，与 C 风格枚举相同 |
| 元组型 | `Write(String)` | 数据按位置排列，适合含义明确的场景 |
| 结构体型 | `Move { x: i32, y: i32 }` | 数据有字段名，适合需明确表达含义的场景 |

```rust
#[derive(Debug)]
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    Color(i32, i32, i32),
}
```

### 变体嵌套枚举

枚举变体可携带另一个枚举类型：

```rust
#[derive(Debug)]
enum IpAddrKind {
    V4,
    V6,
}

#[derive(Debug)]
enum NetworkEvent {
    Connect(IpAddrKind, String),
    Disconnect(IpAddrKind),
    Timeout,
}
```

## 枚举方法

与 [[结构体类型]] 一样，枚举可通过 `impl` 块定义方法：

```rust
#[derive(Debug, Copy, Clone)]
enum Color {
    Red, Green, Blue,
}

impl Color {
    fn as_str(&self) -> &'static str {
        match self {
            Color::Red => "red",
            Color::Green => "green",
            Color::Blue => "blue",
        }
    }
}
```

方法中的 `self` 参数同样支持 `self`、`&self`、`&mut self` 三种形式，语义与结构体方法完全一致。

## Option 与 Result

`Option<T>` 和 `Result<T, E>` 是 Rust 标准库中最常用的两个枚举，因过于常用，其变体可直接写 `Some(...)`、`None`、`Ok(...)`、`Err(...)`。

### Option<T>

表示**可能有值，也可能无值**：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

Rust 没有 `null`，可能不存在的值用 `Option<T>` 表示。直接写 `None` 时通常需要显式标注类型：

```rust
let x = Some(1);
let y: Option<i32> = None;
```

### Result<T, E>

表示**操作可能成功，也可能失败**：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

常作为函数返回类型用于错误处理：

```rust
fn divide(n: i32, m: i32) -> Result<i32, String> {
    if m == 0 {
        Err(String::from("cannot divide by zero"))
    } else {
        Ok(n / m)
    }
}
```

## 用枚举表示富数据结构

枚举非常适合表达**有限种类，但每种种类携带不同数据**的结构。例如 JSON 值：

```rust
use std::collections::HashMap;

enum Json {
    Null,
    Boolean(bool),
    Number(f64),
    String(String),
    Array(Vec<Json>),
    Object(HashMap<String, Json>),
}
```

每个变体对应一种 JSON 值，且支持嵌套（数组和对象的元素仍是 `Json`）。递归数据结构在 Rust 中可能涉及 `Box` 等智能指针，以避免编译期无法确定内存大小的问题。

## 内存表示

### C 风格枚举

通常用一个整数标签表示当前变体。对于仅有三个变体的 `Color`，编译器使用 1 字节存储：

```rust
println!("{}", std::mem::size_of::<Color>());  // 1
```

可用 `#[repr(u32)]` 等显式指定底层整数表示，普通代码通常不需要关心，仅在 FFI、二进制格式等底层场景使用。

### 带数据枚举

由两部分组成：一个标签（记录当前变体）和一块数据区域（存储当前变体携带的数据）。Rust 默认不承诺具体内存布局，编译器会根据类型特点优化，普通代码不应依赖具体布局。

### 空值优化

Rust 对某些枚举做内存优化。典型例子是 `Option<&T>`：引用本身不能是空指针，因此可用空指针表示 `None`，非空指针表示 `Some(...)`。这样 `Option<&T>` 通常与 `&T` 大小相同。
