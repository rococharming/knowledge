# 语义化版本

## 定义

语义化版本（Semantic Versioning）是 Rust/Cargo 依赖管理中的版本号解析规则。通过版本前缀符号指定允许的版本范围，Cargo 自动解析并锁定兼容版本。

## 版本前缀规则

### Caret（`^`）— 默认规则

`^X.Y.Z` 表示兼容版本范围：**>= X.Y.Z, < (X+1).0.0**

允许不破坏兼容性的更新。

```bash
cargo add serde@^1.0    # 等价于 cargo add serde
```

**0.x 版本例外**：

- `^0.3.5` ⇒ >= 0.3.5, < 0.4.0（次版本视为主版本）
- `^0.0.5` ⇒ >= 0.0.5, < 0.0.6（补丁版本视为主版本）

### Tilde（`~`）

`~X.Y.Z` 表示：**>= X.Y.Z, < X.(Y+1).Z**

只允许补丁版本更新：

```bash
cargo add serde@~1.0    # >= 1.0.0, < 1.1.0
```

### 精确版本

不加前缀表示精确版本：

```bash
cargo add ferris-says@0.3.2    # 精确 0.3.2
```

## 在 Cargo.toml 中的体现

```toml
[dependencies]
serde = "1"                    # 默认 ^1.0.0
serde = "^1.0"                 # 显式 caret
tokio = { version = "~1.0" }   # tilde
anyhow = "=1.0.86"             # 精确版本（= 前缀）
```

## Cargo.lock 的作用

虽然 `Cargo.toml` 使用语义化版本指定范围，但 `Cargo.lock` 会锁定实际解析的确切版本。这样确保：只要项目在本机能编译成功，将 `Cargo.lock` 发给别人，别人也一定能编译成功。
