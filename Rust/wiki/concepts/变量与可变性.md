# 变量与可变性

Rust 的变量系统以**默认不可变**为核心设计，通过 `let` 和 `mut` 提供精确的可变性控制，并通过**变量遮蔽（shadowing）**实现作用域内的名字复用。

## let 绑定

使用 `let` 定义变量，编译器支持类型推导，也可显式注解：

```rust
let a = 10;          // i32（默认）
let a: u32 = 1;      // 显式注解
```

未使用的变量会产生警告：

![[Image 20.png|500]]

加前缀下划线 `_` 可抑制：

```rust
let _a: u32 = 10;
```

### 命名规范

- 变量与函数：蛇形命名法 `snake_case`
- 结构体、枚举、Trait：帕斯卡命名法 `PascalCase`

## 类型转换

使用 `as` 进行显式强制转换：

```rust
let a = 3.14 as i32; // a = 3，浮点截断
println!("a = {}", a);
```

![[Image 21.png]]

**注意**：Rust 无隐式类型转换，函数传参时 `u32` 不能自动当作 `i32`。

## 不可变性与 mut

变量默认不可变（immutable），重新赋值会编译错误：

```rust
let a = 10;
a = 20; // error: cannot assign twice to immutable variable
```

编译报错：

![[Image 22.png]]

使用 `mut` 声明可变性：

```rust
let mut a = 10;
a = 20; // OK
```

不可变性是 Rust 内存安全的核心支柱之一，它消除了大量数据竞争和意外修改的 bug。

## 变量遮蔽（Shadowing）

变量遮蔽不是重新赋值，而是**在相同作用域内声明一个同名的新变量**，旧的变量被隐藏但仍存在。

```rust
let x = 5;
let x = x + 1;   // 新 x 遮蔽旧 x
{
    let x = x * 2; // 内部作用域再次遮蔽
    println!("{}", x); // 12
}
println!("{}", x); // 6，外部 x 恢复可见
```

运行结果：

![[Image 23.png]]

### 遮蔽的典型用途

1. **复用变量名**：避免 `spaces_str` / `spaces_count` 这类冗余命名
2. **改变类型**：`let spaces = " "; let spaces = spaces.len();`
3. **增加可变性**：`let a = 1; let mut a = a;`

**本质**：内存的再次分配，创建了新变量，只是名字相同。

## 作用域

Rust 使用块级作用域（`{}`），变量在其声明所在的块内有效。这与变量遮蔽结合使用时，可以在内部作用域临时改变语义而不影响外部。
