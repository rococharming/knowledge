Rust 所有权（Ownership）是 Rust 内存管理的核心机制，通过编译期静态检查在不依赖垃圾回收（GC）的前提下保证内存安全。

## 内存管理的三条路线

| 路线 | 代表语言 | 机制 | 特点 |
|---|---|---|---|
| 垃圾回收（GC） | Java、Python | 运行时追踪并回收不再使用的对象 | 开发体验轻松，但有运行时开销与暂停 |
| 手动管理 | C/C++ | `malloc/free`、`new/delete` | 控制能力强，但易产生内存泄漏、悬垂指针等问题 |
| 编译期所有权系统 | Rust | **所有权 + 借用 + 生命周期** | 编译阶段静态检查，不依赖 GC，兼顾性能与安全 |

Rust 将内存正确性交给编译器规则系统，在安全 Rust 代码（非 `unsafe`）的前提下，大多数内存安全问题在编译期即可被发现。

## 所有权三大规则

1. **每一个值都有一个所有者**
2. **同一时刻，一个值只有一个所有者**
3. **当所有者离开作用域，值被丢弃**

这三条规则配合编译器检查，使 Rust 能在不依赖 GC 的情况下避免大量内存错误。

> 默认所有权模型下，一个值只有一个所有者。Rust 也允许共享所有权，但需要通过 `Rc`、`Arc` 等特殊类型显式表达。

## 作用域与 Drop

变量作用域（Scope）由 `{}` 块界定。变量进入作用域后可被使用，离开作用域后绑定失效，对应的值按所有权规则被处理。

Rust 遵循 **RAII**（资源获取即初始化）原则：当值离开作用域，会自动调用其析构逻辑（`drop`）。

- 对栈上的纯数据：离开作用域后栈空间自然失效
- 对拥有堆内存或系统资源的值：在 `drop` 时释放堆内存、关闭文件句柄、释放锁等

```rust
{
    let s = String::from("hello");
} // s 离开作用域，自动 drop，释放 String 持有的堆内存
```

## String 内存模型

`String` 是理解所有权的典型例子。它在栈上保存三个字段：

- `ptr`：指向堆缓冲区的指针
- `len`：当前长度
- `cap`：当前容量

堆上存放的是 **UTF-8 字节序列**本体。

当 `s` 离开作用域，栈上的 `String` 值失效，同时触发 `drop`，自动释放 `ptr` 指向的堆内存。

## 所有权体系中的核心机制

Rust 的所有权规则严格，但标准库提供了受控机制表达更复杂的场景：

- **移动（Move）**：值的所有权从一个绑定转移到另一个绑定，原绑定失效
- **复制（Copy）**：对实现了 `Copy` 的类型，赋值时复制独立值，原变量仍可用
- **借用（Borrow）**：通过 `&T` 或 `&mut T` 临时访问值，不取得所有权。详见 [[引用与借用]]
- **共享所有权（Rc/Arc）**：通过引用计数让多个所有者共同持有数据，最后一个所有者离开作用域时释放
