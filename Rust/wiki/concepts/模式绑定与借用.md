# 模式绑定与借用

模式匹配不只是判断值的形状，还会**绑定变量**。绑定过程中 Rust 严格遵守所有权规则，理解这一点对避免编译错误至关重要。

## 默认绑定：移动

如果匹配的是一个拥有所有权的值，并且绑定了其中的非 `Copy` 字段，该字段会被**移动**出来。

```rust
struct User {
    name: String,
    lang: String,
    age: u8,
}

fn demo(user: User) {
    match user {
        User { name, lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

这里的 `name` 和 `lang` 是 `String`（非 `Copy`），会被移动到分支内部的局部变量中。`user` 在 `match` 之后不可再用。

### 部分移动

如果只移动了结构体的一部分字段，原结构体会发生**部分移动**（partial move），之后不能再把它作为完整值使用。

```rust
let user = User {
    name: String::from("Alice"),
    lang: String::from("Rust"),
    age: 30,
};

match user {
    User { name, .. } => println!("{}", name),
}

// 错误：user 已部分移动
// println!("{:?}", user);
```

## ref 与 ref mut

如果只想借用字段而非移动出来，使用 `ref` 关键字。

```rust
fn demo(user: User) {
    match user {
        User { ref name, ref lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
    // user 仍然可用
}
```

`ref name` 表示不移动 `name` 字段，而是借用。此时 `name` 的类型是 `&String`。

需要可变借用时使用 `ref mut`：

```rust
#[derive(Debug)]
struct User {
    name: String,
    lang: String,
    age: u8,
}

fn demo(mut user: User) {
    match user {
        User { ref name, ref mut lang, .. } => {
            lang.clear();
            lang.push_str("Rust");
            println!("name: {}, lang: {}", name, lang);
        }
    }
    println!("{:?}", user);
}
```

要使用 `ref mut`，被匹配的值本身必须允许可变借用，因此参数写为 `mut user`。

**关键区分**：`ref` 和 `ref mut 出现在**模式**中，不是创建引用的表达式，而是改变绑定方式——从"移动绑定"变成"借用绑定"。

## 引用模式

`&` 出现在表达式中表示创建引用，出现在**模式**中表示匹配引用。

- 表达式：`&v` 产生 `&T`
- 模式：`&pat` 只匹配 `&T` 类型的值，匹配成功后"解开一层"引用，再用内部值匹配 `pat`

```rust
let x = 10;
let r = &x;

match r {
    &n => println!("{}", n),
}
```

`r` 的类型是 `&i32`，模式 `&n` 匹配这个引用，把 `i32` 复制到 `n`。因为 `i32` 是 `Copy`，这样写没问题。

结构体同样适用：

```rust
fn show(point: &Point) {
    match point {
        &Point { x, y } => {
            println!("({}, {})", x, y);
        }
    }
}
```

### 非 Copy 类型的引用匹配

如果字段不是 `Copy` 类型，不能通过共享引用把字段移动出来：

```rust
// 错误：cannot move out of `user.name` which is behind a shared reference
match user {
    &User { name, lang, .. } => { /* ... */ }
}
```

正确做法是用 `ref` 借用：

```rust
fn show(user: &User) {
    match user {
        &User { ref name, ref lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

### 人体工学自动解引用

实际开发中，如果匹配的是引用，Rust 会自动把绑定变成引用，因此常可写得更简洁：

```rust
fn show(user: &User) {
    match user {
        User { name, lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

虽然模式里没显式写 `&User` 或 `ref`，但因为被匹配的值是 `&User`，`name` 和 `lang` 会被自动绑定为引用（`&String`）。

这在匹配 `Option<&T>`、迭代器元素、切片引用时很常见：

```rust
fn demo(next: Option<&char>) {
    match next {
        Some(&c) => println!("next char = {}", c),
        None => println!("end"),
    }
}
```

`char` 是 `Copy`，所以 `Some(&c)` 可以直接复制出字符。若 `T` 不是 `Copy`（如 `String`），通常直接保持引用：

```rust
fn demo(value: Option<&String>) {
    match value {
        Some(text) => println!("{}", text),
        None => println!("none"),
    }
}
```
