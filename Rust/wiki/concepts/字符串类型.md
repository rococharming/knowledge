Rust 中处理 UTF-8 文本的核心类型包括字符串字面量、`str`、`&str` 和 `String`。它们的关系可以概括为：`str` 是原始切片类型，`&str` 是对 `str` 的引用，`String` 是在堆上拥有所有权的字符串。

## 字符串字面量

### 普通字符串

字符串字面量使用双引号包围，类型为 `&str`（更准确地说是 `&'static str`）。

```rust
let s = "hello";  // &str 类型
```

支持常见转义序列：

| 转义 | 含义 |
|------|------|
| `\\` | 反斜杠 |
| `\"` | 双引号 |
| `\n` | 换行 |
| `\t` | 制表符 |

字符串字面量可以直接写成多行，换行和缩进空格会原样保留：

```rust
let s = "Hello, world
    你好，世界
Rust";
```

如果一行末尾写 `\`，Rust 会忽略该反斜杠后的换行及下一行开头的连续空白，用于将长字符串拆成多行书写但运行时仍为一行：

```rust
let s = "It was a beautiful day in March, \
    there were four of us";
// 输出: It was a beautiful day in March, there were four of us
```

### 原始字符串

原始字符串以 `r` 为前缀，写作 `r"..."`，其中反斜杠、换行和空格都原样保留，不识别转义序列：

```rust
let path = r"C:\foo\bar";  // 无需转义反斜杠
let s = r"hello\nworld";    // 输出 hello\nworld，\n 不会被解释为换行
```

如果原始字符串中需要包含双引号，可以在边界加 `#`：

```rust
let s = r#"He said "hi""#;  // 边界为 "#..."#
```

若内容包含 `"#` 会与边界冲突，则继续增加 `#` 的数量：

```rust
let s = r##"He said "# hello""##;  // 边界为 "##..."##
```

### 字节串

字节串字面量带有 `b` 前缀，类型为 `&[u8; N]`（数组引用），面向字节而非 Unicode 文本：

```rust
let s = b"Hello";  // &[u8; 5]
assert_eq!(s, &[b'H', b'e', b'l', b'l', b'o']);
```

字节串中不能直接写非 ASCII 字符。字节串同样支持多行、转义和原始形式：

```rust
let path = br"C:\foo\bar";  // 原始字节串
```

## UTF-8 编码

Rust 中的 `String` 和 `&str` 都采用 UTF-8 编码，这是一种可变长度编码：

- ASCII 字符占用 1 字节
- 中文、日文、韩文等通常占用 3 字节
- 某些特殊符号和 emoji 可能占用 4 字节

```rust
assert_eq!("hello".len(), 5);      // 5 个 ASCII 字符 = 5 字节
assert_eq!("你好".len(), 6);       // 2 个汉字 = 6 字节
assert_eq!("你好".chars().count(), 2);  // 字符数为 2
```

> `len()` 返回的是字节数，不是字符数。统计字符数量应使用 `chars().count()`。

## str、&str 和 String

| 类型 | 说明 | 所有权 |
|------|------|--------|
| `str` | 字符串切片类型，表示一段 UTF-8 文本。长度不固定，通常不直接使用 | — |
| `&str` | 对 `str` 的引用，不拥有数据 | 借用 |
| `String` | 在堆上分配并管理字符串数据 | 拥有 |

可以把 `str` 视为 `[u8]` + UTF-8 语义。`&str` 的数据指针可以指向 `String` 的堆内存，也可以指向字符串字面量所在的只读静态内存。

```rust
let s1 = "hello";                    // &str，指向静态只读内存
let s2 = String::from("hello");      // String，堆上拥有数据
let s3 = &s2[1..];                   // &str，借用 s2 的一部分
```

### String 的内存结构

`String` 在栈上保存三个字段，实际字节存放在堆上：

- `ptr` — 指向堆缓冲区的指针
- `len` — 当前长度（字节数）
- `cap` — 当前容量

```rust
let s = String::from("hello");
```

当 `s` 离开作用域时，`String` 会自动释放持有的堆内存。

### &str 的内存结构

`&str` 本质是一个胖指针，包含两部分：

- 指向字符串数据起始位置的指针
- 字符串切片的字节长度

```rust
let noodles = String::from("noodles");
let oodles = &noodles[1..];  // "oodles"
```

![[Pasted image 20260513180152.png|300]]

`oodles` 没有复制字符串内容，也没有取得 `noodles` 的所有权，只是借用了从下标 1 开始的一段 UTF-8 文本。

### 字符串字面量的类型

字符串字面量的完整类型是 `&'static str`。`"hello"` 的内容在编译时被硬编码进二进制程序的静态只读区域，运行时变量只是一个指向这段数据的切片引用。

## String 的创建方式

### 从字面量创建

```rust
let s = String::from("hello");
let s = "hello".to_string();
let s = "hello".to_owned();
```

`String::from("hello")` 会在堆上开辟空间，复制字符串字面量的数据到堆中。`to_string()` 和 `to_owned()` 常用于将 `&str` 转换为 `String`。

### 使用 format!

`format!` 的用法与 `println!` 类似，但返回 `String` 而不是打印到终端：

```rust
let age = 20;
let s = format!("I am {} years old", age);
```

> `format!` 不会在末尾自动添加换行符。

### 拼接多段字符串

```rust
let fruit = vec!["apple", "orange", "banana"];
assert_eq!(fruit.concat(), "appleorangebanana");      // 直接拼接
assert_eq!(fruit.join(", "), "apple, orange, banana"); // 插入分隔符
```

## 字符串的修改

### String 的修改方法

`String` 拥有数据，声明为 `mut` 后可以修改：

| 方法 | 作用 |
|------|------|
| `push(ch)` | 追加一个字符 |
| `push_str(s)` | 追加一个字符串切片 |
| `clear()` | 清空字符串 |

```rust
let mut s = String::from("hello");
s.push('!');          // hello!
s.push_str(" Rust");  // hello! Rust
s.clear();            // 空字符串
```

### &str 不能修改

`&str` 是共享引用，不能通过它修改原字符串内容。可以让 `&str` 变量重新绑定到另一个切片，但这只是改变了绑定关系：

```rust
let mut s = "hello";
s = "world";  // 重新绑定，不是修改原字符串
```

### &mut str 的局限

`&mut str` 存在，但使用范围很窄。因为 `str` 必须始终保持合法的 UTF-8，而 UTF-8 字符的字节长度不固定，随意修改可能破坏编码结构或改变切片长度。标准库中常见的 `&mut str` 操作主要集中在 ASCII 范围：

```rust
let mut s = String::from("Hello, Rust! 123 abc XYZ 你好");
s.as_mut_str().make_ascii_uppercase();
// HELLO, RUST! 123 ABC XYZ 你好
```

`make_ascii_uppercase` 只修改 ASCII 字母，不会改变字符串总长度，中文等非 ASCII 内容不受影响。

## 字符串索引与遍历

### 不能按下标访问字符

Rust 字符串不能直接使用下标访问字符：

```rust
let s = String::from("hello");
let c = s[0];  // 错误：无法索引 String
```

原因是 UTF-8 编码中一个字符可能占多个字节，下标访问容易引起误解（以为是字符，实际是字节）。

### 按字符遍历

使用 `chars()` 按 Unicode 标量值遍历：

```rust
let s = "你好";
for ch in s.chars() {
    println!("{}", ch);
}
```

### 按字节遍历

使用 `bytes()` 查看底层 UTF-8 编码字节：

```rust
let s = "你好";
for byte in s.bytes() {
    println!("{}", byte);
}
```

![[Pasted image 20260514004611.png|100]]

### 字符串切片边界

可以对字符串创建切片，但范围必须落在合法的 UTF-8 字符边界上：

```rust
let s = String::from("hello");
let part = &s[0..2];  // "he"，合法

let s = String::from("你好");
let part = &s[0..3];  // "你"，合法（"你"占 3 字节）
```

若切片不落在字符边界上会 panic：

```rust
let s = String::from("你好");
let part = &s[0..1];  // panic！0..1 截取了 "你" 的部分字节
```
