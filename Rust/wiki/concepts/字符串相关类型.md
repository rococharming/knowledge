Rust 的 `String` 和 `&str` 始终保证内容是合法的 UTF-8，适合表示真正的 Unicode 文本。但在与操作系统或外部库交互时，程序可能遇到并不保证是 UTF-8 的数据，Rust 为不同场景提供了专门的字符串相关类型。

## 文件系统路径：PathBuf 和 &Path

处理文件系统路径时，优先使用 `PathBuf`（拥有所有权）和 `&Path`（借用）。路径不完全等同于普通文本，不同操作系统上路径的底层表示可能不同。

```rust
use std::path::PathBuf;

let mut path = PathBuf::from("/tmp");
path.push("hello.txt");
println!("{}", path.display());  // /tmp/hello.txt
```

`Path` 和 `PathBuf` 提供路径拼接、扩展名、文件名、父目录等路径语义相关操作。

## 二进制数据：Vec<u8> 和 &[u8]

处理二进制数据（如网络协议、压缩数据、哈希值、图片文件内容等）时，应使用 `Vec<u8>` 或 `&[u8]`，不应直接当作字符串处理。

如果确实知道某段字节是 UTF-8，可以使用 `std::str::from_utf8` 做验证性转换：

```rust
let bytes = vec![104, 101, 108, 108, 111];
let text = std::str::from_utf8(&bytes).unwrap();
println!("{}", text);  // hello
```

> `std::str::from_utf8` 返回 `Result`，失败时说明字节序列不是合法的 UTF-8。

## 操作系统字符串：OsString 和 &OsStr

当需要保存或传递操作系统原生形式的字符串时（如环境变量、命令行参数、文件名），使用 `OsString`（拥有所有权）和 `&OsStr`（借用）。它们允许存储平台特定的非 UTF-8 数据。

## C 字符串：CString 和 &CStr

与 C 语言库交互时，经常会遇到以 `\0` 结尾的字符串。Rust 中对应的类型是 `CString`（拥有所有权）和 `&CStr`（借用）。

`CString` 保证末尾有 `\0`，并且内部不能包含额外的 `\0`。这类类型主要用于 FFI（Foreign Function Interface）场景。

## 类型选择指南

| 场景 | 推荐类型 | 说明 |
|------|---------|------|
| 普通 UTF-8 文本，需要拥有/修改/长期保存 | `String` | 堆分配，可变 |
| 临时读取 UTF-8 文本 | `&str` | 借用，零拷贝 |
| 文件系统路径 | `PathBuf` / `&Path` | 跨平台路径语义 |
| 二进制数据 | `Vec<u8>` / `&[u8]` | 不假设编码 |
| 操作系统原生字符串 | `OsString` / `&OsStr` | 允许非 UTF-8 |
| C 语言接口 | `CString` / `&CStr` | 以 `\0` 结尾 |
