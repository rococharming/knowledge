# 模式类型

Rust 提供多种模式形式，用于匹配不同形状的数据。根据目标值的结构，可选择对应的模式进行精确匹配或部分提取。

## 字面量模式

用于精确匹配某个值。

```rust
fn traffic_light(color: &str) -> &'static str {
    match color {
        "red" => "stop",
        "green" => "go",
        "yellow" => "slow down",
        _ => "unknown",
    }
}
```

## 变量模式

匹配任何值，并把值绑定到变量名上。

```rust
fn show(n: i32) {
    match n {
        0 => println!("zero"),
        x => println!("non-zero: {}", x),
    }
}
```

变量模式会捕获所有未被前面分支处理的值。由于它能匹配任何值，应放在具体模式之后，否则会导致后续分支不可达。

## 通配符模式

`_` 匹配任何值，但不绑定变量，用于明确表示"忽略这个值"。

```rust
fn maybe_print(n: i32) {
    match n {
        0 => println!("zero"),
        _ => {}
    }
}
```

变量模式与 `_` 的区别在于：前者会拿到匹配值供后续使用，后者只是忽略。

## 元组模式

适合一次匹配多个位置的数据。

```rust
fn describe_point(x: i32, y: i32) -> &'static str {
    match (x, y) {
        (0, 0) => "origin",
        (_, 0) => "on the x axis",
        (0, _) => "on the y axis",
        (_, _) => "somewhere else",
    }
}
```

元组模式常用于将多个值组合起来一起判断：

```rust
fn compare(a: i32, b: i32) -> &'static str {
    match (a > 0, b > 0) {
        (true, true) => "both positive",
        (true, false) => "only a is positive",
        (false, true) => "only b is positive",
        (false, false) => "neither is positive",
    }
}
```

## 结构体模式

按字段名匹配和绑定。字段名与变量名相同时可简写。

```rust
struct Point {
    x: i32,
    y: i32,
}

fn describe(point: Point) -> String {
    match point {
        Point { x: 0, y } => format!("on y-axis, y = {}", y),
        Point { x, y } => format!("at ({}, {})", x, y),
    }
}
```

字段名与变量名不同时显式映射：

```rust
Point { x: a, y: b }
```

使用 `..` 忽略未关心的其余字段：

```rust
struct User {
    name: String,
    age: u8,
    city: String,
}

fn show_name(user: User) {
    match user {
        User { name, .. } => println!("name = {}", name),
    }
}
```

注意：若被绑定的字段不是 `Copy` 类型（如 `String`），会发生部分移动，原结构体之后不能再作为完整值使用。

## 枚举模式

枚举模式是 Rust 模式匹配最常见的使用场景。不同变体可携带不同形状的数据，模式匹配可根据变体种类分别处理并取出内部数据。

```rust
enum Command {
    Quit,
    Echo(String),
    Move { x: i32, y: i32 },
}

fn run(command: Command) {
    match command {
        Command::Quit => println!("quit"),
        Command::Echo(text) => println!("{}", text),
        Command::Move { x, y } => println!("move to {}, {}", x, y),
    }
}
```

`Option<T>` 和 `Result<T, E>` 本质也是枚举，常与模式匹配一起使用：

```rust
fn print_value(value: Option<i32>) {
    match value {
        Some(n) => println!("{}", n),
        None => println!("no value"),
    }
}

fn parse_number(text: &str) -> i32 {
    match text.parse::<i32>() {
        Ok(n) => n,
        Err(_) => 0,
    }
}
```

## 数组模式与切片模式

数组长度固定，按固定位置匹配：

```rust
fn hsl_to_rgb(hsl: [u8; 3]) -> [u8; 3] {
    match hsl {
        [_, _, 0] => [0, 0, 0],
        [_, _, 255] => [255, 255, 255],
        _ => /* ... */ [0, 0, 0],
    }
}
```

切片长度可变，可配合 `..` 匹配剩余元素：

```rust
fn greet(names: &[&str]) {
    match names {
        [] => println!("Hello, nobody."),
        [one] => println!("Hello, {}.", one),
        [first, second] => println!("Hello, {} and {}.", first, second),
        [first, .., last] => println!("Hello, {} ... {}.", first, last),
    }
}
```

注意：`names` 类型是 `&[&str]`，模式中绑定的 `one`、`first`、`last` 类型是 `&&str`，因为匹配的是切片引用。

切片模式适合处理"开头、结尾、长度不同"的数据结构，如命令参数、路径片段、Token 列表等。
