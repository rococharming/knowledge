## 16、迭代器

## 一、迭代器概述

迭代器是一个值，它会按需产生一系列值，通常用于执行循环操作。

Rust标准库不仅提供了用于遍历向量、字符串、哈希表和其他集合的迭代器，还提供了“从输入流中逐行产生文本”，“从网络服务器不断产生入站连接”，“从通道中不断产生其他线程发送的值”等迭代器。

当然，也可以为了自己的目的实现迭代器。

Rust的 `for` 循环为使用迭代器提供了 **自然语法** ，但迭代器本身还提供了一整套非常丰富的方法，比如映射（ `map`)、过滤（ `filter` ）、展平（ `flatten` ）、链接（ `chain` ）、收集（ `collect` ）、折叠（ `fold` ）等。

Rust的各种迭代器既高效又灵活。例如如下函数，用于返回前n个正整数之和（也称为第n个三角形数）：

展开

自动换行

fn triangle(n: i32) -> i32 {

let mut sum = 0;

for i in 1..=n {

sum += i;

}

sum

}

表达式 `1..=n` 是一个 `RangeInclusive<i32>` 类型的值，它本身就是一个迭代器，会依次生成从起始值到结束值之间的整数。因此，可以将它用作for循环的操作数来对1到n的值求和。

然而，迭代器本身还实现了 `fold` 方法，可以实现完全一样的效果：

展开

自动换行

fn triangle(n: i32) -> i32 {

(1..=n).fold(0, |acc, x| acc + x)

}

开始运行时以0作为总和， `fold` 会获取 `1..=n` 生成的每个值，并以当前总和 `acc` 和当前值 `x` 作为参数调用闭包 `|acc, x| acc + x` 。闭包的返回值作为新的总和，最后一次返回的值就是 `fold` 自身要返回的值。

从这个例子可以看出：Rust的迭代器并不只是给 `for` 循环服务的东西，它本身就是一套完整、强大而且惰性的抽象。

## 二、Iterator特征和IntoIterator特征

## 1、Iterator特征

所有的迭代器类型都实现了 `std::iter::Iterator` 。标准库对其的定义简化如下：

展开

自动换行

trait Iterator {

type Item;

fn next(&mut self) -> Option<Self::Item>;

//... 此处省略很多默认方法

}

说明：

`Item` 是迭代器的关联类型。任何类型实现 `Iterator` 特征时，都必须指定 `Item` 的类型，该类型是迭代器产生的元素类型，也就是 `next()` 方法在 `Some(...)` 中返回的值的类型。

`next()` 方法是迭代器的核心方法。迭代器就是通过不断地调用它来产生每个元素的，其参数为 `&mut self` ，因为每次迭代是，迭代器的内部状态通常会发生变化（例如当前位置向前推进）。

`next()` 的返回类型是 `Option<Self::Item>` ：

`Some(v)` ： `v` 是迭代器的下一个值

`None` ：作为迭代器结束的标志（没有下一个值了）

`Iterator` 特征不只有 `next()` 一个方法；标准库中绝大多数常用的迭代器方法（如 `map` 、 `filter` 、 `fold` 等）都是在 `next()` 的基础上提供的 **默认方法** 。

因此，为某个类型实现 `Iterator` ，通常只需要：

指定关联类型 `Item`

实现 `next()` 方法

## 2、IntoIterator特征

迭代器是实现了 `Iterator` 特征的值，而可迭代者则是实现了 `IntoIterator` 特征的值。可迭代者本身不是迭代器类型，但可以转换为迭代器。通过调用 `IntoIterator` 的 `into_iter()` 方法，可以把一个值转换为对应的迭代器。诸如向量、数组、切片、哈希表等类型，均属于可迭代者。

`IntoIterator` 特征在标准库中的简化定义如下：

展开

自动换行

trait IntoIterator {

// 迭代器产生的值的类型

type Item;

// IntoIter 必须是实现了 Iterator 的类型，并且该迭代器产生的值类型是 Self::Item

type IntoIter: Iterator<Item = Self::Item>;

fn into\_iter(self) -> Self::IntoIter;

}

说明：

`IntoIter` 是“转换后得到的迭代器类型”

`Item` 是这个迭代器生成的值的类型

任何实现了 `IntoIterator` 的类型，都称为可迭代者

当调用 `into_iter()` 时，会消耗方法接收者 `self` ，然后返回一个迭代器。

示例：

展开

自动换行

let arr = \[1, 2, 3\]; // 数组

let iter1 = arr.into\_iter();

println!("{:?}", arr);

let vec = vec!\[4, 5, 6\]; // 向量

let iter2 = vec.into\_iter();

// println!("{:?}", vec); 错误

注意：

`[i32; 3]` 这个数组类型实现了 `Copy` （因为 `i32` 是 `Copy` ，且数组在这种情况下也实现 `Copy` ），所以 `arr.into_iter()` 时实际是复制了一个数组值再去生成迭代器，原来的 `arr` 仍然可用。

`vec.into_iter()` 会直接移动整个 `Vec` ，因此 `vec` 被消耗了；

事实上，Rust的 `for` 循环本质就是对 `into_iter()` 和 `next()` 的语法糖。例如：

展开

自动换行

let v = vec!\[1, 2, 3\];

for element in v {

println!("{}", element);

}

对于诸如向量这样的集合类型，上述代码本质上等价于：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iter = v.into\_iter();

while let Some(element) = iter.next() {

println!("{}", element);

}

也就是说， `for` 循环会先对其操作数调用 `IntoIterator::into_iter()` ，把它转换为迭代器，然后不断调用 `Iterator::next()` ：

每次返回 `Some(element)` 时，执行循环体

当返回 `None` 时，循环结束

很显然，这里的 `v` 被消耗了，得到的 `iter` 会取得 `Vec` 的所有权。这里的 `iter` 的类型是 `std::vec::IntoIter<i32>` ，而 `element` 的类型为 `i32` 。

实际上，还有 `for element in &v` 和 `for element in &mut v` 两种形式：

`for element in &v`

展开

自动换行

let v = vec!\[1, 2, 3\];

for element in &v { // element 的类型为 &i32

println!("{}", element);

}

等价于：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iter = (&v).into\_iter(); // iter 类型是 Iter<'\_, i32>

while let Some(element) = iter.next() { // element 类型为 &i32

println!("{}", element);

}

这里调用的是为 `&Vec<T>` 实现的 `IntoIterator` 。因此：

`iter` 的类型是 `Iter<'_, i32>`

`element` 的类型是 `&i32`

原来的 `v` 仍然可用，因为这里只是借用了它

`for element in &mut v`

展开

自动换行

let mut v = vec!\[1, 2, 3\];

for element in &mut v { // element 的类型为 &mut i32

println!("{}", element);

}

等价于：

展开

自动换行

let mut v = vec!\[1, 2, 3\];

let mut iter = (&mut v).into\_iter(); // iter 类型是 IterMut<'\_, i32>

while let Some(element) = iter.next() { // element 类型为 &mut i32

println!("{}", element);

}

这里调用的是为 `&mut Vec<T>` 实现的 `IntoIterator` 。因此：

`Iter` 的类型是 `IterMut<'_, i32>`

`element` 的类型是 `&mut i32`

可以通过 `element` 修改向量中的元素

`for` 循环总是会对其操作数调用 `into_iter()` ，但 `for` 循环的操作数本身也可以直接是迭代器，例如遍历 `Range` ：

展开

自动换行

for i in 0..10 {

println!("{}", i);

}

这里的 `0..10` 本身就是一个迭代器类型 `Range<i32>` 。事实上， `for` 依然会对它调用 `into_iter()` 。之所以可行， **是因为迭代器本身也实现了** `IntoIterator` 。

因此，可以对一个已经是迭代器的值继续调用 `into_iter()` ，例如：

展开

自动换行

let v = vec!\[1, 2, 3\];

let v\_iter = v.into\_iter().into\_iter().into\_iter();

对于迭代器本身实现 `IntoItertor` 的代码，大概可以理解为：

展开

自动换行

impl<I: Iterator> IntoIterator for I {

type Item = I::Item;

type IntoIter = I;

fn into\_iter(self) -> I {

self

}

}

## 三、创建迭代器

## 1、iter()和iter\_mut()

多数集合类型（如 `Vec<T>` 、数组切片 `&[T]` 和 `&mut [T]` 等）都会提供两类常用方法：

`iter()` ：返回不可变迭代器，逐个产生元素的共享引用 `&T`

示例：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iterator = v.iter();

assert\_eq!(iterator.next(), Some(&1));

assert\_eq!(iterator.next(), Some(&2));

assert\_eq!(iterator.next(), Some(&3));

assert\_eq!(iterator.next(), None);

`iter_mut()` ：返回可变迭代器，逐个产生元素的可变引用 `&mut T`

示例：

展开

自动换行

let mut v = vec!\[1, 2, 3, 4, 5\];

let mut iterator = v.iter\_mut(); // 可变迭代器

let first = iterator.next().unwrap();

\*first += 9;

当不想使用 `for` ，而是手动想和迭代器打交道时， `iter()` 和 `iter_mut()` 是最常见的入口。迭代器就是反复调用 `next()` 前进的。

标准库遵循“通用约定”，但也允许类型根据自身语义设计 `iter` / `iter_mut` 的行为，只要合理即可。例如 `std::path::Path` 的 `iter()` 会返回一个迭代器，每次调用 `next()` 将产生一个路径组件（component）：

展开

自动换行

use std::path::Path;

use std::ffi::OsStr;

fn main() {

let path = Path::new(r"/tmp/foo/bar");

let mut iterator = path.iter();

assert\_eq!(iterator.next(), Some(OsStr::new("/")));

assert\_eq!(iterator.next(), Some(OsStr::new("tmp")));

assert\_eq!(iterator.next(), Some(OsStr::new("foo")));

assert\_eq!(iterator.next(), Some(OsStr::new("bar")));

assert\_eq!(iterator.next(), None);

}

这个迭代器的条目类型是 `&std::ffi::OsStr` ，它表示从操作系统字符串中借用出来的切片。

如果类型有 **不止一种遍历方式** ，通常会为每种遍历方式提供专门的方法，因为普通的 `iter()` 可能会产生歧义。例如， `str` 没有 `iter()` 方法，但提供了 `bytes()` 和 `chars()` 方法：

`s.bytes()` ：逐字节迭代

`s.chars()` ：按UTF-8解码后逐个产生 `Unicode` 标量值

## 2、into\_iter()方法

前面已经介绍过，如果一个类型实现了 `IntoIterator` ，那么就可以调用 `into_iter()` 方法获得一个迭代器。

多数集合通常会提供 `IntoIterator` 的几种实现，用于以下三种情况：

给定一个集合的共享引用， `into_iter()` 返回一个产生共享引用的迭代器

给定一个集合的可变引用， `into_iter()` 返回一个产生可变引用的迭代器

给定一个集合（按值）， `into_iter()` 返回一个取得集合所有权并按值产生元素的迭代器

例如，若 `v` 的类型为 `Vec<String>` ：

`(&v).into_iter()` 得到的迭代器产生的元素类型是 `&String`

`(&mut v).into_iter()` 得到的迭代器产生的元素类型是 `&mut String`

`v.into_iter()` 得到的迭代器产生的元素类型是 `String`

对于最后一种情况 `v.into_iter()` ，需要特别注意：条目所有权会从集合转移给迭代器消费者，原始集合本身会被消耗掉。

由于 `for` 循环会把其操作数交给 `IntoIterator::into_iter()` ，所以上述三种情况在 `for` 中对应如下：

展开

自动换行

for element in &v {... }

for element in &mut v {... }

for element in v {... }

然而，并非每种集合类型都提供上述三种实现。例如 `HashSet` 不会在 `&mut HashSet<T>` 上实现 `IntoIterator` ，因为如果需要可变引用逐个修改元素，就可能改变元素的哈希值或相等关系，从而破坏集合内部不变量。

对于 `HashMap` 这类类型，虽然在 `&mut HashMap<K, V>` 上实现了 `IntoIterator` ，但其可变迭代器产生的是 `(&K, &mut V)` ，也就是说：不能能修改键 `K` ，但可以修改值 `V` 。因为修改键也可能破坏哈希表内部结构0

切片实现了三种 `IntoIterator` 变体中的两种：

`&[T]` ：产生 `&T`

`&mut [T]` ：产生 `&mut T`

切片本身并不拥有元素，因此不存在“按值移动整个切片再逐个产生元素”这一意义上得按值变体。

对于共享引用和可变引用，前两个 `IntoIterator` 变体本质上就分别等价于调用 `iter()` 和 `iter_mut()` ：

`(&v).into_iter()` 与 `v.iter()` 在效果上等价

`(&mut v).into_iter()` 与 `v.iter_mut()` 在效果上等价

`IntoIterator` 是让 `for` 循环工作的关键，因此它是必要的。但当不使用 `for` 循环时，直接写 `v.iter()` 或 `v.iter_mut()` 往往要比写 `(&v).into_iter()` 或 `(&mut v).into_iter()` 更清晰。

## 四、迭代器适配器

## 1、适配器的概念

当已经有了一个迭代器（ `I: Iterator` ）后，可以调用 `Iterator` 特征提供的许多默认方法得到一个新的迭代器。这类方法称为 **适配器** （adaptor）。

迭代器方法会 **消耗旧的迭代器，并返回一个新的迭代器** ：

旧迭代器负责产出原始条目

新迭代器在此基础上做变换、过滤、展开、重排等处理，再产出新条目流

这些适配器可以像管道一样链式调用，每个适配器只做一件小事，组合起来就能优雅地表达数据流。

需要特别注意的是：适配器本身是 **惰性（lazy）** 的。单纯调用 `map` 、 `filter` 、 `take` 等方法并不会立刻处理数据，只要当后续真正开始消费迭代器（如调用 `next()` 、 `collect()` 、 `sum()` 等）时，整个流水线才会运转。

## 2、map（逐项变换）

`map` 的作用是对迭代器的每个条目调用闭包，让条目变成另一个值，最终返回一个新的迭代器。 `map` 常常搭配 `collect` 一起使用，把变换后的迭代结果收集到一个新集合中。

`map` 方法签名可简化理解为：

展开

自动换行

fn map<B, F>(self, f: F) -> Iterator<Item = B>

where

Self: Sized,

F: FnMut(Self::Item) -> B;

说明：

闭包会接收迭代器的条目 `Self::Item`

返回值 `B` 作为新迭代器的条目类型

适配器按值接收 `self` ，因此要求 `Self: Sized` 。

标准库中， `map` 实际返回的是一个专门类型 `std::iter::Map<...>` ，这里用 `Iterator<Item = B>` 便于理解。

示例1：

将向量中的每个元素都转换为它的平方。

展开

自动换行

let v1 = vec!\[1, 2, 3\];

let v2 = v1.iter()

.map(|x| x \* x ) // x的类型是&i32，这里x \* x自动会进行解引用

.collect::<Vec<i32>>();

println!("{:?}",v1);

println!("{:?}",v2);

示例2：

逐行遍历文本，并去掉每行前后的空白，返回一个新的修剪过的 `&str` 向量。

展开

自动换行

let text = " hello \\n world \\n i \\n love \\n rust \\n".to\_string();

let v: Vec<&str> = text.lines()

.map(str::trim)

.collect();

println!("{:#?}", v);

## 3、filter（按条件保留/丢弃）

`filter` 的作用是对迭代器的每个条目调用闭包进行判断：

若闭包返回 `true` ，保留该条目

若闭包返回 `false` ，丢弃该条目

这个判断闭包通常也称为 **谓词（predicate）** 。

`filter` 的签名如下：

展开

自动换行

fn filter<F>(self, f: F) -> impl Iterator<Item = Self::Item>

where

Self: Sized,

F: FnMut(&Self::Item) -> bool;

说明：

闭包参数是 `&Self::Item`

**新迭代器的产出的条目类型和旧迭代器相同**

只是某些条目被过滤掉了。

🔊

迭代器适配器的两点注意事项：

（1）仅仅在迭代器上调用适配器，只会得到一个新的迭代器， **它本身并不立刻消耗条目**

（2）迭代器适配器是一种 **零成本抽象** ，像 `map` 和 `filter` 等这类方法都是泛型方法，编译器会为具体类型生成专门代码，因此通常不会产生额外抽象开销

示例1：

保留向量中的偶数。

展开

自动换行

let v = vec!\[1, 2, 3, 4, 5\];

let mut iter = v.into\_iter()

.filter(|x| x % 2 == 0); // 这里x的类型为&i32，x % 2自动解引用

assert\_eq!(iter.next(), Some(2));

assert\_eq!(iter.next(), Some(4));

assert\_eq!(iter.next(), None);

需要注意：传给 `filter` 的闭包接收的是 `&Self::Item` 。 **如果原迭代器本身产生的就是就是引用，那么闭包参数就会变成双重引用** 。

示例2：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iter = v.iter()

.filter(|x| \*\*x > 1 ); // // x 的类型是 &&i32，因此需要两个 \*

assert\_eq!(iter.next(), Some(&2));

assert\_eq!(iter.next(), Some(&3));

通常更清晰的做法是在参数位置结构一层或两层引用：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iter = v.iter()

.filter(|&x| \*x > 1 ); // // x 的类型是 &i32

assert\_eq!(iter.next(), Some(&2));

assert\_eq!(iter.next(), Some(&3));

或者：

展开

自动换行

let v = vec!\[1, 2, 3\];

let mut iter = v.iter()

.filter(|&&x| x > 1 ); // // x 的类型是i32

assert\_eq!(iter.next(), Some(&2));

assert\_eq!(iter.next(), Some(&3));

对于一个尚未被消费的迭代器而言， `iter.filter(f).next()` 和 `iter.find(f)` 在效果上是等价的：都会找到第一个使谓词为 `true` 的元素。

## 4、filter\_map（过滤映射）

`filter_map` 可以理解为“ `map` + 丢弃部分条目”的组合。与 `map` 不同的是，它的闭包返回的不是 `B` ，而是 `Option<B>` ：

返回 `Some(b)` ：保留该条目，并把它映射为 `b`

返回 `None` ：丢弃该条目

其方法原型可简化为：

展开

自动换行

fn filter\_map<B, F>(self, f: F) -> impl Iterator<Item = B>

where

Self: Sized,

F: FnMut<Self::Item> -> Option<B>;

示例：

展开

自动换行

use std::str::FromStr;

fn main() {

let text = "1\\nfrond.25 289\\n3.1415 estuary\\n";

for number in text

.split\_whitespace()

.filter\_map(|s| f64::from\_str(s).ok())

{

println!("{:4.2}", number);

}

}

说明：

`f64::from_str()` 尝试将字符串解析为 `f64` ，返回 `Result<f64, ParseFloatError>`

再调用 `ok()` 将 `Result<f64, ParseFloatError>` 变为 `Option<f64>`

解析成功时得到 `Some(v)` ，保留该值

解析失败时得到 `None` ，该条目被过滤掉

上述代码也可以用 `map + filter + map` 实现，但会更繁琐：

展开

自动换行

use std::str::FromStr;

fn main() {

let text = "1\\nfrond.25 289\\n3.1415 estuary\\n";

for number in text.split\_whitespace() // &str

.map(|s| f64::from\_str(s)) // Result<f64, ParseFloatError>

.filter(|res| res.is\_ok()) // 只保留Ok

.map(|r| r.unwrap()) // // 解包出 f64

{

println!("{:4.2}", number);

}

}

## 5、flatten（展平）

`flatten` 适配器会把“ **迭代器的每一个条目本身又是一个可迭代者** ”的结构，串联成一个连续迭代器。

其签名可以简化为：

展开

自动换行

fn flatten(self) -> impl Iterator<Item = <Self::Item as IntoIterator>::Item>

where

Self: Sized,

Self::Item: IntoIterator;

也就是说：

旧迭代器的每个条目必须实现 `IntoIterator`

`flatten` 会把这些小序列首尾相接，产生一个连续的大序列

示例：

展开

自动换行

// v 的结构为：\[\[1, 2\], \[3, 4\], \[5\]\]

let v = vec!\[

vec!\[1, 2\],

vec!\[3, 4\],

vec!\[5\]

\];

let v = v.into\_iter().flatten().collect::<Vec<i32>>();

println!("{:?}",v);

事实上， `Option<T>` 也是一个 **可迭代者** ：

`Some(x)` 看作是一个只包含一个元素的序列

`None` 看作是空序列

因此，对于条目为 `Option<T>` 的迭代器，也可以调用 `flatten` ：

展开

自动换行

let v = vec!\[None, Some("day"), None, Some("one")\];

let v2 = v.into\_iter().flatten().collect::<Vec<&str>>();

println!("{:?}",v2);

结果中只保留了 `Some` 里的值。

同样， `Result<T, E>` 也实现了 `IntoIterator` ：

`Ok(x)` 看作单元素序列

`Err(e)` 看作空序列

因此，对 `Iterator<Item = Result<T, E>>` 调用 `flatten()` ，会自动丢弃所有 `Err` 并产生一个成功值流。

## 6、flat\_map（展平映射）

`flat_map` 可以理解为 `map + flatten` 的组合。它会先把 **每个元素映射成一个可迭代者** ，再把这些可迭代者串联成一个连续迭代器。

原型可简化理解为：

展开

自动换行

fn flat\_map<U, F>(self, f: F) -> impl Iterator<Item = U::Item>

where

Self: Sized,

F: FnMut(Self::Item) -> U,

U: IntoIterator;

这里闭包返回的不是单个值，而是一个可迭代者 `U` 。

示例：

展开

自动换行

use std::collections::HashMap;

fn main() {

let mut major\_cities = HashMap::new();

major\_cities.insert("Japan", vec!\["Tokyo", "Kyoto"\]);

major\_cities.insert("The United States", vec!\["Portland", "Nashville"\]);

major\_cities.insert("Brazil", vec!\["São Paulo", "Brasília"\]);

major\_cities.insert("Kenya", vec!\["Nairobi", "Mombasa"\]);

major\_cities.insert("China", vec!\["Shenzhen", "Shanghai"\]);

let contries = vec!\["China", "The United States", "Brazil"\];

for &city in contries.into\_iter().flat\_map(|country| &major\_cities\[country\] ) {

println!("{}", city);

}

}

可以这样理解：对每个国家，先找到对应的城市向量，然后把这些向量串接成一个总的城市序列。

再例如，标准库中的 `str::to_uppercase` 的思路可以立即为：

展开

自动换行

fn to\_uppercase(&self) -> String {

self.chars()

.flat\_map(char::to\_uppercase)

.collect()

`char::uppercase` 返回的不是单个字符，而是一个可迭代序列。例如德语小写字母 `ß` 的大写形式是 `SS` ，因此必须返回一个字符序列而不是单个字符。

## 7、take（取出）和take\_while（当...时取出）

`take` 和 `take_while` 的作用是让一个迭代器“提前结束”。

其签名可以简化为：

展开

自动换行

fn take(self, n: usize) -> impl Iterator<Item = Self::Item>

where

Self: Sized;

fn take\_while<P>(self, predicate: P) -> impl Iterator<Item = Self::Item>

where

Self: Sized,

P: FnMut(&Self::Item) -> bool;

## 9、peekable（窥探）

普通迭代器一旦调用 `next()` ，就会把“下一个条目”消耗掉（推进迭代器位置）。但很多场景需要先看一眼再决定怎么操作。例如，解析连续数字字符时，希望遇到第一个非数字字符就停止，但不能把该非数字字符消耗掉，因为后续还要处理它。此时普通的 `next()` 就不够用了，因为当 `next()` 判断出非数字字符，此时该数字字符已经被丢弃了。

在 Rust 中， `peekable` 是 `Iterator` 的一个适配器方法。对任意迭代器调用它后，会返回一个新的 **包装器类型** ：

展开

自动换行

fn peekable(self) -> std::iter::Peekable<Self>

where

Self: Sized;

注意： `Peekable<Self>` 内部持有原迭代器 `Self` ，它本身仍然实现了 `Iterator<Item= Self::Item>` ，但它额外提供了 `peek()` 方法。

`peek()` 方法可以理解为：

展开

自动换行

fn peek(&mut self) -> Option<&Self::Item>;

其返回含义：

若底层迭代器还有条目：返回 `Some(&next_item)`

若底层迭代器已耗尽：返回 `None`

`peek()` 返回的是引用，因此调用者可以看到下一项，但不会把它移动出来。如果原迭代器本身产生的就是引用，那么 `peek()` 返回的就会是“引用的引用”。

事实上，调用 `peek()` 时，如果底层迭代器还有数据，它 **会从底层迭代器中取出一个条目缓存起来，返回返回对该缓存条目的引用** 。之后再调用 `next()` 时，会优先吐出缓存中的这个条目。

因此，“ `peek()` 不消耗条目”的本质并不是完全没动底层迭代器，而是 **提前取出后把它存进了** `Peekable` **自己的缓存里** ，让外部表现得像没消耗一样。

示例：

展开

自动换行

let iter = vec!\[1,2,3\].into\_iter();

let mut iter2 = iter.peekable();

assert\_eq!(Some(&1), iter2.peek());

assert\_eq!(Some(&1), iter2.peek());

assert\_eq!(Some(1), iter2.next());

assert\_eq!(Some(&2), iter2.peek());

连续调用 `peek()` ，返回都是同一个值的引用，除非显式再调用 `next()` 消耗掉。

使用 `Peekable` 迭代器的典型应用场景就是从字符串中解析出数字：

展开

自动换行

use std::iter::Peekable;

fn parse\_number<I>(mut tokens: Peekable<I>) -> u32

where I: Iterator<Item = char>

{

let mut n = 0;

loop {

match tokens.peek() {

Some(ch) if ch.is\_ascii\_digit() => {

n = n \* 10 + ch.to\_digit(10).unwrap();

tokens.next(); // 真正消耗这个数字字符

}

\_ => return n,

}

}

}

fn main() {

let input = "123456789";

let n = parse\_number(input.chars().peekable());

println!("{}", n);

}

这里的思路是：

● 用 `peek()` 看下一字符是不是数字

● 如果是数字：更新数值，并用 `next()` 真正消费它

● 如果不是数字或已经结束：停止解析，并把这个非数字字符保留在迭代器里，供后续处理

## 10、fuse（保险丝）

`Iterator` 特征并没有规定一旦 `next()` 返回 `None` 后，再次调用 `next()` 必须怎么表现。大多数迭代器会继续返回 `None` ，但并非所有迭代器都如此。

因此，泛型代码不能盲目依赖“返回一次 `None` 后，后续返回永远 `None` ”。为了消除这种不确定性，可使用 `fuse` 适配器方法。该方法会把任意迭代器包装成一个一旦第一次返回 `None` ，后续永远只返回 `None` 的迭代器。

示例：

展开

自动换行

struct Test(bool);

impl Iterator for Test {

type Item = &'static str;

fn next(&mut self) -> Option<Self::Item> {

if self.0 {

self.0 = false;

Some("last item")

} else {

self.0 = true;

None

}

}

}

fn main() {

let mut t = Test(true);

assert\_eq!(t.next(), Some("last item"));

assert\_eq!(t.next(), None);

assert\_eq!(t.next(), Some("last item"));

assert\_eq!(t.next(), None);

}

这里 `test.next()` 的结果是交替出现的，这说明原始迭代器并不是见到 `None` 就永远结束。

可以用 `fuse` 方法修正，其原型如下：

展开

自动换行

fn fuse(self) -> Fuse<Self>

where

Self: Sized,

{

Fuse::new(self)

}

修正的示例代码如下：

展开

自动换行

struct Test(bool);

impl Iterator for Test {

type Item = &'static str;

fn next(&mut self) -> Option<Self::Item> {

if self.0 {

self.0 = false;

Some("last item")

} else {

self.0 = true;

None

}

}

}

fn main() {

let mut t = Test(true).fuse();

assert\_eq!(t.next(), Some("last item"));

assert\_eq!(t.next(), None);

assert\_eq!(t.next(), None);

assert\_eq!(t.next(), None);

}

`fuse()` 在处理来源不明的迭代器时很有用。与其假定所有迭代器都“行为良好”，不如直接加一个“保险丝”。

## 11、可逆迭代器与rev

普通迭代器只能通过 `next()` 从前往后取元素。然而，对于很多容器，例如向量、切片等，其元素天然也可以从后往前取。能做到这一点的迭代器，称为 **双端迭代器（double-ended iterator）** 。

Rust使用 `std::iter::DoubleEndedIterator` 特征描述这种能力，它在 `Iterator` 的基础上额外要求实现：

展开

自动换行

trait DoubleEndedIterator: Iterator {

fn next\_back(&mut self) -> Option<Self::Item>;

}

可以把它想象成两根手指从两端往中间走：

调用 `next()` ：头指针向后一步

调用 `next_back()` ：尾指针向前一步

当两端指针相遇或交错时，说明元素已经取完，之后两边都返回 `None`

示例：

展开

自动换行

let v = \["hello", "world", "rust"\];

let mut iter = v.into\_iter();

assert\_eq!(Some("hello"), iter.next());

assert\_eq!(Some("rust"), iter.next\_back());

assert\_eq!(Some("world"), iter.next());

assert\_eq!(None, iter.next\_back());

并不是所有的迭代器都可以实现 `DoubleEndedIterator` 。例如，从通道 `Receiver` 中不断接收其他线程发送的消息，这种迭代器就无法预先知道最后一个元素是什么，因此只能向前 `next()` ，无法 `next_back()` 。

如果迭代器是双端迭代器，那么就可以调用 `rev()` 适配器将其方向反转，原型如下：

展开

自动换行

fn rev(self) -> impl Iterator<Item = Self::Item>

where

Self: Sized + DoubleEndedIterator;

示例：

展开

自动换行

let v = \["hello", "world", "rust"\];

let mut iter = v.into\_iter().rev(); // 反转

assert\_eq!(Some("rust"), iter.next());

assert\_eq!(Some("world"), iter.next());

assert\_eq!(Some("hello"), iter.next());

**很多适配器方法（如** `map` **、** `filter` **等）在应用于双端迭代器后，返回的新迭代器通常仍然是双端迭代器** 。

## 12、inspect（检查）

Rust中常有如下链式写法：

展开

自动换行

iter.map(...).filter(...).flat\_map(...).collect();

当链路很长时，可能想知道：

`map` 之后数据长什么样

`filter` 之后还剩多少

`flat_map` 展开后是不是符合预期

这时如果把流水线拆成多个变量会打断原始结构，而 `inspect()` 正是用来做这种“中途观察”的。

`inspect` 的核心行为是：

对迭代器产生的每个条目，调用一次传入的闭包

但仍然把该条目原封不动地继续传给后续迭代器

它的闭包参数是 `&Item` ，因此适合做“观察型”操作，如打印、调试、断言等，而不会改变迭代结果。

示例：

展开

自动换行

let upper\_case = "große".chars()

.inspect(|c| println!("before: {:?}", c))

.flat\_map(|c| c.to\_uppercase())

.inspect(|c| println!("after: {:?}", c))

.collect::<String>();

println!("upper\_case: {:?}", upper\_case);

这里：

● 第一个 `inspect` 打印原始字符

● `flat_map(char::to_uppercase)` 做大写转换

● 第二个 `inspect` 打印转换后的字符流

● 最终 `collect::<String>()` 收集成字符串

## 13、chain（链接）

`chain` （链接）适配器会把一个迭代器追加到另一个迭代器之后，也就是把两个序列首尾连接成一个新序列。

其签名可简化理解为：

展开

自动换行

fn chain<U>(self, other: U) -> impl Iterator<Item = Self::Item>

where

Self: Sized,

U: IntoIterator<Item = Self::Item>;

注意：参数 `other` 不必已经是迭代器；只要它实现了 `IntoIterator` 且元素和 `Self::Item` 一致，即可作为 `chain` 的右侧输入。

示例：

展开

自动换行

let v =(0..4).chain(\[4, 5, 6\])

.collect::<Vec<\_>>();

assert\_eq!(v, \[0, 1, 2, 3, 4, 5, 6\]);

如果 `chain` 的两个底层迭代器都是双端迭代器，那么结果迭代器通常也仍然是双端的：

展开

自动换行

let v =(0..4).chain(\[4, 5, 6\])

.rev()

.collect::<Vec<\_>>();

assert\_eq!(v, \[6, 5, 4, 3, 2, 1, 0\]);

## 14、enumerate（枚举）

`enumerate` 适配器方法会把“运行中”的索引附加到迭代结果上，即把：

展开

自动换行

A, B, C,...

变成：

展开

自动换行

(0, A), (1, B), (2, C),...

也就是说，对任何迭代器调用：

展开

自动换行

iter.enumerate()

得到的是一个新的迭代器，其条目是二元组：

第一个是 `usize` 类型的索引，从0开始递增

第二个是原迭代器条目

示例：

展开

自动换行

let names = \["Tom", "Jerry", "Bob"\];

for (i, name) in names.iter().enumerate() {

## 15、zip（拉合）

`zip` 适配器会把两个迭代器并排拉合成一个新的迭代器。新迭代器每次从左右两个底层迭代器各取出一个值，组成一个二元组。

一旦左右两边中任意一方结束，整个 `zip` 结果就结束。

例如，可以把无限范围 `0..`和一个迭代器拉合，得到和 `enumerate` 更一般化的形式。

示例：

展开

自动换行

let names = \["Tom", "Jerry", "Bob"\];

for (i, name) in (0..).zip(names.iter()) {

println!("{}: {}", i, name);

}

`zip` 的参数也不一定必须已经是迭代器，只要它是可迭代者即可：

展开

自动换行

let names = \["Tom", "Jerry", "Bob"\];

for (greet, name) in repeat("Hello").zip(names) {

println!("{}, {}", greet, name);

}

## 16、by\_ref（按引用）

多数迭代器适配器（如 `map` 、 `take_while` 、 `skip_while` 等）都会取得底层迭代器的所有权。一旦这样做，原来的迭代器变量就被移动走了，不能再继续使用。

但有时想分两段消费一个迭代器：先处理前一段，再继续处理剩余部分。这时候就可以使用 `by_ref()` 方法。

`by_ref` 方法的本质是：返回对当前迭代器的可变借用 `&mut self` ，这样后续适配器拿走的是对迭代器的可变引用，而是迭代器本身。

示例：

展开

自动换行

let message = "To: jimb\\r\\n\\

From: id\\r\\n\\

\\r\\n\\

Oooooh, donuts!!\\r\\n";

let mut lines = message.lines();

println!("Headers");

println!("{}", header);

}

println!("\\nBody");

for body in lines {

println!("{}", body);

}

`lines.by_ref()` 借出 `lines` 的可变引用

`takes_while()` 消耗的是这个引用包装的迭代器

因此 `lines` 本体没有被移走，还可以继续用于遍历正文部分

## 17、cloned（克隆）和copied（复制）

很多常见迭代器会产生引用，例如：

展开

自动换行

let a = \['1', '2', '3'\];

let iter = a.iter(); // Item = &char

但有时候希望得到的是值本身，而不是引用，这时候就可以使用 `cloned()` 和 `copied()` 。

### （1）cloned

`cloned()` 的语义可以近似理解为：

展开

自动换行

iter.map(|item| item.clone())

也就是说：

输入通常是 `&T`

输出通常是 `T`

要求 `T: Clone`

示例：

展开

自动换行

let s1 = vec!\["1".to\_string(), "2".to\_string(), "3".to\_string()\];

let s2: Vec<String> = s1.iter().cloned().collect();

println!("{:?}", s1);

println!("{:?}", s2);

### （2）copied

`copied()` 的思想与 `cloned()` 类似，但要求更严格：元素类型必须实现 `Copy` 。

其语义可以近似理解为：

展开

自动换行

iter.map(|r| \*r)

也就是说，它只是从引用中复制出值本身。

示例：

展开

自动换行

let v = vec!\[1, 2, 3\];

let v2: Vec<i32> = v.iter().copied().collect();

assert\_eq!(v, v2);

这里 `i32` 是 `Copy` 类型，因此 `copied()` 只是把 `&i32` 解引用复制为 `i32` 。

需要注意：

所有实现了 `Copy` 的类型也一定实现了 `Clone`

因此 `cloned()` 更通用

但如果条目类型是简单的 `Copy` 类型，那么 `copied()` 往往更明确，也通常更高效

## 18、cycle（循环）

`cycle()` 适配器会把一段有序序列变成一个无限重复的序列。它会保存迭代器的初始状态，并在每次走到结尾时重新从头开始。因此， **调用** `cycle()` **的迭代器类型本身必须实现** `Clone` **，以便在需要时恢复初始状态** 。

示例：

展开

自动换行

let dirs = \["North", "East", "South", "West"\];

let mut spin = dirs.into\_iter().cycle();

assert\_eq!(spin.next(), Some("North"));

assert\_eq!(spin.next(), Some("East"));

assert\_eq!(spin.next(), Some("South"));

assert\_eq!(spin.next(), Some("West"));

assert\_eq!(spin.next(), Some("North"));

assert\_eq!(spin.next(), Some("East"));

assert\_eq!(spin.next(), Some("South"));

assert\_eq!(spin.next(), Some("West"));

## 五、消耗迭代器

## 1、迭代器消费器的概念

前面介绍的大多是“创建迭代器”和“适配迭代器”的方法。而在 `Iterator` 特征中，还有一类方法称为 **消费器（consumer）** 。 **它们会真正消耗迭代器，并计算出一个最终结果，例如数字、集合、布尔值等** 。

这些方法的内部本质都是在不断调用 `next()` ，直到得到所需结果或迭代器结束。

## 2、collect（收集）

`collect()` 是最常见的消费器之一，它会把迭代器产生的条目收集到 **某种目标容器** 中，只要该容器实现了 `FromIterator` 。

示例：

展开

自动换行

let v = (1..=5).map(|x| x \* x).collect::<Vec<\_>>();

assert\_eq!(v, vec!\[1, 4, 9, 16, 25\]);

`collect()` **的目标类型往往需要turbofish明确指出，否则编译器无法推断** 。

它不仅能收集成 `Vec<T>` ，还常用于：

`String`

`HashMap<K, V>`

`HashSet<T>`

...

## 3、count（计数）

`count()` 会从迭代器中不断提取条目，直到迭代器返回 `None` ，最终返回条目总数。

示例：

展开

自动换行

use std::io::prelude::\*;

fn main() {

let stdin = std::io::stdin();

println!("{}", stdin.lock().lines().count());

}

注意事项：

`std::io::stdin()` 返回的是全局共享的标准输入句柄 `Stdin` ， `lock()` 方法用户获取一个锁定的输入句柄，防止多处同时读取造成混乱

`lines()` 是 `BufRead` 特征的方法，它会返回一个迭代器，该迭代器会产生每一行字符串。需要把 `BufRead` 引入作用域；导入 `use std::io::prelude::*;`也可以，因为其中包含了 `BufRead` 。

当然，对于很多集合类型而言，它们本身就有 `len()` 方法，因此无需为了计数而先构造迭代器再调用 `count()` 。

## 4、sum（求和）和product（乘积）

`sum()` 和 `product()` 分别计算迭代器条目之和与条目之积。它们都是消费器，本质上内部不断调用 `next()` 并累积结果：

`sum()` 从加法单位元开始（例如整数的 `0` ）

`product()` 从乘法单位元开始（例如整数的 `1`)

以 `sum()` 为例，其简化定义如下：

展开

自动换行

fn sum<S>(self) -> S

where

Self: Sized,

S: Sum<Self::Item>

{

Sum::sum(self)

}

说明：

`sum()` 是泛型函数，需要知道最终返回值类型 `S`

返回值类型不一定必须和迭代器条目类型相同

`S: Sum<Self::Item>` 表示： `S` 必须知道如何把一个 `Iterator<Item = Self::Item>` 里的值求和成自己。

`Sum` 特征核心定义可以理解为：

展开

自动换行

trait Sum<A = Self>: Sized {

fn sum<I: Iterator<Item = A>>(iter: I) -> Self;

}

示例1:

展开

自动换行

fn triangle(n: u64) -> u64 {

(1..=n).sum()

}

fn factorial(n: u64) -> u64 {

(1..=n).product()

}

fn main() {

assert\_eq!(triangle(3), 6);

assert\_eq!(factorial(4), 24);

}

示例2：

展开

自动换行

fn main() {

let v = vec!\[1, 2, 3\];

let iter = v.iter(); // 迭代器元素类型为&i32

let s = iter.sum::<i32>(); // i32实现了Sum<&i32>

println!("{}", s);

}

## 5、min（最小）和max（最大）

`min()` 和 `max()` 会分别返回迭代器中的最小条目和最大条目。它们要求条目类型实现 `std::cmp::Ord` ，这样条目之间才能进行全序比较。

两个方法的返回值都是 `Option<Self::Item>` ，以便在迭代器为空时返回 `None` 。

示例：

展开

自动换行

assert\_eq!(\[-2, 0, 3, -5, 1\].iter().max(), Some(&3));

assert\_eq!(\[-2, 0, 3, -5, 1\].iter().min(), Some(&-5));

需要注意的是， `f32` 和 `f64` 类型只实现了 `PartialOrd` ，没有实现 `Ord` ，因此不能直接对浮点数迭代器调用 `min()` 和 `max()` 。原因在于浮点数中存在 `NaN` ， **它不满足普通的全序比较规则** 。

## 6、min\_by（据...最小）和max\_by（据...最大）

`min_by()` 和 `max_by()` 会根据用户提供的比较函数决定“谁更小”或“谁更大”。

以 `max_by()` 为例，其函数原型可以简化为：

展开

自动换行

fn max\_by<F>(self, compare: F) -> Option<Self::Item>

where

Self: Sized,

F: FnMut(&Self::Item, &Self::Item) -> std::cmp::Ordering;

比较函数需要返回 `std::cmp::Ordering` ， `Ordering` 是一个枚举类型：

展开

自动换行

pub enum Ordering {

Less, // 左值小于右值

Equal, // 左值等于右值

Greater, // 左值大于右值

}

示例：

展开

自动换行

use std::cmp::Ordering;

fn cmp(lhs: &f64, rhs: &f64) -> Ordering {

lhs.partial\_cmp(rhs).unwrap()

}

fn main() {

let numbers = \[1.0, 3.0, 2.0\];

assert\_eq!(numbers.iter().copied().max\_by(cmp), Some(3.0));

assert\_eq!(numbers.iter().copied().min\_by(cmp), Some(1.0));

let numbers = \[1.0, 4.0, 2.0, f64::NAN\];

// numbers.iter().copied().max\_by(cmp); // 会 panic

}

这里之所以可能 panic，是因为当参与比较的值中含有 `NaN` 时， `partial_cmp()` 会返回 `None` ， `unwrap()` 就会触发 panic。

因此，对于浮点数：

● 若能保证没有 `NaN` ，可以这样写

● 若不能保证，应该先过滤掉 `NaN`

展开

自动换行

assert\_eq!(numbers.iter().copied().filter(|x|!x.is\_nan()).max\_by(cmp), Some(4.0));

## 7、min\_by\_key（据键最小）和max\_by\_key（据键最大）

有时候不想直接比较条目自身，而是想根据某个键来比较，此时可以使用 `min_by_key()` 和 `max_by_key()` 。

以 `max_by_key()` 为例，其签名可以简化理解为：

展开

自动换行

fn max\_by\_key<B: Ord, F>(self, f: F) -> option<Self::Item>

where

Self: Sized,

F: FnMut(&Self::Item) -> B;

它们的工作方式是：

对每个条目调用闭包 `f(&item)` 得到一个键 `B`

再用键 `B` 的顺序来决定哪个条目最小或最大

最终返回的仍然是原始条目，而不是键

示例：

展开

自动换行

use std::collections::HashMap;

fn main() {

let mut populations = HashMap::new();

populations.insert("Portland", 583\_776);

populations.insert("Fossil", 449);

populations.insert("Greenhorn", 2);

populations.insert("Boring", 7\_762);

populations.insert("The Dalles", 15\_340);

assert\_eq!(populations.into\_iter().max\_by\_key(|&(\_name, population)| population),

Some(("Portland", 583\_776)));

}

## 8、对条目序列进行比较

字符串、数组、切片、向量等值本身，通常可以直接使用 `<` 、 `>` 、 `==` 进行比较，前提是其中元素类型实现了对应的特征，例如：

`&str` 的比较按字典序逐字符进行

`Vec<T>` 、 `[T]` 的比较按逐元素字典序进行

示例：

展开

自动换行

let v1 = vec!\[1, 2, 3\];

let v2 = vec!\[1, 2, 4\];

assert!(v1 < v2);

但迭代器本身不能直接使用这些运算符比较，因为迭代器表示的是按需产生值的过程，而不是一个一次性静态值。为此，Rust迭代器提供了一组专门的方法来比较“条目序列”：

相等比较： `eq` 、 `ne`

有序比较： `lt` 、 `le` 、 `gt` 、 `ge`

直接得到顺序结果： `cmp` 、 `partial_cmp`

这些方法的行为是：从两个迭代器中成对取出条目，按字典序逐个比较，直到分出胜负或结束。

示例：

展开

自动换行

let packed\_rust = "hello world rust";

let spaced\_rust = "hello world rust";

let packed\_c = "hello world c";

assert!(packed\_rust!= spaced\_rust);

assert!(packed\_rust.split\_whitespace().eq(spaced\_rust.split\_whitespace()));

assert!(packed\_rust.split\_whitespace().gt(packed\_c.split\_whitespace()));

## 9、any（任意）和all（全部）

`any()` 和 `all()` 都接受一个闭包，对闭包里的元素逐个判断，并返回 `bool` ：

`any(pred)` ：主要存在至少一个元素使 `pred(element)` 为 `true` ，就返回 `true`

`all(pred)` ：只有当所有元素都使 `pred(element)` 为 `true` ，才返回 `true`

它们都具有短路行为：

`any()` 找到第一个满足条件的元素后立刻停止

`all()` 找到第一个不满足条件的元素后立刻停止

示例：

展开

自动换行

let id = "Iterator";

assert!(id.chars().any(char::is\_uppercase));

assert!(!id.chars().all(char::is\_uppercase));

## 10、position（位置）和rposition（右起位置）

`position()` 会针对迭代器的每个条目调用闭包，并返回第一个使闭包返回 `true` 条目的索引。若没找到，返回 `None` 。

示例：

展开

自动换行

let s = "hello";

assert\_eq!(s.chars().position(|c| c == 'e'), Some(1));

assert\_eq!(s.chars().position(|c| c == 'a'), None);

`rposition()` 与之类似，只不过从右侧开始搜索。

虽然 `rposition()` 也是 `Iterator` 上的方法，但调用它的类型还必须额外实现：

`DoubleEndedIterator` ：因为要从右往左找

`ExactSizeIterator` ：因为它需要精确知道剩余长度，从而给出正确索引

`ExactSizeIterator` 的定义可简化为：

展开

自动换行

trait ExactSizeIterator: Iterator {

fn len(&self) -> usize;

fn is\_empty(&self) -> bool;

}

注意： `len()` **返回的是当前未被消耗掉的条目数** 。

示例：

展开

自动换行

let v = vec!\[1, 2, 3, 4, 5\];

let mut iter = v.iter();

println!("{}", iter.len()); // 5

iter.next();

println!("{}", iter.len()); // 4

并不是所有迭代器都能实现 `ExactSizeIterator` 。例如，有些迭代器无法在不额外工作的前提下精确知道还剩多少个条目，因此就不会实现它。

**rposition** 方法示例：

展开

自动换行

let s = b"hello";

assert\_eq!(s.iter().rposition(|&b| b == b'l'), Some(3));

assert\_eq!(s.iter().rposition(|&b| b == b'a'), None);

## 11、fold（折叠）和右起折叠（rfold）

`fold()` 是一个非常通用的消费器，用于把某个条目序列累积成某个结果。给定一个初始累加器和闭包， `fold()` 会反复把“当前累加器”和“下一条目”传入闭包，闭包返回的新值再作为新的累加器，继续处理下一个条目。

如果序列为空，则 `fold()` 直接返回初始累加器。

示例：

展开

自动换行

let a = \[5, 6, 7, 8, 9, 10\];

assert\_eq!(a.iter().fold(0, |acc, \_| acc + 1), 6); // 计数

assert\_eq!(a.iter().fold(0, |acc, x| acc + x), 45); // 求和

assert\_eq!(a.iter().fold(1, |acc, x| acc \* x), 151200); // 乘积

assert\_eq!(a.iter().copied().fold(i32::MIN, std::cmp::max), 10); // 求最大值

`rfold()` 与 `fold()` 基本相同，但要求底层迭代器为双端迭代器，它会从后向前处理条目。

示例：

展开

自动换行

let words = \["hello", "world", "rust"\];

assert\_eq!(words.iter().rfold(String::new(), |acc, x| acc + x + " "),

"rust world hello ");
