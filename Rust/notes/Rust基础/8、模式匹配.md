## 8、模式匹配

## 一、模式匹配的基本用法

在Rust里， **match** 会对一个值做模式匹配：

它会按从上到下的顺序尝试匹配每个分支的模式

第一个匹配成功分支会被执行，后面的分支不再检查

**模式不仅能判断是哪一种情况，还可以在匹配成功时将值的某些部分绑定到变量，供分支使用**

很多时候，模式长得像创建值的语法（例如结构体构造/枚举变体），这是刻意设计：

表达式用来生成值

模式用来拆开值、检查形状并提取内部数据

示例：用match拆枚举并取出数据

展开

自动换行

enum Msg {

Text(String),

Ping,

Move { x: i32, y: i32 }

}

fn describe(m: Msg) -> String {

match m {

Msg::Text(s) => format!("text: {}", s),

Msg::Ping => "ping".to\_string(),

Msg::Move{ x, y} => format!("move to {},{}",x,y),

}

}

说明：

\=>左边的就是模式，=>右边的是要执行的分支语句

如果模式的分支语句只有一条，最后需要使用逗号。如果模式的分支语句有多条，使用花括号{}括起来，并且{}后可加逗号也可不加

Msg::Ping这种不带数据的变体，只能匹配“它是不是 Ping”

Msg::Text(s)这种带一个元素的元组型变体，匹配成功后会将m的内部字段绑定到s，由于s为String类型，因此这里发生了移动

Msg::Move{ x, y }这种带两个字段的结构型变体，匹配成功后将m的内部字段分别绑定到x和y，由于x和y为i32类型（Copy类型），因此这里发生了拷贝

**分支的顺序很重要** ，更具体的模式要放在最前面：

展开

自动换行

let x = 10;

match x {

n => println!("{}",n),

1 => println!("one"),

}

这里第二个分支永远也到不了，因为第一个n会匹配所有情况。编译器会给出警告：unreachable pattern，即不可达的模式。

## 二、模式的类型

模式的类型有很多种，下面依次介绍。

## 1、模式中的字面量、变量和通配符

match不止能匹配枚举，还能匹配很多普通值。最常见的就是三类模式：字面量模式、变量模式和通配符模式。

### （1）字面量模式

字面量模式用于 **精确匹配某个值** 。

示例：

展开

自动换行

fn traffic\_light(color: &str) -> &'static str {

match color {

"red" => "stop",

"green" => "go",

"yellow" => "slow down",

\_ => "unknown",

}

}

注：\_是通配符（见下），表示其他情况。

### （2）变量模式

变量模式用于匹配任何值，并将值绑定到变量。

示例：

展开

自动换行

fn show(n: i32) {

match n {

0 => println("zero"),

x => println!("non-zero: {}", x),

}

}

这里的x是变量模式，它能匹配任何没被前面分支匹配到的值，并把该值绑定到变量。这类变量模式类似于C/C++的switch里的default分支，但它能拿到值。

### （3）通配符模式

**通配符模式用\_表示** ，表示匹配任何值，但不做绑定。

示例：

展开

自动换行

fn maybe\_print(n: i32) {

match n {

0 => println("zero"),

\_ => {}

}

}

match会穷尽所有情况，即Rust会要求每个match覆盖所有可能值。

如果没有覆盖完，编译会报错

如果不想写所有具体情况，通常就用\_兜底

## 2、元组型模式和结构型模式

当想在一次匹配里同时处理多个值，或在结构中提取多个多个字段时，元组/结构体模式很有用。

**元组型模式用于一次匹配多个位置** 。

示例：

展开

自动换行

fn describe\_point(x: i32, y: i32) -> &'static str {

use std::cmp::Ordering::\*;

match (x.cmp(&0), y.cmp(&0)) {

(Equal, Equal) => "origin",

(\_, Equal) => "on the x axis",

(Equal, \_) => "on the y axis",

(Greater, Greater) => "in Q1",

(Less, Greater) => "in Q2",

other => "somewhere else",

}

}

结构体模式按字段名匹配并绑定。

示例：

展开

自动换行

struct Point {

x: i32,

y: i32

}

fn describe(p: Point) -> String {

match p {

Point{ x: 0, y: b} => { format!("at y-axis, y={}", b) },

Point{x: a, y: b} => { format!("at ({}, {})", a, b) },

}

}

Point { x: 0, y: b }：要求x必须是0，同时把y绑定出来

Point { x: a, y: b }：把两个字段都绑定出来。

**注意，绑定的变量可以写成与字段同名的变量，因此Point { x: a, y: b }可以写为Point { x: x, y: y }，由于和字段同名，该可以简写为Point { x, y }** 。

展开

自动换行

fn describe(p: Point) -> String {

match p {

Point{ x: 0, y} => { format!("at y-axis, y={}", y) },

Point{x, y} => { format!("at ({}, {})", x, y) },

}

}

当结构体的字段很多，而模式匹配时只关注少数字段时，可以使用**..**忽略其他字段。

示例：

展开

自动换行

struct User {

name: String,

age: u8,

city: String,

}

fn show\_name(user: User) {

match user {

User { name,.. } => println!("name = {}", name),

}

}

说明：User { name,.. } 的..表示忽略age和city字段。

## 3、数组型模式和切片型模式

数组和切片都表示一段连续元素，所以在模式中都使用 **\[...\]** 书写。

数组型模式用来匹配数组。 **数组型模式长度固定，适合位置有特殊含义的数据** 。

例如，将HSL（色相、饱和度、亮度）颜色值转换为RGB（红色、绿色、蓝色）颜色值时，具有零亮度或全亮度的颜色只会是黑色或白色。可以同match表达式简单处理。

展开

自动换行

fn hsl\_to\_rgb(hsl: \[u8; 3\]) -> \[u8; 3\] {

match hsl {

\[\_, \_, 0\] => \[0, 0, 0\],

\[\_, \_, 255\] => \[255, 255, 255\],

...

}

}

**切片型模式长度可变，可配合..**。

示例：

展开

自动换行

fn greet(names: &\[&str\]) {

match names {

\[\] => println!("Hello, nobody."),

\[one\] => println!("Hello, {}.", one),

\[first, second\] => println!("Hello, {} and {}.", first, second),

\[first,.., last\] => println!("Hello, {}... {}.", first, last),

}

}

注意：这里的one，first等变量的类型都是 **&&str** 。这是因为match的对象是names: &\[&str\]（对切片的引用）。 **因为是引用，Rust为了避免不把借来的东西搬走，会自动把模式里的变量绑定改为借用绑定** ：如果one是&str，会转移走names所引用的切片的对应元素的所有权，而names本身就是引用。

接下来介绍引用型模式时会详细介绍。

## 4、引用型模式

Rust的模式匹配里，和 **引用/借用直接相关** 的两类写法是：

**ref / ref mut模式：** 在匹配时借用（而不是移动）被匹配值的某个部分

**&模式：** 在模式中匹配一个引用，并从引用里取出里面的东西（ **本质是对引用做一次解引用再继续匹配** ）

### （1）ref / ref mut模式

如果 **匹配的是一个拥有所有权的值（不是引用）** ，并且把其中的字段绑定到变量上，那么这些这段可能被移动出来。

示例：

展开

自动换行

struct User {

name: String,

lang: String,

age: u8,

}

fn demo(user: User) {

match user {

User { name, lang,.. } => {

println!("{} uses {}", name, lang);

// println!("{:?}", user); // 错：user 的字段被 move 出来了，user 已经“部分移动”

}

}

}

这里 name、lang 是 String，不是 Copy，所以会被移动到局部变量里；此时 user 变成“部分被移动”，你就不能再借用/使用 user 了。

可以用 ref 借用字段，这样字段不会移动，原值还可以继续使用。通过用ref让绑定变量变成对字段的引用：

展开

自动换行

fn demo(user: User) {

match user {

User { ref name, ref lang,.. } => {

// name: &String, lang: &String

println!("{} uses {}", name, lang);

// 这里 user 没有被消耗（字段没 move 出去），因此你仍然可以继续使用 user

// （在这个 match 分支里是否还能直接用 user，取决于你是否还持有 user 的所有权变量；

// 关键点是：不会出现“部分移动”问题）

}

}

}

ref name表示：不要把name移动出去，改为借用。此时的name的类型就是&String。

如果需要在匹配成功的分支里直接修改匹配到的值，可以使用 **ref mut** 借出一个可变引用。

示例：

展开

自动换行

#\[derive(Debug)\]

struct User {

name: String,

lang: String,

age: u8,

}

fn demo(mut user: User) {

match user {

User { ref name, ref mut lang,..} => {

lang.clear();

lang.push\_str("C++");

println!("name: {}, lang: {}", name, lang);

println!("{:?}", user);

}

}

}

**注意：要产生&mut借用，被匹配的对象必须允许可变借用，因此demo的user参数声明为mut**

### （2）&模式

&出现在表达式里表示创建引用，但出现在模式里表示匹配引用：

表达式：&value产生&T

模式：&pat只匹配&T类型的值，并对其做一次解引用后再用pat继续匹配

示例：

展开

自动换行

struct Point { x: i32, y: i32 }

fn show(p: &Point) {

match p {

&Point { x, y } => {

// 这里 x, y 是 i32（因为 i32 是 Copy，会复制出来）

println!("({}, {})", x, y);

}

}

}

这里 p 的类型是 &Point，模式 &Point { x, y } 的意思是：

先确认它是一个引用 &Point；

解引用得到Point

再按结构体模式取出字段

此时的借用规则仍然有效，如果引用指向的结构体字段不是Copy（比如String），则不能通过&模式把它移动出来：

展开

自动换行

#\[derive(Debug)\]

struct User {

name: String,

lang: String,

age: u8,

}

fn demo(user: &User) {

match user {

&User { name, lang,..} => {

println!("name:{},lang:{}", name, lang);

}

}

}

上述代码是错误的，因为此时name和lang会移动所有权，但user是共享引用。

可以这样解决：

展开

自动换行

fn demo(user: &User) {

match user {

&User { ref name, ref lang,..} => {

println!("name:{},lang:{}", name, lang);

}

}

}

&在Option/迭代器里很常见。当拿到的是Option<&T>，常见的写法是Some(&x)：Some(&x)只在T: Copy时特别顺手，因为它相当于是把T复制出来。

示例：

展开

自动换行

fn demo(next: Option<&char>) {

match next {

Some(&c) => println!("next char = {}", c),

None => println!("end"),

}

}

如果T不是Copy（比如String），一般匹配成Some(s)，让s保持引用类型，而不是转移所有权。

## 三、匹配守卫

有时候模式本身匹配成功还不够，还需要额外条件为真，这个额外条件就称为匹配守卫（match guard）。

**匹配守卫的写法就是在模式和=>之间加上if条件：**

展开

自动换行

PATTERN if CONDITION => EXPR

匹配流程：

先判断值是否匹配 PATTERN；

若匹配成功，再计算 CONDITION；

只有条件为真才执行该分支；否则该分支视为不匹配，继续尝试下一个分支

示例：

展开

自动换行

fn choose\_page(current: u32, clicked: Option<u32>) -> Result<u32, &'static str> {

match clicked {

None => Err("not on a page"),

Some(p) => {

if p == current { Err("you clicked the current page") }

else { Ok(p) }

}

}

}

用匹配守卫，可改写为：

展开

自动换行

fn choose\_page(current: u32, cliked: Option<u32>) -> Result<u32, &'static str> {

match clicked {

None => Err("not on a page"),

Some(p) if p == current => Err("you clicked the current page"),

Some(p) => Ok(p),

}

}

## 四、匹配多种可能性

有时希望一个分支能匹配多种情况。Rust提供了两种常用写法：

**或模式：** pat1 | pat2 | pat3

**范围模式：** start..=end（含端点），start..end（不含端点）

### （1）或模式

用|将多个情况合并为一个分支。

展开

自动换行

fn is\_vowel(c: char) -> bool {

'a' | 'e' | 'i' | 'o' | 'u' => true,

\_ => false,

}

这样写的好处是： **多个分支返回同样的结果时，可以合并** ，减少重复代码。

|可以与解构/引用模式一起用。

示例：

展开

自动换行

fn at\_end(peeked: Option<&char>) -> bool {

match peeked {

Some(&'\\n') | Some(&'\\r') | None => true,

\_ => false,

}

}

### （2）范围模式

范围模式匹配一个区间的值。start..=end表示匹配从start到end之间的所有值，包括端点。start..end表示匹配start到end-1之间的所有值，不含端点。

示例：

展开

自动换行

fn classify(c: char) -> &'static str {

match c {

'0'..='9' => "digit",

'a'..='z' | 'A'..='Z' => "letter",

' ' | '\\n' | '\\t' => "whitespace",

\_ => "other",

}

}

此外，还有 **x..**的形式，表示匹配从x开始直到该类型最大值的所有值。

示例：

展开

自动换行

fn level(score: u32) -> &'static str {

match score {

0..=59 => "fail",

60..=89 => "pass",

90.. => "excellent", // 90 到 u32::MAX

}

}

## 五、使用@模式绑定

**x @ pattern** 的作用是：

先用pattern去匹配

匹配成功时，把“整个匹配的值”绑定到变量x

同时仍然可以在pattern内部进行更细的匹配

可以将其理解为： **既想按照某个模式匹配和检查又想把原值整体保存下来** 。

示例：

展开

自动换行

#\[derive(Debug)\]

enum Msg {

Text(String),

Ping,

}

fn handle(m: &Msg) {

println!("{:?}", m);

}

fn demo(m: Msg) {

match m {

whole @ Msg::Text(\_) => handle(&whole),

\_ => {}

}

}

**@对范围模式特别有用，既可以检查范围又可以拿到值** ：

展开

自动换行

fn read\_token(c: char) -> String {

match c {

num @ '0'..='9' => format!("digit: {}", num),

\_ => "other".to\_string()

}

}

## 六、简洁控制流

## 1、if let

当只关心一种模式时，使用match会显得有些啰嗦：必须写一个兜底分支。if let就是为这种场景准确的简洁写法。

基本形式：

展开

自动换行

if let PATTERN = EXPR {

// 匹配成功：这里可以使用PATTERN绑定的变量

}

EXPR匹配了PATTERN：进入代码块，并完成变量绑定

不匹配：跳过此块，什么也不做

示例：

展开

自动换行

let some\_value = Some(10u8);

if let Some(val) = some\_value {

println!("Got val {}", val);

}

这等价于：

展开

自动换行

let some\_value = Some(10u8);

match some\_value {

Some(val) => println!("Got val {}", val),

\_ => {},

}

因此， **if let可以理解为match的语法糖：只写关系的模式，其他情况自动忽略** 。

如果还是想处理不匹配的情况，还可以增加else分支：

展开

自动换行

let some\_value = Some(10u8);

if let Some(val) = some\_value {

println!("Got val {}", val);

} else {

println!("No val");

}

## 2、let else

**let else用于匹配失败就立即退出的场景，可以显著减少嵌套，让主流程更清楚** 。

基本语法：

展开

自动换行

let PATTERN = EXPR else {

// 匹配失败时执行

// 这里必须提前退出：return / break / continue / panic!

}

若EXPR匹配PATTERN：把绑定写入变量，继续向下执行

若不匹配，执行else块，并且必须在块内提前退出（编译器会强制这一点）

示例1:

Option失败就return。

展开

自动换行

fn process(value: Option<i32>) {

let Some(v) = value else {

return;

};

println!("{}", v);

}

示例2:

Result失败就panic（或返回错误）。

展开

自动换行

fn parse(text: &str) -> i32 {

let Ok(num) = text.parse::<i32>() else {

panic!("not a number");

};

num

}

## 3、while let

while let是把 **循环 + 模式匹配** 合在一起的简洁写法：只要匹配成功就继续循环；匹配失败就退出循环。它特别适合不断尝试获取下一个值，知道没有/失败为止的流程。

基本语法：

展开

自动换行

while let PATTERN = EXPR {

// 循环体：可以使用PATTERN绑定的变量

}

执行规则：
