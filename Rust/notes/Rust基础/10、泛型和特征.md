# 一、泛型

## 1、泛型的概念

泛型（Generics）是对 **具体类型的抽象替代** ：通过在类型层面做抽象编程，让函数、结构体、枚举等可以对不同类型复用同一套实现，从而减少重复代码，提升复用能力。

泛型代码本身不是最终的代码，而是一种模板或蓝图，其中包含一些类型/常量的占位符。编译器在编译期会根据实际使用的具体类型生成对应的专用代码，这称为 **单态化** 。

## 2、泛型函数

在函数中使用泛型，需要声明 **类型参数** 。具体来说，在函数名后使用 **<T>** 声明类型参数。类型参数通常使用 **大驼峰（PascalCase）命名** ，且一般很短，通常使用一个大写字母。

基本语法：

```text
fn add<T>(a: T, b: T) -> T {

// 函数体

a + b

}
```
说明：上述写法中a和b使用了同一个T，表示它们必须是同一种具体类型。

示例（实现两个值的交换）：
```rust
fn swap<T>(a: &mut T, b: &mut T) {

std::mem::swap(a, b);

}

fn main() {

let mut n1 = 1;

let mut n2 = 2;

println!("Before swap: n1={}, n2={}", n1, n2);

swap(&mut n1, &mut n2);

println!("After swap: n1={}, n2={}", n1, n2);

let mut s1 = String::from("hello");

let mut s2 = String::from("world");

println!("Before swap: s1={}, s2={}", s1, s2);

swap(&mut s1, &mut s2);

println!("After swap: s1={}, s2={}", s1, s2);

}
```
结果说明：

swap<T>中的T是声明的泛型参数

调用时编译器会根据实参自动推导T的具体类型：

swap(&mut n1, &mut n2)推导出T的类型为i32

swap(&mut s1, &mut s2)推导出T的类型为String

除了自动推导，也可以 **显式指定具体类型** ，通过 **turbofish（金鱼叉）语法::<...>标注** 。
```rust
swap::<String>(&mut s1, &mut s2);
```
注：如果实参已经足够推导，一般不必显式写 turbofish；但在编译器推导不出来、或想强制选择某个类型/实现时非常有用。

示例：
```rust
// 调用无参数的泛型方法

let v1 = (0..1000).collect(); // 错误，无法推断类型

let v2 = (0..1000).collect::<Vec<i32>>(); // 正确
```
## 3、泛型结构体

结构体也可以定义泛型。结构体的泛型参数通常用于描述字段中出现的类型，也可以在后续为结构体实现方法（impl块）时参与方法参数和返回值类型声明。

基本语法：

```rust
struct Point<T> {

x: T,

y: T,

}
```
说明：Point<T> 使用同一个 T，因此 x 与 y 必须是同一种具体类型。

可以定义多个泛型参数：
```rust
struct Point<T, U> {

x: T,

y: U,

}
```
说明：x 与 y 可以是不同类型，例如 Point<i32, f64>。
## 4、泛型枚举

枚举中也可以定义泛型。枚举的泛型参数通常用于变体所携带的数据类型（变体字段类型）。定义时在枚举名后加 \`<>\` 声明泛型参数。注意：并不是每个变体都一定要用到该泛型参数。

在 Rust 中有两个广泛使用的泛型枚举： **Option<T> 和 Result<T, E>** 。（它们在预导入模块（prelude）中，通常无需显式 use 就能直接使用。）

### （1）Option<T>

Option<T> 常作为函数返回值，表示“有值/无值”：

有值：Some(v)，v的类型为T

无值：None

原型如下：

```rust
pub enum Option<T> {

Some(T),

None,

}
```
### （2）Result<T, E>

Result<T, E> 常作为函数返回值，表示“成功/失败”：

Ok(v)：成功，v的类型为T，为期望返回值类型

Err(e)：失败，e通常为错误类型（error type）

原型如下：

```rust
pub enum Result<T, E> {

Ok(T),

Err(E),

}
```
## 5、泛型方法

在Rust中，泛型不仅可用于结构体、枚举的定义，也可用于为结构体/枚举实现的方法。方法既可以使用类型本身的泛型参数，也可以在在方法级别单独引入新的泛型参数。

下面以结构体为例，总结三种常见情形。

### （1）在impl块中声明结构体已有的泛型参数

如果结构体本身是泛型结构体，那么为该结构体实现方法时，需要在impl块后声明这些泛型参数，并在impl后的结构体类型中也要加上。

示例：

```rust
struct Point<T> {

x: T,

y: T,

}

impl<T> Point<T> {

fn new(x: T, y: T) -> Self {

Self { x, y }

}

fn x(&self) -> &T {

&self.x

}

fn y(&self) -> &T {

&self.y

}

}

fn main() {

let p = Point::new(1, 2);

println!("p.x = {}", p.x());

println!("p.y = {}", p.y());

}
```
注意：

**impl<T> Point<T>中，impl<T>表示声明泛型参数，Point<T>表示该实现适用于任意类型** 。

方法中可直接使用T，因为它在impl<T>中已经声明

观察new函数，其返回值类型为Self，指代的就是Point<T>。在函数体内，Self也指代Point<T>。

也可以写成如下形式：
```text
fn new(x: T, y: T) -> Point<T> {

Point { x, y }

}
```
函数体内的Point可以省略<T>，编译器会帮助我们自动推断。
### （2）为方法引入单独的额外参数

方法不仅可使用结构体自身的泛型参数，也可以在方法级别单独定义新的泛型参数，这些泛型参数只在该方法内部有效。

```rust
#\[derive(Debug)\]

struct Point<T, U> {

x: T,

y: U,

}

impl<T, U> Point<T, U> {

fn new(x: T, y: U) -> Self {

Self { x, y }

}

fn x(&self) -> &T {

&self.x

}

fn y(&self) -> &U {

&self.y

}

fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {

Point {

x: self.x,

y: other.y,

}

}

}

fn main() {

let p1 = Point::new(10, 3.14);

let p2 = Point::new(1.11, 20);

let p = p1.mixup(p2);

println!("p: {:?}", p);

}
```
注意：

mixup<V, W> 为该方法单独声明新的泛型参数 V、W

V、W 与结构体原有的 T、U 没有任何关系

Rust 支持方法级别引入额外泛型参数，提高方法灵活性
### （3）为具体类型参数实现方法

当只想为结构体的某个具体类型版本实现方法时，可以直接在结构体名后指定具体类型，而无需声明泛型参数。

示例：

```rust
#\[derive(Debug)\]

struct Point<T> {

x: T,

y: T,

}

impl Point<f64> {

fn new(x: f64, y: f64) -> Self {

Self { x, y }

}

fn distance\_from\_origin(&self) -> f64 {

(self.x.powi(2) + self.y.powi(2)).sqrt()

}

}

fn main() {

let p = Point::new(3.0, 4.0);

println!("dist: {}", p.distance\_from\_origin());

}
```
注意：

impl Point<f64> 中已明确指定 T = f64

不需要再写 impl<T>

该 impl 块中的方法只适用于 Point<f64>
## 6、泛型代码的性能

**使用泛型代码与手写具体类型代码在运行时的性能表现通常等价** 。原因是 Rust 在编译期采用 **单态化（Monomorphization）** 策略：编译器会根据泛型定义，在编译阶段为每一种实际使用到的具体类型生成对应的专用代码，因此运行时不存在额外抽象开销，实现 **“零成本抽象”** 。

以最常用的泛型枚举为例：

```rust
let some\_integer = Some(5);

let some\_float = Some(3.14);

Option<T> 的定义：
```
```rust
enum Option<T> {

Some(T),

None,

}
```
编译器在编译时发现Option<T>实际被用于i32和f64类型，于是会分别生成专用版本（仅用于理解，并非真实存在的类型名）：
```rust
enum Option\_i32 {

Some(i32),

None,

}

enum Option\_f64 {

Some(f64),

None,

}
```
因此，泛型通常不会引入运行时性能损耗（静态分发）。但代价主要体现在编译期与二进制体积：具体类型越多，实例化出的代码越多。

**后面介绍Trait对象（&dyn Trait/Box<dyn Trait>）时，采用的则是动态分发的形式，会有一次间接调用开销。而泛型 + Trait Bound（后面介绍）通常是静态分发** 。
## 7、const泛型

前面讨论的都是类型泛型（如T），Rust还支持针对 **编译期常量值** 的泛型，称为 `const泛型` 。它允许在泛型参数中使用编译期已知的常量（常见如 `usize` ，也可为 `bool` 等），从而让类型在编译期依赖一个常量参数。

注意： **const泛型参数必须是编译期可确定的常量表达式，而不是运行时任意值** 。

示例1：

```rust
fn dot\_product<cosnt N: usize>(a: \[f64; N\], b: \[f64; N\]) -> f64 {

let mut sum = 0.;

for i in 0..N {

sum += a\[i\] \* b\[i\];

}

sum

}
```
短语 `<const N: usize>` 指出函数dot\_product需要一个泛型参数N，该参数必须是 `usize` 类型。给定N，函数能接受两个\[f64; N\]的数组参数，并将对应的元素乘积相加。

**N和普通usize参数的区别是，它可以在函数签名或函数体的类型中使用** 。

与类型参数一样，既可以显式提供常量参数，也可以让Rust推断它们。
```rust
let r1 = dot\_product(\[1.1, 2.2\], \[3.3, 4.4\]);

let r2 = dot\_product::<3>(\[1.0, 1.0, 1.0\], \[2.0, 2.0, 2.0\]);
```
示例2：
```rust
struct Buffer<T, const N: usize> {

data: \[T; N\],

}

impl<T, const N: usize> Buffer<T, N> {

pub fn new(data: \[T; N\]) -> Self {

Self { data }

}

}
```
上述的泛型结构体Buffer包含类型参数T和常量泛型N，其中定义了一个\[T; N\]的data数组。

示例3：

const泛型也可以结合const fn（常量函数）生成编译期常量，用作const泛型实参。
```rust
struct Buffer<const N: usize> {

data: \[u8; N\];

}

const fn compute\_buffer\_size(factor: usize) -> usize {

factor \* 1024

}

const BUFFER\_SIZE: usize = compute\_buffer\_size(4); // 4KB

fn main() {

let buffer = Buffer::<BUFFER\_SIZE> {

data: \[0u8; BUFFER\_SIZE\];

};

println!("size: {}", buffer.data.len());

}
```
# 二、特征

## 1、Trait的核心概念

Rust通过两个特性来支持多态：泛型和特征。泛型前面已经介绍，但前述的泛型不涉及特征，事实上，泛型和特征紧密相关，例如泛型函数会在限界（Trait Bound）中使用特征限制它能使用哪些类型的参数进行调用。

`Trait（特征，也称特型）` ，它是Rust语言体系中的 **接口或抽象基类** ，它告诉编译器：某个类型具备一组与其他类型共享的能力（方法集合），主要包含一组方法签名（以及可选的默认实现）。

例如下面的println!之所以可以使用{}格式符打印String类型s的值，是因为String类型实现了 `std::fmt::Display` 特征。

```rust
let s = String::from("Hello");

println!("{}", s);
```
再例如 `std::io::Write` 特征，其简化定义如下：
```rust
trait Write {

fn write(&mut self, &\[u8\]) -> Result<usize>;

fn flush(&mut self) -> Result<()>;

fn write\_all(&mut self, buf: &\[u8\]) -> Result<()>;

// Write特征还提供了其他方法，这里只展示上述三个。

}
```
标准库的 `File` 、 `TcpStream` 以及 `Vec<u8>` 等都实现了Write Trait，因此这些类型可以调用Write Trait的方法`.write()` 、`.flush()` 等。

此外，一些内置类型还是Rust为运算符重载和其他特性提供的 **语言级钩子** 。语言级钩子是编译器/语言本身定义和识别的特殊接口和约定，开发者只要按规定实现，语言就会在特定语法和行为发生时，自动调用代码。

例如执行：
```rust
let c = a + b;
```
此处没有调用任何函数，但Rust编译器会将其翻译为 `Add::add(a, b)` ，但前提是a、b的类型实现了 `std::ops::Add` 特征。
## 2、Trait的定义

Rust使用 `trait` 关键字定义Trait。

基本语法：

```rust
pub trait Interface {

fn func(&self); // 方法签名

}
```
注意：pub关键字用于该Trait对外可见。

示例（定义一个Animal Trait）：
```rust
pub trait Animal {

fn eat(&self); // 对 Animal 而言都有 eat 共同行为

}
```
## 3、Trait的实现

### （1）为具体类型实现Trait

使用 `impl Trait for Type` 语法为具体类型实现Trait。以Animal Trait为例，定义Dog结构体并实现Animal。

```rust
pub trait Animal {

fn eat(&self);

}

struct Dog {

name: String,

}

impl Animal for Dog {

fn eat(&self) {

println!("Dog {} is eating meat.", self.name);

}

}

fn main() {

let dog = Dog { name: String::from("Bob") };

dog.eat();

}
```
注意事项：

孤儿规则（Orphan Rule）：要为 `A` 类型实现特征 `T` ， `A` 或 `T` 至少有一个在当前crate中。更准确来说，实现块所在的crate中必须拥有 `A` 或 `T` 的定义。 例如，不可以在自己的crate写 `impl Write for u8` ，因为Write和u8都是标准库定义的。如果Rust允许这么做，那么不同的crate可能有u8的多个Write实现，那么编译器就不知道使用哪一个实现了。
### （2）默认方法实现

Rust 允许在 Trait 中给部分方法提供默认实现。类型实现 Trait 时可以：

直接复用默认实现

或重写该方法

示例：

```rust
pub trait Animal {

fn eat(&self);

fn sound(&self) {

println!("The animal is making sound...");

}

}

struct Dog {

name: String,

}

impl Animal for Dog {

fn eat(&self) {

println!("Dog {} is eating meat.", self.name);

}

fn sound(&self) {

println!("Dog {} is making sound: wang wang wang", self.name);

}

}

struct Cat {

name: String,

}

impl Animal for Cat {

fn eat(&self) {

println!("Cat {} is eating fish.", self.name);

}

// 复用 sound 的默认实现

}

fn main() {

let dog = Dog { name: String::from("Bob") };

let cat = Cat { name: String::from("Tom") };
```
实际上，之前的 `Write` 特征的方法除了 `write()` 和 `flush()` 方法，其他方法都有默认实现。
### （3）默认实现调用其他方法

Trait 的默认实现可以调用 Trait 中的其他方法，即使这些方法没有默认实现。但当类型实现该 Trait 时，必须实现默认实现所依赖的方法。

示例：

```rust
pub trait Animal {

fn name(&self) -> String;

fn eat(&self);

fn sound(&self) {

println!("The animal {} is making sound...", self.name());

}

}

struct Dog {

name: String,

}

impl Animal for Dog {

fn name(&self) -> String {

self.name.clone()

}

fn eat(&self) {

println!("Dog {} is eating meat.", self.name());

}

fn sound(&self) {

println!("Dog {} is making sound: wang wang wang", self.name());

}

}

struct Cat {

name: String,

}

impl Animal for Cat {

fn name(&self) -> String {

self.name.clone()

}
```
这种方式在 `std::iter::Iterator` 上尤为突出，该特征除了 `next()` 方法外，其他方法都是默认实现。很多默认实现都会间接调用 `next()` 。
## 4、使用Trait

定义Trait并为类型实现Trait后，就可以调用Trait的方法了。但前面的例子都是在当前crate中定义Trait和类型，因此此时还察觉不到问题。

事实上，调用Trait的方法有一个需要注意的规则：Trait本身需要在作用域内，否则Trait的所有方法都不可见。

例如，创建一个文件获得 `File` 句柄进行文件写入的时候（ `File` 实现了 `Write` ）:

```rust
let mut file = File::crate("hello.txt")?;

file.write\_all(b"hello world\\n")?; // 错误，没有名为\`write\_all\`的方法
```
**注：?操作符用于传递错误，在后面错误处理会介绍** 。这是需要知道 `File::crate()` 返回Result，如果是Err变体代表打开失败，?会提前结束，将Err传递给上层。

但上述代码报错了：没有write方法。

必须使用 `use` 关键字将 `Write` 特征引入作用域：
```rust
use std::io::Write;

let mut file = File::crate("hello.txt")?;

file.write\_all(b"hello world\\n")?; // 正确
```
之所以有这样的规则，是因为可以使用特征为任意类型实现方法。但如果多个库定义了同名特征，则会产生命令冲突。当然，如果确实遇到冲突，也可以使用 **带完全限定符** 的方法名明确（后续介绍），

当然，也会看到 `Clone` 和 `Iterator` 特征的方法在没有任何特殊导入情况下就能正常工作，因为它们是标准库预导入（prelude）的一部分，Rust会将这些名称自动导入每一个模块中。
## 5、Trait作为函数参数

### （1）Trait Bound

**不能直接以Trait作为函数的参数** 。因为Trait不是一个具体的类型，它属于DST（动态类型大小），大小在编译期不可确定，而编译器在编译期间必须知道参数的具体大小。

例如，以下写法会编译失败：

```text
fn animal\_eating(animal: Animal) {

}
```
因此，Trait作为函数参数时常常需要配合泛型一起使用，称为 `Trait Bound` （特征约束，也称限界）。

例如，编写一个say\_hello函数，该函数不关心具体类型，但要求类型必须实现 `Write` 特征。

示例：
```rust
use std::io::Write;

fn say\_hello<W: Write>(writer: &mut W) -> Result<()>

writer.write\_all(b"hello world\\n")?;

writer.flush

}

`W: Write` 就是限界，表示W必须是实现了Write特征的类型。上述的 **say\_hello本质上还是泛型函数，它是一个“模板”，只有当实际用具体类型调用时，编译器在编译器才会为生成专用代码，这称为静态分发** 。

Trait Bound还有一种语法糖形式： `impl Trait` 。 `impl Trait` 仍然表示任意实现了该Trait的类型。前面的say\_hello可以改写为：
```
```rust
use std::io::Write;

fn say\_hello(writer: &impl Write) -> Result<()>

writer.write\_all(b"hello world\\n")?;

writer.flush

}
```
注意： `impl Trait` 参数形式常见三种（取决于所有权/可变性需求）：

impl Trait（获取所有权）

&impl Trait（共享引用）

&mut impl Trait（可变引用）

在简单场景 ， `impl Trait` 足够；更复杂场景更推荐 Trait Bound，例如两个参数需要“同一种具体类型”时：

impl Trait
```rust
fn func(animal1: &impl Animal, animal2: &impl Animal ) {

// animal1 与 animal2 可以是不同具体类型，只要都实现 Animal

}

Trait Bound
```
```text
fn func<A: Animal>(animal1: &A, animal2: &A, ) {

// animal1 与 animal2 必须是同一种具体类型 T

}
```
当一个泛型参数需要同时满足多个Trait时（多重特征约束），通过 `+` 组合:

impl Trait
```rust
use std::fmt::Debug;

fn show\_animal(animal: &(impl Animal + Debug)) {

println!("{:?}", animal);

}

Trait Bound
```
```rust
use std::fmt::Debug;

fn show\_animal<A: Animal + Debug>(animal: &A) {

println!("{:?}", animal);

}
```
如果 `+` 组合的Trait太多，会导致函数签名可读性较差，因此Rust还提供了 `where子句` 。

示例：
```rust
use std::fmt::Debug;

fn show\_animal<A>(animal: &A)

where

A: Animal + Debug,

{

println!("{:?}", animal);

}
```
### （2）Trait Object

`Trait Object` （特征对象）用于在 **运行时通过动态分发调用实现某个Trait的不同类型** 。它允许在不知道具体类型的情况下，通过Trait接口操作对象。

在Rust中， `Trait` 这类名字在语境中可能代表两件事：

`Trait Bound` ：用于泛型/静态分发，例如T: Write、impl Write。

`Trait Object` ：用于动态分发，例如&dyn Write、Box<dyn Write>。

特征对象需要 `dyn` 关键字标注，目的就是为了 **消除歧义：明确告诉编译器这里要用特征对象，采用动态分发** 。

Trait对象的常见形式有：

&dyn Trait：Trait对象的共享引用（不拥有值）

&mut dyn Trait：Trait对象的可变引用（不拥有值）

Box<dyn Trait>：拥有Trait对象（堆分配）

Rc<dyn Trait>、Arc<dyn Trait>：引用计数拥有（共享所有权）

不能直接写如下代码：

```rust
let writer: dyn std::io::Write; // 错误
```
因为dyn Trait是DST，它的大小在编译期不确定，而局部变量/字段默认要求是 `Sized` ，必须在编译期知道大小。

因此，Trait Object必须放在指针/智能指针后面。

Trait对象在内存中本质是一个 **胖指针** ，由两部分组成：

data\_ptr（数据指针）：指向具体实现了Trait的值

vtable\_ptr（虚表指针）：指向该具体类型实现了该Trait的vtable表

对&dyn Trait来说可理解为：
栈

&dyn Trait

┌────────────────────────┐

│ data\_ptr -> 实例地址 │

│ vtable\_ptr -> vtable │

└────────────────────────┘

vtable在概念上通常包含：

该类型为该Trait实现的各个方法的函数指针

drop\_in\_place等析构的相关入口（用于Box<dyn Trait>等需释放时，例如Box<dyn Write>被擦除掉具体类型，vtable保存指向具体类型析构的入口，确保正确析构，例如Vec<u8>要释放堆区内存，File要关闭文件句柄）

该具体类型的size（大小）和align（对齐）（用于释放堆内存）

当使用Trait对象调用Trait方法时，会发生 **动态分发** ：

通过vtable\_ptr找到虚函数表vtable

在vtable中找到对应方法的函数指针

以data\_ptr作为self进行间接调用

因此，特征对象调用的函数地址不是编译期写死的，而是运行时从vtable取出再调用，这会带来轻微的开销，但也带来了巨大的灵活性（运行时用统一接口操作不同具体类型）。

现在用特征对象对之前的say\_hello函数进行改写：
```rust
use std::io::Write;

fn say\_hello(writer: &mut dyn Write) -> std::io::Result<()> {

writer.write\_all(b"hello world\\n"?;

writer.flush

}
```
**这里的say\_hello则不再是泛型函数，而是一个实打实的普通函数** 。

调用：
```rust
let mut file = File::create("hello.txt")?;

say\_hello(&mut file)?;
```
**注意：say\_hello调用时，会把&mut File（普通引用）自动强制转换为特征对象&mut dyn Write，本质上就是在原本数据指针的基础上，额外补上了vtable指针，从而形成胖指针** 。

动态分发的典型应用场景就是 **异构容器** ，可以将不同具体类型放进同一个容器中。
```rust
trait Animal { fn speak(&self); }

struct Dog;

impl Animal for Dog { fn speak(&self) { println!("Woof!"); } }

struct Cat;

impl Animal for Cat { fn speak(&self) { println!("Meow!"); } }

fn main() {

let animals: Vec<Box<dyn Animal>> = vec!\[Box::new(Dog), Box::new(Cat)\];

for a in animals {

a.speak();

}

}
```
## 6、Trait作为函数返回值类型

### （1）使用impl Trait返回（静态分发）

函数返回值也可以用 `impl Trait` ，表示返回某个实现了该Trait的具体类型。

示例：

```rust
pub trait Animal {

fn name(&self) -> String;

fn eat(&self);

}

struct Dog {

name: String,

}

impl Animal for Dog {

fn name(&self) -> String { self.name.clone() }

fn eat(&self) { println!("Dog {} is eating meat.", self.name()); }

}

fn return\_animal() -> impl Animal {

Dog { name: String::from("Bob") }

}

fn main() {

let animal = return\_animal();

animal.eat();

}
```
说明：调用方只需知道返回值实现了Animal，不必知道具体类型。

但当 `impl Trait` 作为返回值类型时，函数内部必须返回单一具体类型，不能在不同分支返回不同具体类型。 `impl Trait` 作为返回值时，编译期会确定这个函数实际返回的具体类型，只是这个具体类型对调用发来说被隐藏了。
### （2）Trait对象返回多种类型

如果需要返回多种类型，可用Trait对象：

```rust
fn return\_animal\_dyn(flag: bool) -> Box<dyn Animal> {

if flag {

Box::new(Dog { name: "Bob".into() })

} else {

Box::new(Dog { name: "Alice".into() })

}

}
```
## 7、完全限定方法调用

方法本质上是一种特殊的函数。例如：

```text
"hello".to\_string()
```
等价于把self作为第一个参数传入的函数调用：
```rust
str::to\_string("hello");
```
说明：

`to_string` 对于 `str` 来说是一个方法（接收者是 `&self`

`str::to_string("hello")` 这种写法看起来像类型关联函数调用，但本质仍是方法，只是显式地把self放在第一个参数位置

同一个调用可能有多种等价写法。因为 `to_string` 来自 `ToString` 特征，所以还可以写：
```rust
ToString::to\_string("hello");
```
以及 **完全限定（UFCS）** 的方式：
```text
<str as ToString>::to\_string("hello");
```
完全限定调用（UFCS）的标准形态如下：
```text
<Type as Trait>::method(receiver, args...)
```
含义：

**Type:** 接收者（ `self` ）的具体类型

**Trait** ：方法来自哪个特征

**receiver** ：显式传入 `self` （值 / & / &mut），作为第一个参数

完全限定调用通常应用在如下场景：

多个Trait定义了同名方法，调用时出现了歧义

示例：

例如同一个类型同时实现了两个 trait，它们都提供了同名 draw 方法：
```rust
trait Render { fn draw(&self); }

trait Shoot { fn draw(&self); } // 故意同名：draw=拔枪

struct Outlaw;

impl Render for Outlaw {

fn draw(&self) { println!("draw on screen"); }

}

impl Shoot for Outlaw {

fn draw(&self) { println!("draw a pistol"); }

}
```
这时：
```rust
let o = Outlaw;

// o.draw(); // 歧义：到底是 Render::draw 还是 Shoot::draw？
```
必须明确指定：
```text
Render::draw(&o);

Shoot::draw(&o);
```
或者更完全限定：
```text
<Outlaw as Render>::draw(&o);

<Outlaw as Shoot>::draw(&o);
```
编译器无法推断self的具体类型时

示例：
```rust
let zero = 0;

// zero.abs(); // 0 的类型未定（i32? i64? i128?...），无法选 abs 的实现

i32、i64等都实现了abs方法，调用哪一个不知道。此时需要指定类型：
```
```rust
i32::abs(zero);
```
把方法当作“函数值”传递

在学习了迭代器和闭包之后，通常会写这样的代码：
```text
line.split\_whitespace().map(|s| s.to\_string())
```
这里map传入的其实是闭包（匿名函数）。

也可以写为：
```text
line.split\_whitespace().map(ToString::to\_string)
```
这里的 `ToString::to_string` **不是在调用** ，而是一个“函数值/函数项”（function item），可以直接当作map的参数。
# 三、特征进阶

## 1、基于Trait Bound的条件方法实现

可以在impl块上使用Trait Bound，为满足约束的类型有条件地提供额外方法。

示例：

```rust
use std::fmt::Display;

struct Pair<T> {

x: T,

y: T,

}

// 为所有类型T实现方法

impl<T> Pair<T> {

fn new(x: T, y: T) -> Self {

Self { x, y }

}

}

// 只为实现了Display和PartialOrd的类型T实现方法

impl<T: Display + PartialOrd> Pair<T> {

fn cmp\_display(&self) {

if self.x >= self.y {

println!("The largest number is x: {}", self.x);

} else {

println!("The largest number is y: {}", self.y);

}

}

}
```
## 2、基于Trait Bound的条件Trait实现

Rust支持为所有满足某Trait的类型批量实现另一个Trait（Blanket Implementation，毯子实现）。

基本语法：

```rust
impl<T: Trait1> Trait2 for T {

}
```
标准库示例：为所有实现了Display的类型实现ToString
```rust
impl<T: std::fmt::Display> ToString for T {

//...

}
```
因此，常见写法：
```rust
let s1 = 3.to\_string();

let s2 = "hello".to\_string();
```
因为i32和&str都实现了Display，所以标准库为它们统一实现了ToString。后续自定义类型时，只要为该类型实现了Display，则自动也为其实现了ToString。
## 3、特征中的Self

特征可以使用关键字 `Self` 作为类型。例如， `Clone` 特征的简化定义如下：

```rust
pub trait Clone {

fn clone(&self) -> Self;

//...

}
```
含义：对任意实现了 `Clone` 的类型 `T` 来说， `clone` 返回的类型一定是 **同一个** `T` 。比如 `String` 调用 `clone()` 得到的就是 `String` ，而不可能是 `dyn Clone` 或别的类型。

Self在每个impl块里都会被绑定为具体了类型。

示例：
```rust
/// 表示同类型合并：只能和同类型的另一个值合并，合并后还是同类型

pub trait Combine {

fn combine(&self, other: &Self) -> Self;

}
```
两种实现：
```rust
impl Combine for String {

fn combine(&self, other: &Self) -> Self {

format!("{}{}", self, other);

}

}

impl Combine for i32 {

fn combine(&self, other: &Self) -> Self {

self + other

}

}
```
在impl Combine for String中，Self就是String

impl Combine for i32中，Self就是i32

需要注意的是， **使用了Self的特征与特征对象不兼容** 。例如如下的代码会报错：
```rust
fn combine\_any(left: &dyn Combine, right: &dyn Combine) {

let \_x = left.combine(right);

}

left: &dyn Combine 表示：left 可能是 &String 也可能是 &i32，然后Combine::combine的签名是：
```
```text
fn combine(&self, other: &Self) -> Self
```
这里的Self要求self和other必须是同一种类型，但self已经被类型擦除成了dyn Combine，编译器在编译期无法保证right的真实类型和left相同，

因此，Trait对象要求方法签名在擦除具体对象后仍然可调用，而other: &Self，-> Self的这种签名把具体类型必须相同的信息绑定到Self上，动态分发下无法在类型系统里检查，因此这种Trait往往不满足对 **象安全** （dyn 兼容）。

如果确实想要合并不同类型，可以把签名改成特征对象的形式：
```rust
pub trait MetaCombine {

fn meta\_combine(&self, other: &dyn MetaCombine) -> Box<dyn MetaCombine>;

}
```
## 4、子特征和超特征

Rust里可以声明一个特征是另一个特征的扩展，这通常被称为：

`子特征（Subtrait)` ：扩展别人能力的那个trait

`超特征（Supertrait）` ：被要求先具备能力的那个trait

基本语法：

```rust
trait 子: 超 {

}
```
等价于：
```rust
trait 子

where

Self: 超

{

}
```
本质上子特征是给实现者加上一条 `Self: 超特征` 的约束，也是说实现子特征的类型必须也实现了超特征。

示例：

定义子特征
```rust
/// 能被渲染（画出来）

trait Render {

fn render(&self);

}

/// 能移动（位置+朝向），并且它一定能被渲染

trait Movable: Render {

fn position(&self) -> (i32, i32);

fn facing(&self) -> i32;

// 子特征的默认方法里可以直接使用超特征的方法

fn report(&self) {

let (x, y) = self.position();

println!("pos=({x},{y}), facing={}", self.facing());

self.render(); // Ok

}

}
```
类型实现子特征时，必须同时实现超特征：
```rust
struct Player {

x: i32,

y: i32,

dir: i32,

}

impl Render for Player {

fn render(&self) {

println!("draw Player");

}

}

impl Movable for Player {

fn position(&self) -> (i32, i32) { (self.x, self.y) }

fn facing(&self) -> i32 { self.dir }

}
```
注意： `impl Render for Player` 和 `impl Movable for Player` 顺序无所谓
## 5、类型关联函数

大多数面向对象语言中，接口不能包含静态方法或构造函数，但 **Rust的特征可以包含类型关联函数** 。

示例：

```rust
trait StringSet {

/// 创建一个新的空集合 (类型关联函数)

fn new() -> Self;

/// 返回一个包含\`strings\`中所有字符串的集合(类型关联函数)

fn from\_slice(strings: &\[&str\]) -> Self;

/// 判断这个集合中是否包含特定的\`string\`

fn contains(&self, string: &str) -> bool;

/// 把一个字符串添加到集合中

fn add(&mut self, string: &str);

}
```
每个实现StringSet特征的类型都必须实现这4个关联函数。注意， `new()` 和 `from_slice()` 不接受self参数，属于 **类型关联函数** ，它们也扮演着构造函数的角色。

在非泛型代码里，通过`::`语法调用这些函数。假设有两个实现了StringSet的类型SortedStringSet和HashedStringSet：
```rust
let set1 = SortedStringSet::new();

let set2 = HashedStringSet::new();
```
在泛型代码里，也可以使用`::`语法，因为泛型代码（静态分发常见下，类型参数最终会被替换为具体类型（单态化）：
```rust
fn unknown\_words<S: StringSet>(document: &\[String\], wordlist: &S) -> S {

let unknowns = S::new();

for w in document {

if!wordlist.contains(w) {

unknowns.add(w);

}

}

unknowns

}
```
**含有类型关联函数的Trait不兼容特征对象（dyn兼容）** 。当尝试把该Trait用作Trait Object时，编译器需要确保在对象语境下仍然可调用。对于fn new() -> Self这类签名，在对象检查等价于要求返回dyn Trait，但dyn Trait是DST，不能按值返回。此外，类型关联函数没有self接收者，调用时没有对象实例提供vtable指针，因此无法进行动态分发。

下面的代码就会报错：
```rust
trait StringSet {

/// 创建一个新的空集合 (类型关联函数)

fn new() -> Self;

/// 返回一个包含\`strings\`中所有字符串的集合(类型关联函数)

fn from\_slice(strings: &\[&str\]) -> Self;

/// 判断这个集合中是否包含特定的\`string\`

fn contains(&self, string: &str) -> bool;

/// 把一个字符串添加到集合中

fn add(&mut self, string: &str);

}

fn use\_dyn(set: &mut dyn StringSet) {

set.add("hello");

}
```

通常可以给这些关联函数加上 `where Self: Sized` 限制，把它们排除在trait object的接口之外，从而让剩余带&self/&mut self的方法仍然可用于dyn Trait：
```rust
trait StringSet {

/// 创建一个新的空集合 (类型关联函数)

fn new() -> Self

where Self: Sized;

/// 返回一个包含\`strings\`中所有字符串的集合(类型关联函数)

fn from\_slice(strings: &\[&str\]) -> Self

where Self: Sized;

/// 判断这个集合中是否包含特定的\`string\`

fn contains(&self, string: &str) -> bool;

/// 把一个字符串添加到集合中

fn add(&mut self, string: &str);

}

fn use\_dyn(set: &mut dyn StringSet) {

set.add("hello");

}
```
# 四、定义类型之间关系的特征

## 1、关联类型

迭代器是用以遍历某种值序列的对象。Rust内置了对迭代器的支持，标准库提供了 `std::iter::Iterator` 特征，其简化定义如下：

```rust
pub trait Iterator {

type Item;

fn next(&mut self) -> Option<Self::Item>;

//... 其他默认实现方法

}
```
说明：

`type Item` 使用 `type` 关键字定义了一个 `Item` 关联类型，它是迭代器生成的条目值的类型。实现 `Iterator` 的每个类型都必须指定 `Item` 的类型。

`next()` 方法返回迭代器的下一项，其返回值 `Option<Self::Item>` 使用到了关联类型，表示要么返回的是 `Some(item)` ，或者无值访问时返回 `None` 。类型要写成 `Self::Item` 是因为这里的 `Item` 是每个迭代器类型下的一个特性，而不是一个独立的类型。

下面给出了标准库的 `std::env::Args` 实现 `Iterator` 特征的范例：
```rust
impl Iterator for Args {

type Item = String;

fn next(&mut self) -> Option<String> {

//...

}

}

`std::env::Args` 是访问命令行参数函数 `std::env::args()` 返回的迭代器类型，它会生成每一个String类型的命令行参数，因为在 `impl` 中声明了 `type Item = String` 。
```
**泛型代码中可以使用关联函数** ，例如：
```rust
fn collect\_into\_vector<I: Iterator>(iter: I) -> Vec<I::Item> {

let mut results = Vec::new();

for value in iter {

results.push(value);

}

results

}
```
返回值类型为 `Vec<I::Item>` ，表示Vec中的元素类型是迭代器的关联类型。

再例如，实现一个打印迭代器所生成的所有值的函数：
```rust
fn dump<I>(iter: I)

where I: Iterator

{

for(index, value) in iter.enumerate() {

println!("{}: {:?}", index, value);

}

}
```
但上述代码会报错： `<I as Iterator>::Item` doesn't implement `Debug` 。

因此，还需要为迭代器的关联类型增加一个限界：
```rust
use std::fmt::Debug;

fn dump<I>(iter: I)

where I: Iterator, I::Item: Debug

{

for(index, value) in iter.enumerate() {

println!("{}: {:?}", index, value);

}

}
```
这里的 `I::Item: Debug` 限制 `I::Item` 必须是实现 `Debug` 的类型。

也可以具体一点，例如规定迭代器的关联类型就是String，使用 `Iterator<Item=String>` 。 `Iterator<Item=String>` 本身就是一个特征，如果把 `Iterator` 视为所有迭代器类型的全集，则 `Iterator<Item=String>` 就是 `Iterator` 的子集：
```rust
fn dump<I>(iter: I)

where I: Iterator<Item=String>

{

for(index, value) in iter.enumerate() {

println!("{}: {:?}", index, value);

}

}

`Iterator<Item=String>` 与特征对象兼容，但必须确保关联类型都明确写出来：
```
```rust
fn dump(iter: &mut dyn Iterator<Item=String>)

{

for(index, value) in iter.enumerate() {

println!("{}: {:?}", index, value);

}

}
```
## 2、泛型特征

在Rust里， **特征本身也可以带类型参数** ，这种特征称为 **泛型特征** （generic trait）。它和泛型类型（如 `Vec<T>` ）类似，不同的类型参数会得到不同的trait实例。

泛型特征最常见的出现地点就是 **运算符重载** ：Rust把+、\*等运算符翻译为对某个trait方法的调用。

例如，当在程序中写：

```text
a + b

Rust会将其解析为：
```
```text
std::ops::Add::add(a, b)

`std::ops::Add` 特征的核心定义如下：
```
```rust
/// std::ops::Add，用于标记支持\`+\`的类型

pub trait Add<RHS> {

/// \`+\` 运算后的结果类型

type Output;

/// 实现\`+\`运算符的方法

fn add(self, rhs: RHS) -> Self::Output;

}
```
要点：

`Add` 是一个带类型参数 `RHS` 的trait， `RHS` 表示右操作数的类型。因为带类型参数，所以 `Add<i32>` 、 `Add<f64>` 、 `Add<String>` 等都是不同的特征。因此，一个类型理论上可以实现多份 `Add` ，分别对应不同的 `RHS` ，并且每份实现都可以有自己的 `Output` 。

`type Output` 是 **关联类型** ：表示加法结果的类型。

`fn add(self, rhs: RHS)` 中 `self` 默认按值传入（会“消耗”左操作数）；对 `Copy` 类型（如整数）而言这通常没问题。

Trait的孤儿规则要求 `impl Trait for Type` 中 `Trait` 和 `Type` 至少有一个是当前crate定义的本地项。对于带泛型参数的Trait，格式为 `impl Trait<T> for Type` 。 `Trait` 和 `Type` 可以是外部crate定义的，但此时trait的泛型参数 `T` 必须是本地类型。

例如，定义本地类型WindowsSize，则可以为外部类型 `f64` 实现外部trait `Add<WindowsSize>` 以及为本地 `WindowSize` 实现外部trait `Add<f64>` 。
```rust
use std::ops::Add;

struct WindowSize(pub f64);

impl Add<WindowSize> for f64 {

type Output = f64;

fn add(self, rhs: WindowSize) -> f64 {

self + rhs.0

}

}

impl Add<f64> for WindowSize {

type Output = f64;

fn add(self, rhs: f64) -> f64 {

self.0 + rhs

}

}
```
前面的 `Add` 特征还不完善，真正的 `Add` 特征是这样的：
```rust
pub trait Add<RHS=Self> {

}
```
语法 `RHS=Self` 表示 `RHS` 默认是 `Self` ，例如写 `impl Add for Complex` ，而不指定 `Add` 的参数类型，则表示 `impl Add<Complex> for Complex` 。在类型限界里，如果写 `where T: Add` ，则表示 `where T: Add<T>` 。
## 3、关联常量

与结构体和枚举一样， **特征也可以定义关联常量** 。可以使用与结构体或枚举相同的语法声明带有关联常量的特征。

示例：

```rust
trait Greet {

const GREETING: &'static str = "Hello";

fn greet(&self) -> String;

}
```
然而，关联常量在特征中具有特殊功能。与关联类型和函数一样，可以只声明它们但不为其定义值：
```rust
trait Float {

const ZERO: Self;

const ONE: Self;

}
```
之后，特征的实现者可以定义这些值：
```rust
impl Float for f32 {

const ZERO: f32 = 0.0;

const ONE: f32 = 1.0;

}

impl Float for f64 {

const ZERO: f64 = 0.0;

const ONE: f64 = 1.0;

}
```
可以编写使用这些值的泛型代码：
```text
fn add\_one<T: Float + Add<Output<T>>(value: T) -> T {

value + T::ONE

}
```
注意， **关联常量不能与特征对象一起使用** ，因为为了在编译期选择正确的值，编译器会依赖相关实现的类型信息。

即使是没有任何行为的简单特型（如 Float），也可以提供有关类型的足够信息，再结合一些运算符，以实现像斐波那契数列这样常见的数学函数：
```rust
fn fib<T: Float + Add<Output=T>>(n: usize) -> T {

match n {

0 => T::ZERO,

1 => T::ONE,

n => fib::<T>(n-1) + fib::<T>(n-2)

}

}
```
