# 一、枚举的定义与使用

枚举（enum）也是一种自定义类型，用于表示一组可能的状态。每种状态称为枚举的 **变体或构造器** 。相比于C语言中只能是整型常量的枚举，Rust枚举更加强大：每个变体还可以携带不同类型的数据。因此，可以把Rust枚举分为 **C风格枚举和带数据枚举** 。

Rust使用 **enum关键字** 定义枚举类型，使用 **枚举名::变体名** 创建枚举值。

## 1、C风格枚举

Rust的C风格枚举很直接，例如标准库里的 **Ordering枚举** ：

```rust
enum Ordering {

Less = -1,

Equal = 0,

Greater = 1,

}
```
这声明了一个具有3个可能值的Ordering类型值： **Ordering::Less、Ordering::Equal、Ordering::Greater** 。

使用示例：
```rust
use std::cmp::Ordering;

fn compare(n: i32, m: i32) -> Ordering {

if n < m {

Ordering::Less

} else if n > m {

Ordering::Greater

} else {

Ordering::Equal

}

}
```
这里需要使用 **枚举名::变体名** 的方式导入变体。如果想直接用变体名，可以将枚举的各个变体一起导入：
```rust
use std::cmp::Ordering::{self, *};

fn compare(n: i32, m: i32) -> Ordering {

if n < m {

Less

} else if n > m {

Greater

} else {

Equal

}

}
```
但这样写意思不明确，通常认为不导入变体可读性更好，除非是常用的几种枚举变体（后面介绍的Option和Result的变体会在预导入模块中，可直接使用变体名）

可以自己定义一个枚举，例如下面的Color枚举：
```rust
#[derive(Debug)]

enum Color {

Red,

Green,

Blue,

}

fn main() {

let c1 = Color::Red;

let c2 = c1;

// println!("{:?}", c1); // 错误，发生移动，c1所有权转移到c2

println!("{:?}", c2);

}

🔊
```
**结构体或枚举默认是非Copy类型，因此会发生所有权的转移。如果结构体的所有字段或枚举的所有变体都是Copy类型，可以通过属性派生宏#[derive(Copy, Clone)]将结构体或枚举变为Copy类型** 。

对于C风格枚举，如果不指定具体的值，则默认从0开始依次分配数值，因此这里的Red为0，Green为1，Blue为2。

有些情况下，需要自行分配整数值：
```rust
enum HttpStatus {

Ok = 200,

NotModified = 304,

NotFound = 404,

…

}
```
**和结构体类似，枚举也可以有方法** 。

示例：
```rust
#[derive(Debug, Copy, Clone)]

enum Color {

Red,

Green,

Blue,

}

impl Color {

fn as_str(&self) -> &'static str {

match self {

Color::Red => "red",

Color::Green => "green",

Color::Blue => "blue"

}

}

}

fn main() {

let c = Color::Red;

println!("{:?} -> {}", c, c.as_str());

}
```
这里用到的 **match模式匹配** 将在下一节中介绍。match self自动对self进行了解引用，因为self是&Color类型。
## 2、带数据枚举

Rust枚举的变体除了可以是单元型变体（无数据），还可以是 **结构体型变体和元组型变体** ，它们可以携带其他类型数据。

示例：

```rust
#[derive(Debug)]

enum Message {

Quit, // 单元型变体，无数据

Move { x: i32, y: i32 }, // 结构体型变体

Write(String), // 元组型变体

Color(i32, i32, i32) // 元组型变体

}

fn main() {

let m1 = Message::Move{x: 10, y: 20};

let m2 = Message::Write("Hello".to_string());

let m3 = Message::Color(255, 255, 255);

println!("{:?}", m1);

println!("{:?}", m2);

println!("{:?}", m3);

}
```

此外，枚举的变体携带的数据还可以是另一个枚举类型。

示例：
```rust
#[derive(Debug)]

enum IpAddrKind {

V4,

V6,

}

#[derive(Debug)]

enum NetworkEvent {

Connect(IpAddrKind, String),

Disconnect(IpAddrKind),

Timeout,

}

fn main() {

let event1 = NetworkEvent::Connect(IpAddrKind::V4, String::from("127.0.0.1"));

let event2 = NetworkEvent::Timeout;

println!("{:?}", event1);

println!("{:?}", event2);

}
```
# 二、Option和Result枚举

Rust标准库提供了两种十分常用的枚举： **Option<T>和Result<T, E>** ，它们在Rust代码中几乎无处不在。 **由于太常用，使用其变体时通常不需要加枚举类型前缀（例如直接写Some(...)、None、Ok(...)、Err(...)）** 。

注意：这里的<T>以及<T, E>是泛型参数，关于泛型将在后续介绍。

## 1、Option<T>

Option<T>的定义大致如下：

```rust
enum Option<T> {

Some(T),

None,

}

Rust 中没有NULL的概念，而是用Option来表示“有值/无值”：

Some(v)：有值，值的类型为T

None：无值
```
示例：
```rust
fn main() {

let x = Some(1);

let y: Option<i32> = None;

println!("{:?}", x);

println!("{:?}", y);

}
```
**注意：如果使用None，通常需要显式指定类型，否则编译器可能无法推断T的类型** 。
## 2、Result<T, E>

Result<T, E> 的定义大致如下：

```rust
enum Result<T, E> {

Ok(T),

Err(E),

}

Result<T, E> 表示结果是成功还是失败，常作为函数返回类型用于错误处理：

Ok(v)：成功，v的类型为T，T是期望的正确类型

Err(e)：失败，e的类型为E
```
# 三、内存中的枚举

## 1、C风格枚举内存表示法

在内存中，C风格枚举的各个值会存储为 **整数** 。默认情况下，Rust会使用 **可以容纳它们的最小内置整数类型** 来存储C风格枚举，最适合的就是单字节：

```rust
enum Color {

Red,

Green,

Blue,

}

fn main() {

println!("size of Color: {}", size_of::<Color>()); // 1

}
```
因为Blue最大值为2，单个字节就能表示。

可以通过 **#[repr(...)]** 来改变Rust对内存中表示法的默认选择：
```rust
#[repr(u32)]

enum Color {

Red,

Green,

Blue,

}

fn main() {

println!("size of Color: {}", size_of::<Color>()); // 4

}
```
**可以将C风格枚举值转换为整数，但无法将整数直接转换为C风格枚举值：**
```rust
enum Color {

Red,

Green,

Blue,

}

fn main() {

let x = Color::Red as u32; // C风格枚举值转换为整数

println!("{}", x);

}
```
因为枚举的变体值是限定好的，给定的整数值不一定在变体值的范围内。

不过，可以自行编写先检查再转换的逻辑（返回Option<EnumType>），例如：
```rust
#[derive(Debug, PartialEq, Eq)]

enum Color {

Red,

Green,

Blue,

}

fn color_from_u8(x: u8) -> Option<Color> {

match x {

0 => Some(Color::Red),

1 => Some(Color::Green),

2 => Some(Color::Blue),

_ => None,

}

}

fn main() {

let x = Color::Red as u32; // C风格枚举值转换为整数

println!("{}", x);

assert_eq!(color_from_u8(1), Some(Color::Green));

assert_eq!(color_from_u8(4), None);

}
```
也可以借助 **enum_primitive crate** 提供的宏自动完成转换代码。
## 2、带数据枚举的内存表示法

**在内存中，带有数据的枚举以一个小型整数标签加上足以容纳最大变体的所有字段的内存块的格式进行存储** 。 **标签字段供Rust内部使用，它会区分由哪个变体构造值，进而决定该值应该有哪些字段** 。

**不过，为了给将来的优化留下余地，Rust并没有对枚举的内存布局做出任何承诺** 。例如，有些泛型结构体可以在不需要标签的情况下存储。

# 四、用枚举表示富数据结构

枚举用于 **快速实现树形数据结构** 也很有用。当程序需要处理任意JSON数据时，一个很自然的做法就是使用Rust枚举把JSON的6种类型值直接建模出来。

示例：

```rust
use std::collections::HashMap;

enum Json {

Null, // 空值

Boolean(bool), // 布尔

Number(f64), // 数字

String(String), // 字符串

Array(Vec<Json>), // 数组

Object(Box<HashMap<String, Json>>) // 对象

}
```
说明：

枚举非常适合用来表达JSON，因为JSON的值类型集合是有限且封闭的（只有6种），这正好对应enum的强项：

每个变体就是一种JSON值类型

变体还可以携带该类型所需要的数据

枚举适合用来表达 **树形结构（递归定义）** ，关键在于：

Array(Vec<Json>)：数组的每个元素仍是Json，可以无限嵌套

Object(Box<HashMap<String, Json>>)：对象的值（value）仍是Json，同样可以无限嵌套

Object外面加Box是为了让Json枚举本身更紧凑，移动更便宜。因为String和Vec<T>容器类型是胖指针结构，64-bit平台上是3个机器字，而HashMap本体通常比String、Vec<T>更大，如果把HashMap直接塞进结构体变体，则枚举的最大变体将导致Json类型本身很大。
