# 一、字符串字面量

## 1、普通字符串字面量

Rust中的字符串字面量使用引号`" "`包起来。

示例：

```rust
let s = "hello";  // &str类型
```

字符串字面量可以包含转义字符。常见转义写法包括：

```rust
let s = "\"Ouch\"\n";
```

这里：

- `\"` 表示字符串中的双引号。

- `\n` 表示换行符。

**字符串字面量可以直接写成多行**。多行字符串的换行会真实保留下来，第二行开头的空格也会保留。

示例：

```rust
fn main() {
    let s = "Hello, world
    你好，世界
Rust";
    println!("{}", s);
}
```

结果：

![[Image 46.png|300]]

**如果一行的末尾写`\`，Rust会忽略这个反斜杠后面的换行，以及下一行开头连续的空白**。这样可以把很长的字符串拆成多行书写，但运行时仍然是一行。

```rust
fn main() {  
    let s = "It was a beautiful day in March, \  
    there were four of us";  
    println!("{}", s);  
}
```

结果：

![[Image 47.png|400]]

输出结果中，`March,` 和 `there` 之间不会因为源码换行而多出一个换行符。

## 2、原始字符串字面量

有些场景需要写很多反斜杠，例如Windows路径、正则表达式等。如果使用普通字符串，需要频繁转义，代码可读性较差。

例如：

```rust
fn main() {
    let path = "C:\\foo\\bar";

    println!("{}", path);
}
```

结果：

![[Image 48.png|400]]

为了减少转义，Rust 提供了 **原始字符串字面量**，写作 `r"..."`。

原始字符串中，反斜杠、换行和空格都会原样保留，不会识别普通字符串中的转义序列。

示例：

```rust
fn main() {
    let path = r"C:\foo\bar";

    println!("{}", path);
}
```

原始字符串不会把 `\n` 当作换行符处理：

```rust
fn main() {
    let s = r"hello\nworld";

    println!("{}", s);
}
```

输出结果是：

```text
hello\nworld
```

但此时产生了一个问题：原始字符串里无法使用`\"`来转义双引号了。如果原始字符串中需要包含双引号，可以在边界加 `#`。


```rust
fn main() {
    let s = r#"He said "hi""#;

    println!("{}", s);
}
```

结果：

![[Image 49.png|400]]

如果内容里本身包含 `"#` 这种会和 `r#"..."#` 结束边界冲突的片段，就继续增加 `#`：

```rust
let s = r##"He said "# hello""##;
```

此时结束边界变成`"##`，所以内容里的 `"#` 不会结束字符串。

## 3、字节串字面量

**字节串字面量是指带有`b`前缀的字符串字面量**。

示例：

```rust
fn main() {
    let s = b"Hello";
    assert_eq!(s, &[b'H', b'e', b'l', b'l', b'o']);
}
```

普通字符串字面量和原始字符串字面量的类型是 `&str`（马上介绍）。

字节串字面量的类型是 `&[u8; N]`（即数组的引用），其中 `N` 是字节数量。

例如，`b"Hello"`的类型为`&[u8; 5]`。

**字节串面向的是字节，不是 Unicode 文本。因此字节串中不能直接写非 ASCII 字符**。

下面这种写法是错误的：

```rust
let s = b"你好";
```

字节串也可以使用前面所述的字符串字面量的语法：跨多行、使用转义序列、通过反斜杠连接行等。

字节串也有原始字节串形式，写作`br"..."`或`br#"..."#`。

示例：

```rust
fn main() {
    let path = br"C:\foo\bar";

    println!("{:?}", path);
}
```


# 二、内存中的字符串

## 1、Rust字符串使用UTF-8存储

Rust中的`String`和`&str`都表示`UTF-8`文本。`UTF-8`是一种**可变长度编码**，不同字符占用的字节数可能不同：

- ASCII字符占用1字节
- 中文、日文、韩文等字符串通常占多个字节
- 一些特殊符号和emoji也可能占多个字节

示例：

```rust
fn main() {
    assert_eq!("hello".len(), 5);

    assert_eq!("你好".len(), 6);
}
```

`"hello"` 有 5 个 ASCII 字符，占 5 个字节。`"你好"` 有 2 个汉字，但在 UTF-8 中占 6 个字节。

> 字符串的 len() 返回的是字节数，不是字符数。

如果要统计字符数量，可以使用`chars().count()`。

示例：

```rust
fn main() {
    assert_eq!("你好".chars().count(), 2);

    assert_eq!("ಠ_ಠ".len(), 7);

    assert_eq!("ಠ_ಠ".chars().count(), 3);
}
```


## 2、str、&str和String

Rust中常见的字符串相关类型有三个：`str`、`&str`和`String`。

`str`是字符串切片类型，表示一段`UTF-8`文本。由于`str`的长度不固定，所以通常不会直接使用 `str` 本身，而是通过引用使用它。事实上，可以把`str`视为`[u8]` + `UTF-8`语义。

`&str`是字符串切片引用，表示对一段`UTF-8 文本`的借用。它不拥有字符串数据。

`String`是拥有所有权的字符串类型，在堆上分配并管理字符串数据。

示例：

```rust
fn main() {
    let s1 = "hello";

    let s2 = String::from("hello");

    let s3 = &s2[1..];
}
```

这里：

- `s1` 的类型是 `&str`，它指向字符串字面量，字符串字面量被编译进程序的二进制文件中，存放在一块**只读的静态内存区域**里。
- `s2` 的类型是 `String`，它在堆上拥有自己的字符串数据。
- `s3` 的类型是 `&str`，它借用了 `s2` 中的一部分字符串内容。


## 3、String的内存结构

`String`是拥有所有权的字符串类型。它可以理解为一个保证内容是合法`UTF-8`的`Vec<u8>`。

`String` 本身在栈上保存三个字段：

- 指向堆缓冲区的指针 `ptr`

- 当前长度 `len`

- 当前容量 `cap`

实际字符串字节存放在堆上。

示例：

```rust
fn main() {
    let s = String::from("hello");
}
```

当 `s` 离开作用域时，`String` 会自动释放它持有的堆内存。

这就是 `String` 和 `&str` 的关键区别：

> `String` 拥有字符串数据，`&str` 只是借用字符串数据。

## 4、&str 的内存结构

`&str`是字符串切片引用。它本质是一个胖指针，包含两部分信息：

- 指向字符串数据起始位置的指针
- 字符串切片的字节长度

示例：

```rust
fn main() {
    let noodles = String::from("noodles");

    let oodles = &noodles[1..];

    println!("{}", oodles);
}
```

内存关系可以理解为：

![[Pasted image 20260513180152.png|300]]


这里 `oodles` 是一个 `&str`，它没有复制字符串内容，也没有取得 `noodles` 的所有权。它只是借用了 `noodles` 中从下标 `1` 开始的那一段 UTF-8 文本。

`&str` 的数据指针可以指向不同位置：

- 可以指向`String`的堆内存
- 可以指向字符串字面量所在的只读内存

示例：

```rust
fn main() {
    let s1 = String::from("hello");

    let a = &s1[..];

    let b = "world";

    println!("{}", a);

    println!("{}", b);
}
```

这里 `a` 是对 `String` 内容的借用，`b` 是指向字符串字面量的 `&str`。

## 5、字符串字面量的类型

字符串字面量的类型是 `&'static str`。

例如：

```rust
fn main() {
    let s = "hello";

    println!("{}", s);
}
```

这里的变量`s`的类型是`&str`，更完整地说是`&'static str`。

字符串字面量 `"hello"` 并不是在程序运行时临时创建出来的 `String`，也不会在堆上重新分配一块字符串缓冲区。

它的内容会在编译时被**硬编码进最终生成的二进制程序中**。程序运行时，`s` 只是一个字符串切片，指向这段已经存在的静态字符串数据。

不能通过`&str`修改它指向的字符串内容，因为它是不可变切片引用。

示例：
```rust
let  s = "hello";
s[0] = 'c'; // 错误，因为s是共享引用
```


**当然，&mut str类型确实也存在** ，只是它的可用操作范围很窄：因为拿到的是一段UTF-8文本切片的独占可变借用，可以就地修改这段内存，但前提是不能修改这段切片的字节长度。 **然而，很多操作都会导致UTF-8编码字节数变化** 。

在标准库里，str上非常典型也是最常用的一类就地可变操作就是ASCII大小写转换： **make_ascii_uppercase和make_ascii_lowercase** 。它们只对ASCII字母生效，且每个字符仍然是1字节，所以不会更改总字节数，因而可以安全地就地修改。

示例：
```rust
let mut s = String::from("Hello, Rust! 123 abc XYZ 你好");

s.as_mut_str().make_ascii_uppercase(); // s.as_mut_str返回&mut str

println!("{}", s);
```

# 三、String的创建方式

通过String::from()方式创建

```rust
let s = String::from("hello");

.to_string()方法将&str转换为String
```
```rust
let s = "hello".to_string();
```
**.to_owned()** 方法也可以做类似的事情。

format!宏的工作方式和println!宏类似，但它会返回一个新的String，而不是将文本写入标准输出，并且不会在末尾自动增加换行符。
```rust
let s = format!("I am {} years old", 20);

concat 和 join 都能把“很多段字符串”拼成一个新的 String，区别在于：concat() 直接无分隔地拼接；join(sep) 会在每两段之间插入分隔符 sep。
```
```rust
let fruit = vec!["apple", "orange", "banana"];

assert_eq!(fruit.concat(), "appleorangebanana");

assert_eq!(fruit.join(", "), "apple, orange, banana");
```
# 三、其他类似字符串的类型

**Rust的String和&str始终保证内容是合法的UTF-8，因此非常适合表述真正的Unicode文本** 。但在与操作系统或外部库互操作时，程序常常会遇到并不保证是UTF-8、甚至根本不是Unicode的字符串：典型的例子就是文件名、环境变量名、命令行参数、以及来自C接口的以\\0结尾的字符串。为了避免把这些数据强行塞进String导致失败会丢失信息，Rust提供了一组类似字符串的专用类型，按场景选用即可。

对于Unicode文本，坚持使用String和&str，它们提供最丰富的字符串API且保证文本有效

当处理 **文件系统路径** 时（包括文件名），优先使用 **std::path::PathBuf和&Path** 。路径并不等同于文本，在不同平台上路径的底层表示也不同。Path和PathBuf提供了拼接、遍历组建、扩展名等与路径语义相关的操作。当需要显示给用户时再按需转换成字符串（.display()、.to_string_lossy()）

当处理的就是 **二进制数据的** 内容（例如网络协议、压缩数据、哈希、任意编码文本时），使用 **Vec<u8>或&[u8]** ：这类数据不应该当做字符串。如果确切知道它是UTF-8，再使用std::str::from_utf8进行验证性转换。

当需要 **保存/传递操作系统原生形式的字符串** （尤其是环境变量名、命令行参数，或从OS API拿到的文本），使用 **std::ffi::OsString和&OsStr** ：它们能无损表示平台原生字符串。

当与 **C语言库互操作，需要使用\\0结尾的字符串时** ，使用 **std::ffi::CString和&CStr** ：CString表示拥有所有权的C字符串，保证末尾有\\0且内部不允许出现\\0。&CStr是对现有C字符串的借用视图。
