# 一、字符串字面量

## 1、普通字符串

字符串字面量需要用 **双引号""** 包起来，反斜杠转义规则和char类似，例如在字符串里需要放一个双引号就写\\"，要换行就写\\n：

```rust
let s = "\\"Ouch!\\"\\n";
```
和char不同的是，字符串里出现单引号'不需要转义，只有双引号才需要转义。

**字符串字面量可以直接写成多行** ，写出来的多行会真实地转换成换行符，第二行开头的空格也同样会保留并输出。

示例：
```rust
let s = "Hello, world

你好，世界

Rust";

println!("{}", s);
```
结果：

![[Image 46.png]]

**如果一行的末尾加上\\，Rust会把整个反斜杠后面的换行以及下一行开头连续的空白一起丢掉** ，这允许我们将长字符串拆开写，但运行时仍然是一行：
```rust
let s = "It was a beautiful day in March\\

, there were four of us";

println!("{}", s);
```

## 2、原始字符串

有些场景（比如Windows路径、正则表达式等）需要写很多反斜杠，如果用普通字符串写会比较麻烦，可读性也差：

```rust
let path = "C:\\\\foo\\\\bar";

println!("{}", path);
```
结果：

![[Image 48.png]]

如果要打印\\，需要\\\\进行转义。

为了简便起见，Rust提供了 **原始字符串** ，用 **r"..."** 表示。 **原始字符串里的反斜杠、换行和空格都会原样保留，不识别任何转义序列** ：
```rust
let path = r"C:\\foo\\bar";

println!("{}", path);

但此时产生了一个新的问题：原始字符串里无法使用\\"来表示双引号，因为它不识别转义字符。想把双引号也放进原始字符串，可以在边界加#：
```
```rust
let s = r#"He said "hi""#;

println!("{}", s);
```
结果：

![[Image 49.png]]

**原始字符串并不要求#的具体数量，只需要前后数量保持一致，且不会和内容相混淆** 。
## 3、字节串

带有 **b前缀** 的字符串字面量都是 **字节串** ，字节串的类型不是&str，而是&[u8; N]。字节串不能包含任何Unicode字符，它们只能使用ASCII和\\xHH转义序列。

示例：

```rust
let s = b"Hello";

assert_eq!(s, &[b'H', b'e', b'l', b'l', b'o']);
```
关于&str下面会介绍，这里需要知道的是s的类型是&[u8; 5]，也就是u8数组的引用。

字节串也可以使用前面所述的字符串字面量的语法：跨多行、使用转义序列、通过反斜杠连接行等。对于原始字节串，以 **br"** 开头。
# 二、内存中的字符串

## 1、str、&str和String

Rust中的字符串是Unicode字符序列，但是它们并没有以char数组的形式存储在内存中，而是使用了 **UTF-8（一种可变宽度编码）的形式** 。 **字符串中的每个ASCII字符都会存储在单字节中，而其他字符会占用多字节** 。

示例：

```rust
let noodles = "noodles".to_string(); // String

let oodles = &noodles[1..]; // &str

let poodles = "ಠ_ಠ"; // &str
```
内存示意图如下：

![[Image 50.png]]

noodles是 **String类型** 。和Vec一样，每个String在堆上分配了自己的缓冲区，不会与其他String共享。当String变量超出作用域，缓冲区将自动释放，除非这个String已经被移动。可以把String视为Vec<u8>，但它保证格式良好的UTF-8。noodles拥有8字节的缓冲区，其中7字节正在使用中，存储了"noodles"。

oddles是对noodles的切片操作，它的类型是&str。和其他切片引用一样，&str也是一个胖指针，包含实际数据的地址以及长度。可以将&str视为&[u8]，但它保证了格式良好的UTF-8。这里的&str是对noodles（String）的只读借用。

poodles的类型也是&str，它通过字符串字面量直接赋值。 **字符串字面量通常与程序的机器码一起存储在只读内存区** 。这里的poodles指向一块7字节的内存，该内存在程序开始执行时就已创建并一直存续到程序退出。

这里可以看到的是，&str的数据指针可以指向堆区，也可以指向预分配的只读内存。

**一些注意事项：**

**String或&str的len()方法返回的是字节数而不是字符数**
```rust
assert_eq!("ಠ_ಠ".len(), 7);
```
如果需要返回字符数，需要使用chars()：
```rust
assert_eq!("ಠ_ಠ".chars().count(), 3);
```
**不能通过&str修改它指向的字符串内容**

准确来说，可以让一个&str变量重新绑定到别的字符串切片（只是换一个引用），但不能用它去修改它借用的那段数据本身。
```rust
let mut s = "hello";

s = "world"; // 允许

s[0] = 'c'; // 错误，因为s是共享引用
```
**当然，&mut str类型确实也存在** ，只是它的可用操作范围很窄：因为拿到的是一段UTF-8文本切片的独占可变借用，可以就地修改这段内存，但前提是不能修改这段切片的字节长度。 **然而，很多操作都会导致UTF-8编码字节数变化** 。

在标准库里，str上非常典型也是最常用的一类就地可变操作就是ASCII大小写转换： **make_ascii_uppercase和make_ascii_lowercase** 。它们只对ASCII字母生效，且每个字符仍然是1字节，所以不会更改总字节数，因而可以安全地就地修改。

示例：
```rust
let mut s = String::from("Hello, Rust! 123 abc XYZ 你好");

s.as_mut_str().make_ascii_uppercase(); // s.as_mut_str返回&mut str

println!("{}", s);
```

## 2、创建String的方法

通过String::from()方式创建

```rust
let s = String::from("hello");

.to_string()方法将&str转换为String
```
```rust
let s = "hello".to_string();
```
**.to_owned()** 方法也可以做类似的事情。

format!宏的工作方式和println!宏类似，但它会返回一个新的String，而不是将文本写入标准输出，并且不会在末尾自动增加换行符。
```rust
let s = format!("I am {} years old", 20);

concat 和 join 都能把“很多段字符串”拼成一个新的 String，区别在于：concat() 直接无分隔地拼接；join(sep) 会在每两段之间插入分隔符 sep。
```
```rust
let fruit = vec!["apple", "orange", "banana"];

assert_eq!(fruit.concat(), "appleorangebanana");

assert_eq!(fruit.join(", "), "apple, orange, banana");
```
# 三、其他类似字符串的类型

**Rust的String和&str始终保证内容是合法的UTF-8，因此非常适合表述真正的Unicode文本** 。但在与操作系统或外部库互操作时，程序常常会遇到并不保证是UTF-8、甚至根本不是Unicode的字符串：典型的例子就是文件名、环境变量名、命令行参数、以及来自C接口的以\\0结尾的字符串。为了避免把这些数据强行塞进String导致失败会丢失信息，Rust提供了一组类似字符串的专用类型，按场景选用即可。

对于Unicode文本，坚持使用String和&str，它们提供最丰富的字符串API且保证文本有效

当处理 **文件系统路径** 时（包括文件名），优先使用 **std::path::PathBuf和&Path** 。路径并不等同于文本，在不同平台上路径的底层表示也不同。Path和PathBuf提供了拼接、遍历组建、扩展名等与路径语义相关的操作。当需要显示给用户时再按需转换成字符串（.display()、.to_string_lossy()）

当处理的就是 **二进制数据的** 内容（例如网络协议、压缩数据、哈希、任意编码文本时），使用 **Vec<u8>或&[u8]** ：这类数据不应该当做字符串。如果确切知道它是UTF-8，再使用std::str::from_utf8进行验证性转换。

当需要 **保存/传递操作系统原生形式的字符串** （尤其是环境变量名、命令行参数，或从OS API拿到的文本），使用 **std::ffi::OsString和&OsStr** ：它们能无损表示平台原生字符串。

当与 **C语言库互操作，需要使用\\0结尾的字符串时** ，使用 **std::ffi::CString和&CStr** ：CString表示拥有所有权的C字符串，保证末尾有\\0且内部不允许出现\\0。&CStr是对现有C字符串的借用视图。
