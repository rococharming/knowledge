# 一、字符串字面量

## 1、普通字符串字面量

Rust中的字符串字面量使用引号`" "`包起来。

示例：

```rust
let s = "hello";  // &str类型
```

字符串字面量可以包含转义字符。常见转义写法包括：

```rust
let s = "\"Ouch\"\n";
```

这里：

- `\"` 表示字符串中的双引号。

- `\n` 表示换行符。

**字符串字面量可以直接写成多行**。多行字符串的换行会真实保留下来，第二行开头的空格也会保留。

示例：

```rust
fn main() {
    let s = "Hello, world
    你好，世界
Rust";
    println!("{}", s);
}
```

结果：

![[Image 46.png|300]]

**如果一行的末尾写`\`，Rust会忽略这个反斜杠后面的换行，以及下一行开头连续的空白**。这样可以把很长的字符串拆成多行书写，但运行时仍然是一行。

```rust
fn main() {  
    let s = "It was a beautiful day in March, \  
    there were four of us";  
    println!("{}", s);  
}
```

结果：

![[Image 47.png|400]]

输出结果中，`March,` 和 `there` 之间不会因为源码换行而多出一个换行符。

## 2、原始字符串字面量

有些场景需要写很多反斜杠，例如Windows路径、正则表达式等。如果使用普通字符串，需要频繁转义，代码可读性较差。

例如：

```rust
fn main() {
    let path = "C:\\foo\\bar";

    println!("{}", path);
}
```

结果：

![[Image 48.png|400]]

为了减少转义，Rust 提供了 **原始字符串字面量**，写作 `r"..."`。

原始字符串中，反斜杠、换行和空格都会原样保留，不会识别普通字符串中的转义序列。

示例：

```rust
fn main() {
    let path = r"C:\foo\bar";

    println!("{}", path);
}
```

原始字符串不会把 `\n` 当作换行符处理：

```rust
fn main() {
    let s = r"hello\nworld";

    println!("{}", s);
}
```

输出结果是：

```text
hello\nworld
```

但此时产生了一个问题：原始字符串里无法使用`\"`来转义双引号了。如果原始字符串中需要包含双引号，可以在边界加 `#`。


```rust
fn main() {
    let s = r#"He said "hi""#;

    println!("{}", s);
}
```

结果：

![[Image 49.png|400]]

如果内容里本身包含 `"#` 这种会和 `r#"..."#` 结束边界冲突的片段，就继续增加 `#`：

```rust
let s = r##"He said "# hello""##;
```

此时结束边界变成`"##`，所以内容里的 `"#` 不会结束字符串。

## 3、字节串字面量

**字节串字面量是指带有`b`前缀的字符串字面量**。

示例：

```rust
fn main() {
    let s = b"Hello";
    assert_eq!(s, &[b'H', b'e', b'l', b'l', b'o']);
}
```

普通字符串字面量和原始字符串字面量的类型是 `&str`（马上介绍）。

字节串字面量的类型是 `&[u8; N]`（即数组的引用），其中 `N` 是字节数量。

例如，`b"Hello"`的类型为`&[u8; 5]`。

**字节串面向的是字节，不是 Unicode 文本。因此字节串中不能直接写非 ASCII 字符**。

下面这种写法是错误的：

```rust
let s = b"你好";
```

字节串也可以使用前面所述的字符串字面量的语法：跨多行、使用转义序列、通过反斜杠连接行等。

字节串也有原始字节串形式，写作`br"..."`或`br#"..."#`。

示例：

```rust
fn main() {
    let path = br"C:\foo\bar";

    println!("{:?}", path);
}
```


# 二、内存中的字符串

## 1、Rust字符串使用UTF-8存储

Rust中的`String`和`&str`都表示`UTF-8`文本。`UTF-8`是一种**可变长度编码**，不同字符占用的字节数可能不同：

- ASCII字符占用1字节
- 中文、日文、韩文等字符串通常占多个字节
- 一些特殊符号和emoji也可能占多个字节

示例：

```rust
fn main() {
    assert_eq!("hello".len(), 5);

    assert_eq!("你好".len(), 6);
}
```

`"hello"` 有 5 个 ASCII 字符，占 5 个字节。`"你好"` 有 2 个汉字，但在 UTF-8 中占 6 个字节。

> 字符串的 len() 返回的是字节数，不是字符数。

如果要统计字符数量，可以使用`chars().count()`。

示例：

```rust
fn main() {
    assert_eq!("你好".chars().count(), 2);

    assert_eq!("ಠ_ಠ".len(), 7);

    assert_eq!("ಠ_ಠ".chars().count(), 3);
}
```


## 2、str、&str和String

Rust中常见的字符串相关类型有三个：`str`、`&str`和`String`。

`str`是字符串切片类型，表示一段`UTF-8`文本。由于`str`的长度不固定，所以通常不会直接使用 `str` 本身，而是通过引用使用它。事实上，可以把`str`视为`[u8]` + `UTF-8`语义。

`&str`是字符串切片引用，表示对一段`UTF-8 文本`的借用。它不拥有字符串数据。

`String`是拥有所有权的字符串类型，在堆上分配并管理字符串数据。

示例：

```rust
fn main() {
    let s1 = "hello";

    let s2 = String::from("hello");

    let s3 = &s2[1..];
}
```

这里：

- `s1` 的类型是 `&str`，它指向字符串字面量，字符串字面量被编译进程序的二进制文件中，存放在一块**只读的静态内存区域**里。
- `s2` 的类型是 `String`，它在堆上拥有自己的字符串数据。
- `s3` 的类型是 `&str`，它借用了 `s2` 中的一部分字符串内容。


## 3、String的内存结构

`String`是拥有所有权的字符串类型。它可以理解为一个保证内容是合法`UTF-8`的`Vec<u8>`。

`String` 本身在栈上保存三个字段：

- 指向堆缓冲区的指针 `ptr`

- 当前长度 `len`

- 当前容量 `cap`

实际字符串字节存放在堆上。

示例：

```rust
fn main() {
    let s = String::from("hello");
}
```

当 `s` 离开作用域时，`String` 会自动释放它持有的堆内存。

这就是 `String` 和 `&str` 的关键区别：

> `String` 拥有字符串数据，`&str` 只是借用字符串数据。

## 4、&str 的内存结构

`&str`是字符串切片引用。它本质是一个胖指针，包含两部分信息：

- 指向字符串数据起始位置的指针
- 字符串切片的字节长度

示例：

```rust
fn main() {
    let noodles = String::from("noodles");

    let oodles = &noodles[1..];

    println!("{}", oodles);
}
```

内存关系可以理解为：

![[Pasted image 20260513180152.png|300]]


这里 `oodles` 是一个 `&str`，它没有复制字符串内容，也没有取得 `noodles` 的所有权。它只是借用了 `noodles` 中从下标 `1` 开始的那一段 UTF-8 文本。

`&str` 的数据指针可以指向不同位置：

- 可以指向`String`的堆内存
- 可以指向字符串字面量所在的只读内存

示例：

```rust
fn main() {
    let s1 = String::from("hello");

    let a = &s1[..];

    let b = "world";

    println!("{}", a);

    println!("{}", b);
}
```

这里 `a` 是对 `String` 内容的借用，`b` 是指向字符串字面量的 `&str`。

## 5、字符串字面量的类型

字符串字面量的类型是 `&'static str`。

例如：

```rust
fn main() {
    let s = "hello";

    println!("{}", s);
}
```

这里的变量`s`的类型是`&str`，更完整地说是`&'static str`。

字符串字面量`"hello"`的内容会在编译时被**硬编码进最终生成的二进制程序中**。程序运行时，`s` 只是一个字符串切片，指向这段已经存在的**静态字符串数据**。


# 三、String的创建方式

## 1、通过String::from

可以使用`String::from`从字符串字面量创建`String`。

示例：

```rust
fn main() {
    let s = String::from("hello");

    println!("{}", s);
}
```

这里 `"hello"` 是 `&str`，`String::from("hello")` 会创建一个拥有所有权的 `String`。本质就是在堆上开辟空间，复制字符串字面量的数据到堆中。

## 2、通过to_string

也可以使用 `to_string()` 创建 `String`。

```rust
fn main() {
    let s = "hello".to_string();

    println!("{}", s);
}
```

`to_string()` 常用于把 `&str` 转换成 `String`。

## 3、使用to_owned

`to_owned()` 也可以把 `&str` 转换成 `String`。

示例：

```rust
fn main() {
    let s = "hello".to_owned();

    println!("{}", s);
}
```

对于字符串来说，`to_string()` 和 `to_owned()` 在这里都能得到一个新的 `String`。

## 4、使用 format!

`format!` 的用法和 `println!` 类似，但它不会打印到终端，而是返回一个新的 `String`。

示例：

```rust
fn main() {
    let age = 20;

    let s = format!("I am {} years old", age);

    println!("{}", s);
}
```

> `format!` 不会在末尾自动添加换行符。

## 5、使用concat和join

`concat()`和`join()`可以把多段字符串拼接成一个新的`String`。区别在于：

- `concat()`会直接拼接，中间不加分割符
- `join(sep)`会在每两段之间插入`sep`

示例：

```rust
fn main() {
    let fruit = vec!["apple", "orange", "banana"];

    assert_eq!(fruit.concat(), "appleorangebanana");

    assert_eq!(fruit.join(", "), "apple, orange, banana");
}
```


# 四、字符串的修改

## 1、String可以修改

`String` 拥有字符串数据，因此可以修改。

示例：

```rust
fn main() {
    let mut s = String::from("hello");

    s.push_str(", world");   // 追加

    println!("{}", s);
}
```

这里 `s` 必须声明为 `mut`，因为修改字符串内容需要可变变量。

常见修改方法包括：

- `push(ch)`：追加一个字符
- `push_str(s)`：追加一个字符串切片
- `clear()`：清空字符串

示例：

```rust
fn main() {
    let mut s = String::from("hello");

    s.push('!');          // hello!

    s.push_str(" Rust");  // hello! Rust

    println!("{}", s);

    s.clear();

    println!("{}", s);
}
```


## 2、&str不能修改

`&str` 是共享引用，不能通过它修改字符串内容。

示例：

```rust
let s = "hello";
s.push_str(", world");  // 错误
```

准确来说，可以让一个 `&str` 变量重新绑定到另一个字符串切片，但不能通过它修改原来的字符串内容。

示例：

```rust
fn main() {
    let mut s = "hello";

    s = "world";

    println!("{}", s);
}
```

这里修改的是变量 `s` 的绑定关系，让它从引用 `"hello"` 变成引用 `"world"`。

这不是修改原来的字符串内容。

## 3、&mut str

`&mut str`也存在，但它的使用范围很窄。

原因是：`str` 必须始终保持合法的 UTF-8，而 UTF-8 字符的字节长度不固定。如果随意修改，可能破坏字符串的编码结构，或者改变字符串切片原有长度。

所以标准库中常见的 `&mut str` 就地修改操作主要集中在 ASCII 范围，例如：

- `make_ascii_uppercase`
- `make_ascii_lowercase`

它们只修改 ASCII 字母（大小写转换），而 ASCII 字母每个都是 1 字节，所以不会改变字符串总长度。

示例：

```rust
fn main() {
    let mut s = String::from("Hello, Rust! 123 abc XYZ 你好");

    s.as_mut_str().make_ascii_uppercase();

    println!("{}", s);
}
```

ASCII 字母会变成大写，而中文不受影响。

# 五、字符串索引

## 1、字符串不能按下标访问字符

Rust 中的字符串不能直接使用下标访问字符。

下面这种写法是错误的：

```rust
fn main() {
    let s = String::from("hello");

    let c = s[0]; // 错误
}
```

原因是 Rust 字符串使用 UTF-8 编码，一个字符可能占多个字节。

如果允许 `s[0]`，就容易让人误以为它取到的是第一个字符，但实际上第 0 个字节不一定能单独构成一个字符。

## 2、使用chars遍历字符

如果要按字符处理字符串，可以使用`chars()`。

示例：

```rust
fn main() {
    let s = "你好";

    for ch in s.chars() {
        println!("{}", ch);
    }
}
```

结果：

![[Pasted image 20260514004346.png|200]]

`chars()`遍历的是Unicode标量值，通常可以理解为遍历字符。
## 3、使用bytes遍历字节

如果要查看字符串的底层字节，可以使用 `bytes()`。

示例：

```rust
fn main() {
    let s = "你好";

    for byte in s.bytes() {
        println!("{}", byte);
    }
}
```

结果：

![[Pasted image 20260514004611.png|100]]

`bytes()` 遍历的是 UTF-8 编码后的字节。


## 4、字符串切片必须落在字符边界上

可以对字符串创建切片，但范围必须落在合法的UTF-8字符边界上。

示例：

```rust
fn main() {
    let s = String::from("hello");

    let part = &s[0..2];

    println!("{}", part);  // he
}
```

这里是合法的，因为 ASCII 字符每个都是 1 字节。

再看中文：

```rust
fn main() {
    let s = String::from("你好");

    let part = &s[0..3];

    println!("{}", part);
}
```

这是合法的，因为 `"你"` 在 UTF-8 中占 3 个字节，所以 `0..3` 正好落在字符边界上。

下面这样会 panic：

```rust
fn main() {
    let s = String::from("你好");

    let part = &s[0..1];

    println!("{}", part);
}
```

结果：

![[Pasted image 20260514011227.png|500]]

原因是 `0..1` 只截取了 `"你"` 的一部分字节，没有落在合法字符边界上。


# 六、其他类似字符串的类型

Rust的`String`和`&str`始终保证内容是合法的 UTF-8，因此非常适合表示真正的 Unicode 文本。

但在与操作系统或外部库交互时，程序可能遇到并不保证是UTF-8的数据。例如文件路径、环境变量、命令行参数，以及来自 C 接口的字符串。

这些场景不应该都强行塞进`String`，Rust为它们提供了专门的类型。

## 1、路径使用PathBuf和&Path

处理**文件系统路径**时，优先使用`PathBuf`和`&Path`。

路径不完全等同于普通文本。不同操作系统上，路径的底层表示可能不同。

`Path`和`PathBuf`提供了路径拼接、扩展名、文件名、父目录等路径语义相关操作。

示例：

```rust
use std::path::PathBuf;

fn main() {
    let mut path = PathBuf::from("/tmp");

    path.push("hello.txt");

    println!("{}", path.display());  // /tmp/hello.txt

}
```

`PathBuf` 是拥有所有权的路径类型。

`Path` 是借用形式，通常写作 `&Path`。


## 2、二进制数据使用Vec\<u8\>和&\[u8\]

如果处理的是二进制数据，例如网络协议、压缩数据、哈希值、图片文件内容等，应该使用 `Vec<u8>` 或 `&[u8]`。

这类数据不应该直接当作字符串处理。

如果确实知道某段字节是 UTF-8，可以使用 `std::str::from_utf8` 做验证性转换。

示例：

```rust
fn main() {
    let bytes = vec![104, 101, 108, 108, 111];

    let text = std::str::from_utf8(&bytes).unwrap();

    println!("{}", text);
}
```

> `std::str::from_utf8`返回的是`Result`，这里通过`unwrap`获取Ok变体中的&str。

输出：

```text
hello
```

## 3、操作系统字符串使用OsString和&OsStr

当需要保存或传递操作系统原生形式的字符串时，可以使用 `OsString` 和 `&OsStr`。

它们常用于环境变量、命令行参数、文件名等场景。

`OsString` 是拥有所有权的操作系统字符串。

`OsStr` 是借用形式，通常写作 `&OsStr`。


## 4、C字符串使用CString和&CStr

与 C 语言库交互时，经常会遇到以 `\0` 结尾的字符串。

Rust中对应的类型是`CString`和`&CStr`。

`CString` 表示拥有所有权的 C 字符串，保证末尾有 `\0`，并且内部不能包含额外的 `\0`。

`CStr` 表示对已有 C 字符串的借用视图，通常写作 `&CStr`。

这类类型主要用于 FFI（Foreign Function Interface，外部函数接口），也就是 Rust 和 C 语言接口交互的场景。

## 5、字符串类型的选择

日常开发中，可以按场景选择字符串类型。

- 表示普通 UTF-8 文本，并且需要拥有、修改或长期保存内容时，使用 `String`。
- 只是临时读取一段 UTF-8 文本时，使用 `&str`。
- 处理文件系统路径时，使用 `PathBuf` 或 `&Path`。
- 处理二进制数据时，使用 `Vec<u8>` 或 `&[u8]`。
- 处理操作系统原生字符串时，使用 `OsString` 或 `&OsStr`。
- 处理 C 字符串时，使用 `CString` 或 `&CStr`。
