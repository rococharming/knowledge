# 一、结构体的基本概念

结构体（struct）是一种**自定义数据类型**，用于把多个相关的数据组合到一起。

在 Rust 中，使用 `struct` 关键字定义结构体。

结构体适合表达“某个东西由哪些字段组成”。例如，一个用户可以有名字、年龄；一个点可以有横坐标、纵坐标；一个队列可以有内部缓冲区。

Rust 中常见的结构体有三种：

- 具名字段结构体
- 元组结构体
- 单元结构体

这三种结构体的核心作用都是定义新类型，只是字段的组织形式不同。

# 二、具名字段结构体

## 1、定义具名字段结构体

具名字段结构体的每个字段都有名字。

示例：

```rust
struct User {
    name: String,
    age: u32,
}
```

这里定义了一个 `User` 类型，它有两个字段：

- `name` 字段，类型是 `String`。

- `age` 字段，类型是 `u32`。

需要注意：

- Rust结构体定义本身后面不需要写分号。
- 每个字段都要写明字段名和字段类型
- 字段之间用逗号分隔


## 2、实例化结构体

定义结构体类型之后，可以使用**结构体表达式**创建结构体实例。

结构体表达式以结构体类型名开头，后跟一对花括号{}，其中列出每个字段的名称和值。

示例：

```rust
#[derive(Debug)]
struct User {
	name: String,
	age: u32,
}

fn main() {

	let user = User {
		name: String::from("spf"),
		age: 25,
	};
	
	println!("{:?}", user);
	println!("{:#?}", user);

}
```

这里的 `User { ... }` 就是结构体表达式。创建具名字段结构体时，需要为每个字段提供对应的值。

字段顺序不一定要和定义顺序完全一致，但一般建议保持一致，这样更容易阅读。

> 注意，为了能够打印结构体变量，在定义结构体的上方使用了`#[derive(Debug)]` 。它是派生属性宏，用于自动为结构体实现Debug trait。

对于实现了`Debug` Trait的类型变量，格式化输出时可以采用两种形式：

- `{:?}`：普通Debug输出
- `{:#?}`：带缩进的美化输出

## 3、访问结构体字段

可以使用点号 `.` 访问结构体字段。

示例：

```rust
struct User {
    name: String,
    age: u32,
}

fn main() {
    let user = User {
        name: String::from("spf"),
        age: 25,
    };

    println!("{}", user.name);

    println!("{}", user.age);
}
```

这里的 `user.name` 表示访问 `user` 的 `name` 字段。`user.age` 表示访问 `user` 的 `age` 字段

## 4、修改结构体字段

如果要修改结构体字段，结构体变量本身必须声明为 `mut`。

示例：

```rust
#[derive(Debug)]
struct User {
    name: String,
    age: u32,
}

fn main() {
    let mut user = User {
        name: String::from("spf"),
        age: 25,
    };

    user.age += 1;   // 修改结构体字段

    println!("{:?}", user);
}
```

这里 `user` 声明为 `mut`，所以可以修改它的字段。

Rust 的可变性是针对整个变量绑定的。也就是说，只要 `user` 是可变的，就可以修改它的任意字段。

不能只把某个字段单独声明为可变字段。

## 5、字段初始化简写与结构体更新语法

### （1）字段初始化简写

如果变量名与字段名相同，可以使用字段初始化简写。

示例：

```rust
struct User {
    name: String,
    age: u32,
}

fn build_user(name: String, age: u32) -> User {
    User {
        name,
        age,
    }
}
```

这里的：

```rust
User {
    name,
    age,
}
```

等价于：

```rust
User {
    name: name,
    age: age,
}
```

字段初始化简写可以减少重复代码，尤其适合构造函数或转换函数。

### （2）结构体更新语法

创建结构体实例时，可以基于已有实例创建新实例。

这叫做结构体更新语法（struct update syntax）。

示例：

```rust
#[derive(Debug)]
struct User {
    name: String,
    age: u32,
    id: u64,
}

fn main() {
    let user1 = User {
        name: String::from("Tom"),
        age: 20,
        id: 1000,
    };

    let user2 = User {
        name: String::from("John"),
        id: 1001,
        ..user1  // 结构体更新语法
    };

    println!("{:?}", user2);
}
```

`..user1` 表示：没有显式写出的字段，从 `user1` 中补齐。上面的代码中，`user2.age` 来自 `user1.age`。

结构体更新语法本质上会把字段从旧实例移动或复制到新实例中：

- 如果被补齐的字段是`Copy`类型，旧实例对应字段会被复制
- 如果被补齐的字段是非`Copy`类型，旧实例对应字段会被移动

示例：

```rust
#[derive(Debug)]
struct User {
    name: String,
    age: u32,
}

fn main() {
    let user1 = User {
        name: String::from("Tom"),
        age: 20,
    };

    let user2 = User {
        age: 21,
        ..user1
    };

    println!("{:?}", user2);

    // println!("{:?}", user1); // 错误
}
```

这里 `user2` 的 `name` 字段来自 `user1.name`。

由于 `String` 不是 `Copy` 类型，所以 `user1.name` 被移动到了 `user2` 中。

因此，`user1` 不能再作为整体使用。

如果只访问没有被移动的字段，有些情况下仍然可以使用。

示例：

```rust
struct User {
    name: String,
    age: u32,
}

fn main() {
    let user1 = User {
        name: String::from("Tom"),
        age: 20,
    };

    let user2 = User {
        age: 21,
        ..user1
    };

    println!("{}", user1.age);

    println!("{}", user2.name);
}
```

这里 `user1.name` 已经被移动，但 `user1.age` 是 `Copy` 类型，没有被移动，所以仍然可以访问。

> 结构体更新语法可能移动旧实例中的字段，因此不要随意继续使用旧实例。


# 三、结构体的可见性

## 1、结构体默认是私有的

Rust 中的结构体默认是**私有**的。

如果一个结构体定义在某个模块中，那么默认只能在当前模块及其子模块中使用。有关模块详见[[9、crate与模块|crate与模块]]。

如果希望结构体对外部可见，需要使用`pub`关键字。

示例：

```rust
pub struct User {
    name: String,
    age: u32,
}
```

这里 `User` 类型本身是公开的。


## 2、字段默认也是私有的

即使结构体本身是 `pub`，它的字段默认仍然是私有的。

示例：

```rust
pub struct User {
    name: String,
    age: u32,
}
```

这里外部模块可以知道 `User` 这个类型存在，但不能直接访问或构造它的私有字段。

如果希望字段也能被外部访问，需要给字段也加上 `pub`。

示例：

```rust
pub struct User {
    pub name: String,
    pub age: u32,
}
```


## 3、私有字段与构造函数

标准库里的很多类型不会公开内部字段。

例如 `String`、`Vec<T>` 等类型都不会让你直接写结构体表达式创建它们，而是通过**公开的关联函数**创建。

例如：

```rust
let s = String::new();

let v: Vec<i32> = Vec::new();
```

这样做的好处是：类型可以隐藏内部实现，只暴露安全、稳定的使用方式。

自定义结构体也经常这样设计：字段保持私有，通过公开函数创建和修改。


# 四、元组结构体

## 1、定义元组结构体

元组结构体的字段没有名字，只有类型和位置。

示例：

```rust
struct Color(i32, i32, i32); // RGB颜色

struct Point(i32, i32, i32); // 三维坐标
```

`Color` 和 `Point` 的字段类型完全一样，都是三个 `i32`。

但它们是两个不同的类型。

这就是元组结构体和普通元组的重要区别：**元组结构体有自己的类型名，因此语义更清楚，也能避免不同含义的数据被混用**。

## 2、实例化元组结构体

元组结构体使用类型名加圆括号创建实例。

示例：

```rust
struct Color(i32, i32, i32);

fn main() {
    let black = Color(0, 0, 0);
}
```

这里 `Color(0, 0, 0)` 创建了一个 `Color` 类型的值。

从写法上，它像函数调用。**实际上，定义元组结构体时，Rust会生成一个同名的构造函数**。

## 3、访问元组结构体字段

元组结构体的字段可以通过数字索引访问。

示例：

```rust
struct Color(i32, i32, i32);

fn main() {
    let black = Color(0, 0, 0);

    println!("{}", black.0);

    println!("{}", black.1);

    println!("{}", black.2);
}
```

这里 `black.0` 表示第一个字段。

`black.1` 表示第二个字段。

`black.2` 表示第三个字段。

## 4、元组结构体使用场景

元组结构体适合字段含义比较简单，或者字段本身通过类型名已经能表达清楚的场景。       

例如：

```rust
struct Millimeters(u32);

struct Meters(u32);
```

虽然它们内部都是 `u32`，但 `Millimeters` 和 `Meters` 是不同类型。

这样可以避免把不同单位的数据混在一起。

这种写法常被称为**新类型模式**（newtype pattern）。

示例：

```rust
struct UserId(u64);

struct OrderId(u64);

fn find_user(id: UserId) {
    println!("{}", id.0);
}

fn main() {
    let user_id = UserId(100);

    let order_id = OrderId(200);

    find_user(user_id);

    // find_user(order_id); // 错误，OrderId不是UserId
}
```

这里 `UserId` 和 `OrderId` 内部都是 `u64`，但类型不同，因此不能混用。

**元组型结构体的单个元素可以是公共的，也可以不是**。

# 五、单元型结构体

## 1、定义单元结构体

单元结构体是不包含任何字段的结构体。

示例：

```rust
struct Marker;
```

它没有花括号，也没有字段列表。

创建它的值时，直接写类型名即可。

示例：

```rust
struct Marker;

fn main() {
    let m = Marker;
}
```

## 2、单元结构体的特点

单元结构体没有字段，因此不携带运行时数据。

它有点像单元类型`()`，但它仍然是一个独立的自定义类型。

示例：

```rust
struct AlwaysEqual;

fn main() {
    let a = AlwaysEqual;
}
```

`AlwaysEqual` 是一个类型，同时 `AlwaysEqual` 也可以作为这个类型的值来使用。


## 3、单元结构体的使用场景

单元结构体通常不会占用实际数据空间，适合表达“这个类型本身有意义，但不需要存储数据”的场景，如标记类型或零大小类型。

示例：

```rust
struct ConsoleLogger;

impl ConsoleLogger {
    fn log(&self, message: &str) {
        println!("{}", message);
    }
}

fn main() {
    let logger = ConsoleLogger;

    logger.log("hello");
}
```

这里 `ConsoleLogger` 本身不需要保存数据，但它可以拥有方法，用来表达一种行为。

后续学习[[10、泛型和特征|Trait]]后，可以为单元结构体实现某个Trait，让它表示某种策略或行为。

# 六、结构体方法

## 1、方法的基本概念

结构体不仅可以保存数据，也可以定义与数据相关的行为。这些行为通常写成方法。

在Rust中，方法不直接写在结构体内部，而是写在`impl`块中。

示例：

```rust
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    println!("{}", rect.area());
}
```

这里 `impl Rectangle` 表示：下面这些函数是和 `Rectangle` 类型关联的。

`area` 是一个方法，因为它的第一个参数是 `self` 相关形式。


## 2、self、&self、&mut self

方法的第一个参数通常是 `self`、`&self` 或 `&mut self`。以`self`为例，实际上它是`self: Self`的简写，`Self`表示`impl`块后面的那个类型，也就是`Rectangle`。

`self`、`&self`、`&mut self`表示如何使用当前实例：

- `self`表示取得当前实例的所有权。

- `&self`表示共享借用当前实例，只读取数据。

- `&mut self`表示可变借用当前实例，可以修改数据。

示例：

```rust
struct Counter {
    value: u32,
}

impl Counter {
    fn get(&self) -> u32 {
        self.value
    }

    fn increase(&mut self) {
        self.value += 1;
    }

    fn reset(self) {
        println!("counter consumed");
    }
}

fn main() {
    let mut counter = Counter { value: 0 };

    println!("{}", counter.get());

    counter.increase();

    println!("{}", counter.get());

    counter.reset();

    // println!("{}", counter.get()); // 错误，counter已经被移动
}
```

这里：

- `get(&self)` 只是读取数据。

- `increase(&mut self)` 会修改数据。

- `reset(self)` 会取得实例所有权，调用后原变量不能继续使用。

## 3、方法调用中的自动借用

调用方法时，Rust会根据方法的`self`参数形式自动借用或移动。

示例：

```rust
struct Counter {
    value: u32,
}

impl Counter {
    fn increase(&mut self) {
        self.value += 1;
    }
}

fn main() {
    let mut counter = Counter { value: 0 };

    counter.increase();
}
```

这里方法定义需要 `&mut self`。

调用时直接写：

```rust
counter.increase();
```

Rust 会自动理解为：

```rust
Counter::increase(&mut counter);
```

这就是方法调用语法的便利之处。

- 如果方法需要 `&self`，Rust 会自动创建共享引用。
- 如果方法需要 `&mut self`，Rust 会自动创建可变引用。
- 如果方法需要 `self`，Rust 会移动当前实例。

## 4、综合示例：实现Queue

下面用两个 `Vec` 实现一个简单队列。

示例：

```rust
struct Queue {
    older: Vec<i32>,
    younger: Vec<i32>,
}

impl Queue {
    fn push(&mut self, val: i32) {
        self.younger.push(val);
    }

    fn pop(&mut self) -> Option<i32> {
        if self.older.is_empty() {
            if self.younger.is_empty() {
                return None;
            }

            std::mem::swap(&mut self.older, &mut self.younger);

            self.older.reverse();
        }

        self.older.pop()
    }

    fn is_empty(&self) -> bool {
        self.older.is_empty() && self.younger.is_empty()
    }
}

fn main() {
    let mut q = Queue {
        older: Vec::new(),
        younger: Vec::new(),
    };

    q.push(10);

    q.push(20);

    q.push(30);

    println!("{}", q.is_empty());

    println!("{:?}", q.pop());

    println!("{:?}", q.pop());
}
```

这里：

- `push(&mut self)` 需要修改队列，所以使用可变引用。

- `pop(&mut self)` 也需要修改队列，所以使用可变引用。

- `is_empty(&self)` 只是读取队列状态，所以使用共享引用。

# 七、关联函数

## 1、关联函数基本概念

`impl`块中不仅可以定义方法，也可以定义不接收`self`的函数。这种函数也属于某个类型，称为**关联函数**。

关联函数不是方法，因为它没有 `self` 参数。

关联函数通常用于创建实例。

示例：

```rust
struct Queue {
    older: Vec<i32>,
    younger: Vec<i32>,
}

impl Queue {
    fn new() -> Self {
        Self {
            older: Vec::new(),
            younger: Vec::new(),
        }
    }
}
```

这里 `new` 是 `Queue` 的关联函数。

`Self` 表示当前 `impl` 对应的类型，也就是 `Queue`。

所以：

```rust
Self {
    older: Vec::new(),
    younger: Vec::new(),
}
```

等价于：

```rust
Queue {
    older: Vec::new(),
    younger: Vec::new(),
}
```


## 2、调用关联函数

关联函数使用 `类型名::函数名` 调用。

示例：

```rust
struct Queue {
    older: Vec<i32>,
    younger: Vec<i32>,
}

impl Queue {
    fn new() -> Self {
        Self {
            older: Vec::new(),
            younger: Vec::new(),
        }
    }

    fn push(&mut self, val: i32) {
        self.younger.push(val);
    }
}

fn main() {
    let mut q = Queue::new();

    q.push(10);
}
```

`Queue::new()` 表示调用 `Queue` 类型上的 `new` 函数。

在 Rust 中，构造函数通常命名为 `new`。

例如：

- `Vec::new()`

- `String::new()`

- `HashMap::new()`

有些类型也会提供其他构造函数，例如 `Vec::with_capacity()`。


# 八、关联常量

## 1、关联常量的基本概念

结构体的 `impl` 块中也可以定义关联常量。

关联常量和类型本身关联，通常用于表示这个类型下的固定值或常用值。

使用时通过 `类型名::常量名` 访问。

示例：

```rust
#[derive(Debug)]
struct Vector2 {
    x: f64,
    y: f64,
}

impl Vector2 {
    const NAME: &str = "Vector2";

    const ZERO: Self = Self { x: 0.0, y: 0.0 };

    const UNIT: Self = Self { x: 1.0, y: 1.0 };
}

fn main() {
    println!("{}", Vector2::NAME);

    println!("{:?}", Vector2::ZERO);

    println!("{:?}", Vector2::UNIT);
}
```

这里：

- `Vector2::NAME` 是和 `Vector2` 类型关联的字符串常量。

- `Vector2::ZERO` 表示零向量。

- `Vector2::UNIT` 表示一个常用向量值。

## 2、关联常量的类型

关联常量的类型不一定必须是当前结构体类型。

示例：

```rust
struct User;

impl User {
    const DEFAULT_AGE: u32 = 18;
}
```

这里 `DEFAULT_AGE` 的类型是 `u32`，不是 `User`。

关联常量只是定义在某个类型命名空间下的常量。


# 九、结构体内存布局

## 1、结构体在内存的基本理解

结构体可以理解为多个字段组合成的一个值。

例如：

```rust
struct Point {
	x: i32,
	y: i32
}
```

一个 `Point` 值包含两个字段：`x` 和 `y`。

在内存层面，结构体字段会放在同一个结构体值中。编译器会根据**字段类型、对齐要求和优化**需要决定实际布局。


## 2、默认布局不保证字段顺序

Rust 默认结构体布局叫做 `repr(Rust)`。

在默认布局下，Rust 不承诺字段在内存中的具体顺序、间距和填充方式。

因此，普通 Rust 代码不要依赖结构体字段的内存排列顺序。

例如，不要假设下面这个结构体一定按照 `a`、`b` 的顺序存储：

```rust
struct Foo {
    a: u8,
    b: u32,
}
```


## 3、需要稳定布局时使用repr(C)

如果需要和 C 语言交互，或者确实需要接近 C 语言的字段布局规则，可以使用 `#[repr(C)]`。

```rust
#[repr(C)]
struct Foo {
    a: u8,
    b: u32,
}
```

`#[repr(C)]` 表示让 Rust 按照 C ABI 相关规则来布局这个结构体。

这通常用于 FFI 场景，也就是 Rust 和其他语言进行底层接口交互的场景。

普通业务代码里，一般不需要关心 `#[repr(C)]`。
