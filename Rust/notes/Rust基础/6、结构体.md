# 一、结构体的定义与实例化

结构体（struct）是一种自定义数据类型，用于把多个不同类型的数据组合到一起。在Rust中，使用 **struct关键字** 定义结构体。 **Rust结构体分为具名字段型结构体、元组型结构体和单元型结构体** 。

## 1、具名字段型结构体

具名字段结构体的每个字段都有名字，其定义示例如下：

```rust
struct User {

name: String,

age: u32,

}
```
注意：

Rust定义结构体后面不需要分号;

结构体的每一个属性也称为字段（field），由字段名+类型组成

字段类型必须显式声明

定义结构体类型后，就可以通过 **结构体表达式** 构造此类型的值了。结构体表达式以结构体类型名开头，后跟一对花括号{}，其中列出每个字段的名称和值。

示例：
```rust
#[derive(Debug)]

struct User {

name: String,

age: u32,

}

fn main() {

let user = User {

name: String::from("spf"),

age: 25,

};

println!("{:?}", user);

println!("{:#?}", user);

}
```
注意：

为了打印结构体变量，这里在定义结构体的上方使用了# **[derive(Debug)]** 。它是派生属性宏，用于自动为结构体（或枚举）实现Debug trait。

**{:?}为普通Debug输出，{:#?}为带缩紧的美化输出**

结构体默认情况下是 **私有的** ，只能在声明它们的模块及子模块中可见。 **为了使得结构体对外可见，需要加上pub关键字** 。

**结构体的字段默认也是私有的，即使结构体声明为pub，字段对外也是私有的** 。 **通过结构体表达式创建结构体类型的值时，就需要所有字段都是pub** 。（注：前面的例子没有加pub是因为就在当前模块定义，对当前模块可见）。

因此，在创建String、Vec等标准库结构体值时，就没有通过结构体表达式，因为它们的字段都是私有的。只能通过公共类型关联函数创建，例如String::new或Vec::new等。

可以通过 **点号. 访问结构体变量的每一个字段** 。如果想要修改结构体的某个字段，需要事先将结构体声明为 **mut 可变** ：

示例：
```rust
let mut user = User {

name: "spf".to_string(),

age: 25,

};

user.age += 1;

println!("{:?}", user);
```
创建具名字段结构体的值， **Rust提供了快捷的结构体更新语法（struct update syntax）** ，方便基于已有结构体实例快速创建新的实例。

结构体更新语法使用**..某实例名** ，将新结构体实例中未显式赋值的字段从某实例中补齐。

示例：
```rust
let user1 = User {

name: String::from("Tom"),

age: 20,

id: 1000,

};

// user2的age字段值与user1相同

let user2 = User {

name: String::from("John"),

id: 1001,

.. user1

};

// 注意，这里user1是否还能用，取决于字段是否发生move

println!("{:?}", user1);

println!("{:?}", user2);
```
注意事项：

结构体更新语法本质是 **从旧实例把字段值移动/复制到新实例**

只要被补齐的字段里包含非Copy类型（如String），就会发生所有权的转移（move），此时旧实例在这些字段上会失效，从而导致旧实例整体失效

如果被补齐的字段全部都是Copy类型，则旧实例仍可使用
## 2、元组型结构体

元组型结构体类似于元组： **字段没有名字，只有类型与字段索引** 。但元组型结构体又不同于元组， **它有自己的类型名，因此语义更清晰，也能区分同形不同义的数据** 。

定义：

```rust
struct Name(type1, type2, type3);
```
示例：
```rust
struct Color(i32, i32, i32); // RGB颜色

struct Point(i32, i32, i32); // 三维坐标
```
尽管字段都是三个i32，但是Color和Point是两个不同的类型。

实例化元组结构体时，通过Name(val1, val2, val3)形式。这看起来是一个函数调用，实际上确实是。定义元组型结构体时会隐式定义一个函数：
```text
fn Name(elem0: type1, elem1: type2, elem2: type3) -> Name {... }
```
元组结构体保存的值称为元素，可以通过像元组的索引一样访问它们：
```rust
println!("{} {} {}", black.0, black.1, black.2);
```
**元组型结构体的单个元素可以是公共的，也可以不是** 。

在基本层面，具名字段型结构体和元组型结构体非常类似，选择哪一个需要考虑易读性、简洁性和无歧义性：

如果.运算符来获取值的各个组件，用名称标识字段名为读者提供更多信息

如果通常用模式匹配来查找元素，则推荐用元组型结构体

元组型结构体也常用于 **创造新类型** ，即定义一个只包含单一组件的结构体，以获得更为严格的类型检查。例如使用纯ASCII文本时，可以定义：
```rust
struct ASCII(Vec<u8>);
```
将此类型用于ASCII字符串比简单传递Vec<u8>缓冲区并在注释中解释更加清晰，并且在其他类型的字节缓冲区传递给需要ASCII文本的函数时，这种新类型能帮Rust捕获错误。
## 3、单元型结构体

单元型结构体声明了一个不包含任何字段的结构体类型。

示例：

```rust
struct Onesuch;
```
这种类型的值在运行时不需要存储任何的数据，因此看起来非常像单元类型()： **Rust往往不会在内存中真正为它分配空间，也不会生成读写字段之类的代码操作它们** 。

不过，从类型系统角度看，单元型结构体仍然是一个正常的类型，并且它恰好只有一个可能的值，也就是它自己：
```rust
let o = Onesuch;
```
以前见到的 **3..5本质上是Range { start: 3, end: 5 }** 的简写，而**..则是单元类型RangeFull** 的简写形式。

**单元型结构体在处理trait时非常有用，尤其适合用作标记类型（marker type）和零开销策略/配置类型** 。
# 二、结构体内存布局

**在内存层面，具名字段型结构体（struct S {... }）和元组型结构体(struct T(... ))本质上没有区别** ： **它们都是一组字段/元素按照某种规则组合成的连续数据块，编译器会根据对齐（alignment）和填充（padding）等要求决定最终布局** 。

与C/C++不同，Rust默认不承诺结构体字段/元组元素在内存中的具体顺序、间距和填充方式（属于布局细节，编译器可能为了优化而调整），因此：

不要在repr(Rust)（默认表示）下，依赖字段在内存中的排列顺序做FFI、二进制序列化等操作

如果需要和C/C++进行FFI交互，或者需要稳定的ABI布局，应显式标注：

```rust
#[repr(C)]

struct Foo {

a: u8,

b: u32,

}
```
**#[repr(C)]的含义是：要求编译器按C ABI约定来布局结构体字段（包括字段顺序和对齐规则），从而保证与C/C++侧的同名结构体在内存表示上可对齐。**
# 三、结构体方法

目前为止，或多或少都在对各种值调用方法，例如v.push(e)将元素e压进向量，v.len()获取向量的长度等。

可以为自定义结构体类型定义方法。 **和C++不同的是，Rust方法不直接定义在结构体中，而是单独定义在impl块中** 。（注：implementation的缩写）

**使用impl关键字+具体结构体类型+{}定义impl块** 。impl块中的所有内容都与对应结构体类型相关联，方法就定义在其中，也称 **关联函数** 。相对的，未定义在impl块中的函数称为自由函数。

示例1：

通过两个Vec来定义一个Queue，为Queue实现push和pop方法。

```rust
struct Queue {

older: Vec<i32>,

younger: Vec<i32>,

}

impl Queue {

/// 将新元素val入队

fn push(&mut self, val: i32) {

self.younger.push(val);

}

/// 旧元素出队

/// 队列为空，返回 `None`

/// 队列不为空，返回 `Some(val)`

fn pop(&mut self) -> Option<i32> {

// 若older为空，先更新older为younger的值

if self.older.is_empty() {

if self.younger.is_empty() {

return None;

}

use std::mem::swap;

swap(&mut self.older, &mut self.younger);

self.older.reverse();

}

self.older.pop()

}

}

fn main() {
```
注意事项：

由于push和pop需要修改Queue，因此它们接受 **&mut self** 参数。 **push和pop的第一个参数具有特殊名称self，它实际是一个语法糖，是self: Self的简写** 。 **Self则是impl后的类型名，这里就是Queue。因此，这里的&mut self等价于self: &mut Queue** 。

结构体的self参数有三种形式：

self（即 self: Self）：交出所有权，结构体实例所有权将转移进方法中。

&self（即 self: &Self）：结构体实例的共享引用（只读）

&mut self（即 self: &mut Self）：结构体实例的可变引用（可修改）

结构体实例进行方法调用时，.运算符会根据方法的第一个参数自动借用，例如上述的q.push(10)本质是(&mut q).push(10)。

为了演示三种不同的self参数，进一步拓展上述示例：
```rust
struct Queue {

older: Vec<i32>,

younger: Vec<i32>,

}

impl Queue {

/// 将新元素val入队

fn push(&mut self, val: i32) {

self.younger.push(val);

}

/// 旧元素出队

/// 队列为空，返回 `None`

/// 队列不为空，返回 `Some(val)`

fn pop(&mut self) -> Option<i32> {

// 若older为空，先更新older为younger的值

if self.older.is_empty() {

if self.younger.is_empty() {

return None;

}

use std::mem::swap;

swap(&mut self.older, &mut self.younger);

self.older.reverse();

}

self.older.pop()

}

/// 判断队列是否为空

fn is_empty(&self) -> bool { // 共享借用，不修改Queue

self.older.is_empty() && self.younger.is_empty()

}
```
# 四、类型关联函数和关联常量

### 1、类型关联函数

给定类型的impl块中也可以定义 **不以self为参数的函数** ，这些函数也是关联函数，但它们不是方法，因为不接受self参数，称为 **类型关联函数** 。

类型关联函数通常用于提供构造函数：

```rust
impl Queue {

// 类型关联函数，返回一个新的Queue实例

fn new() -> Self {

Self { older: Vec::new(), younger: Vec::new() }

}

}
```
在Rust中，构造函数按照惯例命名为new，例如Vec::new()，HashMap::new等。类型通常还有其他关联函数作为构造函数，如Vec::with_capacity。

使用此函数时，需要通过 **类型名称::函数名** 的方式调用。
```rust
fn main() {

// 类型关联函数

let mut q = Queue::new();

q.push(10);

q.push(20);

q.push(30);

println!("is empty: {}", q.is_empty());

let (older, younger) = q.split();

println!("older: {:?}", older);

println!("younger: {:?}", younger);

}
```
## 2、关联常量

Rust中还可以为结构体类型定义关联常量，它与类型本身关联，它们通常用于表示指定类型下的常用值。使用时使用 **类型名::常量名** 访问。

**关联常量类型不一定是所关联的类型，可以是其他类型** 。

示例：

```rust
#[derive(Debug)]

struct Vector2 {

x: f64,

y: f64,

}

impl Vector2 {

const NAME: &str = "Vector2";

const ZERO: Self = Self {x: 0.0, y: 0.0};

const UNIT: Self = Self {x: 1.0, y: 1.0};

}

fn main() {

println!("{}", Vector2::NAME);

println!("{:?}", Vector2::ZERO);

println!("{:?}", Vector2::UNIT);

}
```
