## 引用和切片

## 一、问题引入

假设有一个哈希表（Rust标准库提供了HashMap类型），其健是类别名（String)，值则是该类别下的一些例子（Vec<String>）。

示例：

展开

自动换行

use std::collections::HashMap;

// 类型别名

type Table = HashMap<String, Vec<String>>;

fn show\_table(table: Table) {

for (tag, examples) in table {

println!("{}:", tag);

for example in examples {

println!("\\t{}", example)

}

}

}

fn main() {

let mut table = Table::new();

table.insert("fruit".to\_string(),

vec!\["apple".to\_string(),

"orange".to\_string(),

"banana".to\_string()\]);

table.insert("animal".to\_string(),

vec!\["dog".to\_string(),

"cat".to\_string(),

"rabbit".to\_string()\]);

table.insert("color".to\_string(),

vec!\["red".to\_string(),

"green".to\_string(),

"blue".to\_string()\]);

show\_table(table);

}

结果：

![[Image 38.png]]

上述的代码没有任何问题。但注意show\_table(table)会将按值传入，即table的所有权会转移进show\_table函数内，因此在执行show\_table函数返回后，table就失效了。

很多情况下不想获取值的所有权，而仅仅想临时使用值，于是就有了 **引用（Reference）** 。

## 二、引用

## 1、引用的概念

引用是 **非拥有型指针** ，允许临时使用被引用的值而不获取值的所有权。引用的生命周期不能超出其引用目标，即若引用的目标离开作用域，不允许还存在指向该目标的引用。为了强调这一点，Rust把创建某个值的引用操作称为 **借用（borrow）** ：凡是借用，终须归还。

## 2、共享引用和可变引用

引用分为两种： **共享引用和可变引用** 。

**共享引用** ： **允许读取但不能修改其引用目标** 。表达式 **&e** 会产生对e的共享引用，若e的类型为T，则&e的类型就是 **&T** 。 **可以根据需同时拥有任意数量对特定值的共享引用** ，因此 **共享引用是Copy类型** 。

示例：

展开

自动换行

fn main() {

let s = String::from("Hello");

let r1 = &s; // r1是对s的共享引用(&String)

let r2 = &s; // r2也是对s的共享引用(&String)

println!("r1: {}", r1);

println!("r2: {}", r2);

}

结果：

如下给出了引用的内存示意图：

**可变引用** ： **允许读取和修改其引用目标** 。表达式 **&mut e** 会产生对e的可变引用，若e的类型为T，则&e的类型就是 **&mut T** 。 **一旦一个值拥有了可变引用，就无法再对该值创建其他种类的引用了** 。

示例：

展开

自动换行

fn main() {

let mut s = String::from("Hello"); // 注意：s要声明为mut

let r1 = &mut s; // r1是对s的可变引用

r1.push\_str(", World"); // 可以通过r1修改数据

println!("r1: {}", r1);

println!("s: {}", s);

}

结果：

![[Image 41.png]]

注意：

因为要创建值的可变引用，因此值本身也需要声明为可变。

进一步地，可以交换println!("r1: {}", r1);和println!("s: {}", s);的顺序，发现编译报错：

![[Image 42.png]]

提示： **不能将s借用为不可变引用因为它已经被借用为了可变引用** 。这里需要说明的一点是， **println!实质是获取s的共享引用** 。

可变引用有一个严格限制： **一旦存在一个值的可变引用，就无法为该值创建其他引用了** 。

实际上， **只要存在对值的共享引用，即使是所有者也不能修改值，该值被锁定。只要存在某个值的可变引用，则它独占该值的访问权。在可变引用消失前，即使是所有者也无法使用该值** 。

总而言之，引用的使用规则概括如下：

可以创建对一个值的多个共享引用

同一时间，只能存在一个值的可变引用，不能存在其他类型的引用

## 3、自动解引用和自动借用

### （1）显式创建与解引用

共享引用：使用 **&运算符** 创建共享引用，通过 **\*运算符** 进行显式解引用以访问底层值。

示例：

展开

自动换行

let x = 10;

let r = &x; // r 的类型是 &i32

assert!(\*r == 10); // 显式解引用：\*r 得到 i32 类型的 10

可变引用：使用 **&mut** 运算符创建可变引用。 **注意：变量本身必须声明为mut才可创建可变引用** 。

示例：

展开

自动换行

let mut y = 32;

let m = &mut y;

\*m += 32;

assert!(\*m == 64);

### （2）隐式解引用

虽然\*用于显式解引用，但在很多场景下，Rust编译器会自动执行解引用操作，使得代码更简洁。

例如通过**.运算符访问结构体字段** 。

示例：

展开

自动换行

struct Person {

name: String,

age: u32,

}

fn main() {

let p = Person {

name: "Tom".to\_string(),

age: 20,

};

let p\_ref = &p; // p\_ref 类型为 &Person

// 等价于 (\*p\_ref).name

// 编译器检测到 p\_ref 是引用，自动解引用一次以访问 name 字段

assert\_eq!(p\_ref.name, "Tom".to\_string());

}

Rust支持对引用再进行引用，此时.运算符会尽可能追踪多层次的引用，解引用到最终的目标：

展开

自动换行

let p = Person {

name: "Tom".to\_string(),

age: 20,

};

let r: &Person = &p;

let rr: &&Person = &r;

let rrr: &&&Person = &rr;

assert\_eq!(rrr.age, 20);

rrr.age会根据rrr的类型指引遍历3层引用取到最终的age字段。

### （3）比较引用

除了.运算符外，比较运算符也可以自动进行解引用。

示例：

展开

自动换行

let x = 10;

let y = 10;

let rx = &x;

let ry = &y;

let rrx = &rx;

let rry = &ry;

assert!(rrx == rry);

\==运算符会追踪所有的引用并对最终的目标x和y进行比较。

需要注意的是，比较运算符左右两边的类型必须完全一致，下面这种写法就是错误的：

展开

自动换行

assert!(rrx == ry); // 错误，rrx是&&i32类型，ry则是&i32类型，不匹配

比较运算符默认会对引用进行解引用，最终比较底层的目标的值是否一样。如果想要知道两个引用是否指向同一个目标，需要使用std::ptr::eq，它会比较地址：

展开

自动换行

assert!(!std::ptr::eq(rx, ry));

### （4）自动借用

在调用方法时，Rust不仅会自动解引用，还会根据需要自动添加引用。

例如：Vec::sort的定义是fn sort(&mut self)，它需要一个可变引用。

示例：

展开

自动换行

let mut v = vec!\[20, 10\];

// 写法 1：直接调用

v.sort();

// 写法 2：显式借用

(&mut v).sort();

// 两者完全等效。编译器在写法 1 中自动插入了 &mut 操作。

## 4、对引用变量赋值

当引用变量声明为 **mut** 时，可以对该引用变量赋新值，使其指向新的目标。

示例：

展开

自动换行

let x = 10;

let y = 20;

let mut r = &x; // r是x的共享引用

r = &y; // 修改r，现在r是y的共享引用

assert!(\*r == 20);

这和C++的引用有很大区别，C++的引用一旦完成初始化就不能改变。

## 5、悬垂引用

C/C++里有悬垂指针的概念，在Rust中类似的也有悬垂引用的概念。所谓悬垂引用是指引用指向的数据已经无效了，但引用仍然指向了该数据。

示例：

展开

自动换行

fn dangle() -> &'static String {

let s = String::from("Hello");

&s //错误，s在函数结束后被释放，返回的引用指向了一个无效的数据

}

注意：这里加'static是 **生命周期** 的概念，先不用管这个，这里只是为了让编译器报悬垂引用的错而非没有指定生命周期的错。

结果：

解决悬垂引用的方式是返回其本身，而非引用：

展开

自动换行

let s = String::from("Hello");

}

## 6、借用任意表达式的值

**在Rust中，&运算符可以应用于任意表达式的结果** 。当对一个非引用类型的表达式结果进行借用时，Rust编译器会执行如下操作：

求值与存储：先计算表达式expr的值，并将其存入一个编译器生成的 **匿名临时变量** 中。

创建引用：让生成的引用指向这个临时变量

生命周期管理：Rust会根据引用的使用上下文，自动调整该临时变量的存活时间，防止悬空引用

示例：

展开

自动换行

fn factorial(n: usize) -> usize {

(1..=n).product()

}

fn main() {

// 1. 计算factorial(6)的结果

// 2. 存入临时变量temp

// 3. r指向temp

let r = &factorial(6);

// 数学运算符也会自动解引用

// 这里&1也会创建临时对象

assert\_eq!(r + &1, 721);

}

## 三、切片

## 1、切片和切片引用

在介绍切片（slice）之前，需要区分切片和切片引用的概念（很多资料会把切片引用简称为切片，但从类型角度需要区分清楚）。

切片： **切片是数组或向量中一个区域（写作不指定长度的\[T\]** 。由于切片\[T\]可以是任意长度，因此它是动态类型大小（DST，Dynamically Sized Type）。 **DST在编译时无法确定大小，因此不能直接存储在变量中或作为函数参数传递** 。

切片引用： **切片引用是对切片的引用** ，形式上写作&\[T\]或&mut \[T\]。与普通引用不同，切片引用本质上是一个 **非拥有型胖指针** ，包括指向切片第一个元素的指针和切片元素的数量（切片长度）。

为方便起见，后文常将切片引用（&\[T\]或&mut \[T\]）简称为切片。

示例：

展开

自动换行

let v = vec!\[0.0, 0.707, 1.0, 0.707\]; // 向量

let a = \[0.0, 0.707, 1.0, 0.707\]; // 数组

let sv: &\[f64\] = &v; // 注意，这里要类型注解，因为&v会被编译器自动推导为&Vec<f64>而非&\[f64\]

let sa: &\[f64\] = &v;

内存示意图如下：

![[Image 45.png]]

普通引用是指向单个值的非拥有型指针，而切片引用则是指向内存中一系列连续值的非拥有型指针。

## 2、创建切片

可以对数组或向量进行切片操作，这里以数组为例。 **arr\[begin..end\]会进行切片操作，获得\[T\]类型的切片** 。

示例：

展开

自动换行

fn main() {

let arr = \[10, 20, 30, 40, 50\];

// 创建不可变切片

let slice = &arr\[1..4\];

println!("slice: {:?}", slice); // \[20, 30, 40\]

// 创建可变切片

let mut arr2 = \[1, 2, 3\];

let slice\_mut = &mut arr2\[..\]; // 全量切片

println!("slice\_mut: {:?}", slice\_mut); // \[1, 2, 3\]

// 通过可变切片修改数据

slice\_mut\[1\] = 20;

println!("slice\_mut: {:?}", slice\_mut); // \[1, 20, 3\]

println!("arr2: {:?}", arr2); // \[1, 20, 3\]

}

说明：

\[..\]表示整个范围

切片索引会做边界检查，越界会panic

## 3、切片引用的本质

切片引用的本质是一个胖指针，它包含两部分信息：

数据指针 -> 指向切片起始元素的地址

长度 -> 切片包含的元素数量（元素个数，不是字节数）

在64位系统中，&\[T\] / &mut\[T\]通常占用16字节：指针8字节 + 长度8字节。

示例：

展开

自动换行

let arr = \[10, 20, 30, 40, 50\];

let slice: &\[i32\] = &arr\[1..4\]; // 切片引用

println!("slice: {:?}", slice);

println!("size of &\[i32\]: {}", size\_of::<&\[i32\]>()); // 16

println!("size of slice: {}", size\_of\_val(&slice)); // 16

注意：size\_of\_val的参数类型是引用&T的形式，它会去求T所占内存大小。这里的size\_of\_val(&slice)求的就是slice本身的大小（胖指针的大小）。如果是size\_of\_val(slice)，由于slice本身就是引用，得到的结果将是切片引用的长度。

## 4、切片引用的常用方法

示例：

展开

自动换行

let arr = \[10, 20, 30, 40, 50\];

let slice = &arr\[..\];

// 获取长度

println!("len of slice: {}", slice.len()); // len()方法是为\[T\]实现的方法

// 判断是否为空

println!("slice is empty: {}", slice.is\_empty());

// 安全访问：get 返回 Option<&T>

println!("{:?}", slice.get(0)); // Some(&10)

println!("{:?}", slice.get(100)); // None

// 切片再切片

let sub\_slice = &slice\[1..4\];

println!("sub\_slice: {:?}", sub\_slice); // \[20, 30, 40\]

get()方法返回的是Option<&T>，越界会返回None，更加安全。
