## 17、智能指针

## 一、智能指针的概念

## 1、指针和引用

指针是存储内存地址的变量，该地址指向另一块数据。在 Rust 中，最常见的“指向其他数据”的类型就是 **引用** ，引用通过 `&` 和 `&mut` 表示。

不过，Rust 中的引用不仅仅是“保存地址”这么简单，它还带有一套严格的语义： **借用** 。也就是说，引用表示“临时访问别人的数据”，但 **并不拥有该数据** 。因此， **引用属于非拥有型指针** 。

引用本身很轻量，通常不会引入额外的运行时开销；Rust 编译器会在编译期检查借用规则，以保证内存安全。

## 2、智能指针

智能指针和普通引用类似，也能“指向”某个值；但它通常比普通引用拥有更复杂的数据结构和更多的额外能力。

例如， `Vec<T>` 和 `String` 都可以看作“智能指针风格”的类型：

它们内部都持有指向堆上数据的指针

还额外保存了长度、容量等元数据

`String` 还保证其内容始终都是合法的UTF-8

智能指针和引用的重要区别在于：

引用只是借用数据，不拥有数据

智能指针通常拥有它所指向的数据

在Rust中，智能指针大多基于 **结构体** 实现，常见特点是实现了：

`Deref` ：使其可以像引用一样使用

`Drop` ：在离开作用域时自动执行清理逻辑

不过要注意，这并不是“所有智能指针都必须同时实现这两个 trait”，而是大多数常见智能指针都会具备这些能力。

## 二、Box<T>

## 1、Box<T>概念

`Box<T>` 是Rust中最基础的智能指针类型。它会把数据存储在 **堆** 上，而 `Box<T>` 本身（一个固定大小的指针）存储在 **栈** 上。

它的核心作用是：为数据提供一层堆分配的间接访问。

适用于：

将较大的数据存放在堆中，减少栈上的直接存储压力

处理递归类型，使类型大小在编译期可确定

保存Trait对象，例如 `Box<dyn Write>`

`Box<T>` 本身很轻量，但使用它会引入：

一次堆分配

一次间接访问（通过指针访问堆上的值）

## 2、Box<T>的使用

`Box<T>` 会在 **堆上** 为 `T` 类型的值分配内存，并返回一个拥有该数据所有权的智能指针。通常通过 `Box::new()` 创建。

示例：

展开

自动换行

let a = Box::new(3);

println!("{}", a); // 3

// let b = a + 1; // 报错

说明：

`3` 存储在堆上，a是一个 `Box<i32>` ，其本身位于栈上

`println!("{}", a)` 可以正常打印，是因为 `Box<T>` 在很多场景下可以像内部的 `T` 一样使用（后面介绍 `Deref` 原理）

`let b = a + 1` 会报错，因为 `Box<i32>` 不能直接和整数相加，需要先解引用： `let b = *a + 1`

当 `a` 离开作用域时， `Box<T>` 会自动释放它在堆上持有的数据

## 3、Box<T>使用场景

### （1）将较大的数据存储在堆上

示例：

展开

自动换行

let arr = Box::new(\[0; 1000\]);

println!("{}", arr.len());

let arr2 = arr;

println!("{}", arr2.len());

说明：

`[0; 1000]` 是一个较大的数组，通过 `Box<T>` 可以把它放到堆上

`let arr2 = arr` 发生的是 **所有权转移** ，不是深拷贝

转移时，栈上的 `Box` 值被移动到 `arr2` ，此后 `arr` 失效

底层堆上的数组始终只有一份数据

注意： `Box<T>` 不是 `Copy` 类型，因此这里不是“智能指针的拷贝”，而是 **move** 。

### （2）处理递归类型

Rust 编译器在编译期必须知道一个类型占用多少空间。如果一个类型定义中直接包含自己，那么它的大小就会变成 **无限递归** ，编译器无法计算。

示例：

展开

自动换行

enum List {

Cons(i32, List),

Nil,

}

上面这段代码无法通过编译，因为：

`List` 包含 `List`

`List` 里面又包含 `List`

如果无限嵌套下去，类型大小无法确定

![[Image 76.png]]

此时想要解决该问题，可以使用 `Box<T>` ：

展开

自动换行

enum List {

Cons(i32, Box<List>),

Nil,

}

use crate::List::{Cons, Nil};

fn main() {

let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));

}

此时：

`Cons` 变体中不再直接保存一个 `List` ，而是保存一个 `Box<List>`

`Box<List>` 自身大小固定，因此整个 `List` 大小就能确定

### （3）Trait对象

在 Rust 中，如果想把不同类型的值放进同一个集合中，常见方案有两种：

枚举

trait对象

其中trait对象通常更灵活。

示例：

展开

自动换行

trait Draw {

fn draw(&self);

}

struct Button {

id: u32,

}

impl Draw for Button {

fn draw(&self) {

println!("Draw Button {}", self.id);

}

}

struct Select {

id: u32,

}

impl Draw for Select {

fn draw(&self) {

println!("Draw Select {}", self.id);

}

}

fn main() {

let elems: Vec<Box<dyn Draw>> = vec!\[

Box::new(Button { id: 1 }),

Box::new(Select { id: 2 }),

\];

for elem in elems {

elem.draw();

}

}

说明：

`dyn Draw` 是一个trait对象类型，本身大小在编译期不可知，因此通常放到指针后面使用，例如 `&dyn Draw` ， `Box<dyn Draw` 。

`Vec<Box<dyn Draw>` 中每个元素都可以是不同的具体类型，只要它们实现了 `Draw`

## 三、Deref特征

智能指针与普通结构体的重要区别之一在于：它们常常实现了 `Deref` 和 `Drop` 。其中， `Deref` 允许一个类型在很多场景下表现得像一个引用，从而支持 `解引用` 行为。

## 1、常规引用

常规引用是一个指向某个值的借用。

示例：

展开

自动换行

fn main() {

let x = 5;

let y = &x;

assert\_eq!(5, x);

assert\_eq!(5, \*y);

}

这里：

`y` 是 `x` 的引用

`*y` 表示沿着引用取到它指向的值

如果写成：

展开

自动换行

assert\_eq!(5, y);

会报错，因为：

`5` 是 `i32`

`y` 是 `&i32`

两者类型不同，不能直接比较。

## 2、像引用一样使用Box<T>

`Box<T>` 可以像普通引用一样，通过 `*` 解引用得到内部的值。

示例：

展开

自动换行

let x = Box::new(5);

assert\_eq!(5, \*x);

## 3、实现Deref特征

标准库中， `std::ops::Deref` 的定义如下：

展开

自动换行

trait Deref {

type Target =?Sized;

fn deref(&self) -> &Self::Target;

}

为了演示 `Deref` 的功能，自定义一个 `MyBox<T>` ：

展开

自动换行

struct MyBox<T>(T);

impl<T> MyBox<T>{

fn new(v: T) -> Self {

Self(v)

}

}

此时使用它：

展开

自动换行

fn main() {

let x = MyBox::new(5);

assert\_eq!(5, \*x);

}

会报错，因为 `MyBox<T>` 没有实现 `Deref` ，所以不能对它使用解引用运算。

为其实现 `Deref` ：

展开

自动换行

use std::ops::Deref;

struct MyBox<T>(T);

impl<T> MyBox<T> {

fn new(v: T) -> Self {

Self(v)

}

}

impl<T> Deref for MyBox<T> {

type Target = T;

fn deref(&self) -> &Self::Target {

&self.0

}

}

fn main() {

let x = MyBox::new(5);

assert\_eq!(5, \*x);

}

此时代码即可通过编译。

从概念上讲， `*x` 会调用 `deref()` 取得一个引用，再进行普通解引用。可以近似理解为：

展开

自动换行

\*(x.deref())

## 4、函数和方法中的Deref强制转换

Deref强制转换（ **deref coercion** ）是Rust提供的一套便利能力：

**当某个类型实现了** `Deref` **，那么它的引用可以在函数传参、方法调用等场景中，自动转换为目标引用类型** 。

最典型的例子就是 `&String` 自动转换成了 `&str` ，这是因为 `String` 实现了 `Deref<Target=str>` 。

示例：

展开

自动换行

fn print\_s(s: &str) {

println!("{s}");

}

fn main() {

let s = String::from("hello");

print\_s(&s); // 传入&String

}

这种Deref强制转换可以持续进行：

展开

自动换行

fn print\_s(s: &str) {

println!("{}", s);

}

fn main() {

let s = Box::new(String::from("hello"));

print\_s(&s); // &Box<String> => &String => &str

}

## 5、DerefMut特征

与 `Deref` 特征对应，Rust还提供了 `DerefMut` ，用于支持可变引用的解引用。

`DerefMut` 特征的定义简化如下：

展开

自动换行

trait DerefMut: Deref {

fn deref\_mut(&mut self) -> &mut Self::Target;

}

示例：

展开

自动换行

use std::ops::{Deref, DerefMut};

struct MyBox<T>(T);

impl<T> MyBox<T> {

fn new(v: T) -> Self {

Self(v)

}

}

impl<T> Deref for MyBox<T> {

type Target = T;

fn deref(&self) -> &Self::Target {

&self.0

}

}

impl<T> DerefMut for MyBox<T> {

fn deref\_mut(&mut self) -> &mut Self::Target {

&mut self.0

}

}

fn main() {

let mut s = MyBox::new(10);

\*s = 11; // 修改

}

## 6、触发Deref强制转换的三种情况

Rust在函数参、方法调用等上下文中，会根据以下规则自动执行 `Deref强制转换` ：

当 `T: Deref<Target=U>` 时，可以把 `&T` 转换成 `&U`

当 `T: DerefMut<Target = U>` 时，可以把 `&mut T` 转换为 `&mut U`

当 `T: Deref<Target = U>` 时，可以把 `&mut T` 转换为 `&U`

第三种情况表示： **可变引用可以降级为不可变引用** 。

示例：

展开

自动换行

fn print(s: &str) {

println!("{}", s);

}

fn main() {

let mut s = String::from("hello");

print(&mut s); // &mut String -> &str

}

## 四、Drop 特征

## 1、Drop定义

在C这类没有 GC 的语言中，程序员通常需要手动管理资源释放；一旦忘记释放，就可能造成内存泄漏或其他安全问题。

Rust 采用 **RAII** （资源获取即初始化）的方式管理资源：当一个值离开作用域时，编译器会自动执行清理逻辑。若某个类型希望在销毁时执行自定义逻辑，就可以实现 `Drop` 特征。

因此， `Drop` 的作用是：

在值生命周期结束时自动执行清理代码

典型场景包括：关闭文件、释放堆内存、断开连接、打印日志等

在标准库中， `Drop` 特征定义如下：

展开

自动换行

trait Drop {

fn drop(&mut self);

}

只需要实现一个方法： **当值即将被销毁时，Rust 会自动调用该方法** 。

## 2、实现Drop

示例：

展开

自动换行

struct Foo {

data: String,

}

impl Drop for Foo {

fn drop(&mut self) {

println!("Drop Foo with data: {}", self.data);

}

}

fn main() {

let \_f = Foo {

data: "Hello".to\_string(),

};

println!("created Foo");

}

示例：

struct HasDrop1;

struct HasDrop2;

结论：

**局部变量级别：** 按照创建的顺序 **逆序** drop

结构体字段级别：字段会按照 **定义顺序** 依次drop

注意：如果一个类型实现了 `Drop` ，会先执行自己的drop，然后在自动去drop它的字段。

## 4、没有实现 Drop 的结构体

即使不给结构体手动实现 `Drop` ，它的字段依然会在离开作用域时自动被清理。

例如，去掉下面这段代码：

展开

自动换行

impl Drop for HasTwoDrops {

fn drop(&mut self) {

println!("Dropp HasTwoDrops");

}

}

此时：

`HasTwoDrops` 自己不会打印任何东西

但它内部的 `one` 和 `two` 仍然会被自动销毁

Rust编译器会为类型生成必要的 `drop glue` （析构逻辑），自动递归清理它的字段。

## 5、手动回收

**Rust不允许手动直接调用** `drop()` **方法** ，原因是 `Drop::drop(&mut self)` 接收的是一个 **可变借用** 。如果允许手动调用，那么值在销毁后，原变量仍然还在作用域，可能继续被访问，这将破坏Rust的安全保证。

示例：

展开

自动换行

struct Foo {

data: String,

}

impl Drop for Foo {

fn drop(&mut self) {

println!("Dropp Foo with data = {}", self.data);

}

}

fn main() {

let foo = Foo {

data: String::from("Hello"),

};

// foo.drop(); // 报错

}

**正确做法是使用标准库中的** `drop` **函数：**

展开

自动换行

pub fn drop<T>(\_x: T) {}

它本质上是一个空实现，但它会接收参数的所有权，因此参数\_x在函数结束时离开作用域，于是Rust自动析构它。

展开

自动换行

struct Foo {

data: String,

}

impl Drop for Foo {

fn drop(&mut self) {

println!("Dropp Foo with data = {}", self.data);

}

}

fn main() {

let foo = Foo {

data: String::from("Hello"),

};

drop(foo);

}

说明： `drop(foo)` 的本质是把 `foo` 的所有权转移给 `drop` 函数，调用后， `foo` 不再可用。

## 五、Rc<T>

## 1、引用计数机制

Rust的所有权规则要求： **一个值在任意时刻只能有一个所有者** 。这在大多数情况下没有问题，但在某些数据结构中，会希望多个地方共同拥有同一份数据：

图数据结构中，多个边可能指向相同的节点。只有当节点没有边指向时，才做清理释放工作。

多线程场景下，多个线程希望持有同一个数据。但受限于Rust引用的安全机制，无法做到。

为了解决这些场景的需求，Rust在所有权机制之外又引入了额外的措施来简化相应地实现：通过 **引用计数的机制** ，允许同一份数据在同一时刻拥有多个所有者。具体来说：

记录当前有多少个所有者

当引用计数归零时，才释放底层数据

Rust中提供了两种引用计数智能指针：

`Rc<T>` ： **单线程** 引用计数

`Arc<T>` ： **多线程** 引用计数（原子引用计数）

由于两者的接口几乎完全相同，只是适用场景不同，这里主要介绍 `Rc<T>` 。

## 2、Rc<T>的使用

`Rc` 是Reference Counting（引用计数）的缩写。当一个值需要在同一线程内被多个地方共同拥有时，可以使用 `Rc<T>` 。

示例1：

展开

自动换行

let a = Rc::new(String::from("Hello"));

let b = Rc::clone(&a);

assert\_eq!(Rc::strong\_count(&a), 2);

assert\_eq!(Rc::strong\_count(&a), Rc::strong\_count(&b));

说明：

`Rc::new(s)` 创建一个新的引用计数智能指针，指向底层的 `String` 数据值

`Rc::clone(&a)` 不会深拷贝底层字符串，只会让引用计数加1，也可以写成 `a.clone()` 。但需要你注意的是，这里的clone语义与一般clone语义完全不同。

`Rc::strong_count(&a)` 可以查看当前强引用数量。

`Rc` 不在 prelude 预导入模块中，需要手动导入： `use std::rc::Rc`

示例2：

展开

自动换行

struct Owner {

name: String,

}

struct Gadget {

id: u32,

owner: Rc<Owner>,

}

fn main() {

let owner = Rc::new(Owner {

name: "Tom".to\_string()

});

let g1 = Gadget{

id: 1,

owner: Rc::clone(&owner)

};

let g2 = Gadget{

id: 2,

owner: Rc::clone(&owner)

};

drop(owner);

println!("Gadget {} owned by {}", g1.id, g1.owner.name);

println!("Gadget {} owned by {}", g2.id, g2.owner.name);

}

说明：

`owner` 、 `g1.owner` 和 `g2.owner` 共同拥有同一个 `Owner`

即使 `owner` 被提前 `drop` ，只要还有其他 `Rc` 存在，底层数据就不会释放

`Rc<T>` 实现了 `Deref` ，因此可以直接写 `g1.owner.name`

## 3、Rc<T>只提供共享访问

`Rc<T>` 是一个 **共享所有权** 的智能指针，它允许多个所有者共同拥有同一份数据。但 **默认情况下，只能通过共享引用访问访问内部值，无法直接修改内部数据** 。

示例

展开

自动换行

let mut a = Rc::new(String::from("Hello"));

let b = Rc::clone(&a);

a.push\_str(" world"); // 错误

说明：

这里a可以直接调用 `push_str` ，是因为 `Rc` 也实现了 `Deref` 特征，这里会进行Deref coercion。

`a.push_str(" world")` 的直接原因是 `Rc<T>` 只实现了 `Deref<Target = T>` ，没有实现 `DerefMut`

因此只能共享访问 `Rc` 指向的数据，不可以修改

## 4、Rc<T>注意事项

`Rc<T>` 适用于单线程

`RC<T>` 允许多个所有者共享同一个值，但默认只提供共享访问，不能直接修改内部值。若要修改，通常需要搭配内部可变性类型使用，例如 `RefCell<T>`

多线程场景下使用 `Arc<T>` ，不要使用 `Rc<T>` 。

## 六、Cell<T>和RefCell<T>

## 1、内部可变性

Rust默认遵循这条规则：

要修改数据，必须拿到 `&mut T`

如果只有 `&T` ，那么就不能修改

这能在编译期保证安全。

但有些场景下，在只有 `&self` 的不可变引用前提下，却仍然希望修改对象内部的某个字段。这就是 **内部可变性（interior mutability）** 。

**内部可变性允许通过不可变引用修改内部数据** ，但安全检查会转移到特定机制上。

Rust标准库里最常见的两个内部可变性工具就是 `Cell<T>` 和 `RefCell<T>` 。

## 2、Cell<T>

### （1）问题引入

`Cell<T>` 用于实现内部可变性。它解决的问题是：当手里只有不可变引用 `&self` / `&T` 时，仍然想修改内部的某个值。

例如：

展开

自动换行

struct Counter {

counter: i32,

}

impl Counter {

fn increment(&self) {

self.counter += 1; // 报错

}

}

由于 `increment()` 的参数是 `&self` ，因此无法修改修改 `self.counter` 。

这里当然可以将参数修改为 `&mut self` ：

展开

自动换行

struct Counter {

count: i32,

}

impl Counter {

fn increment(&mut self) {

self.count += 1;

}

}

fn main() {

let mut c = Counter { count: 0 };

c.increment();

println!("{}", c.count);

}

这就是最自然、最推荐的写法。

但是，有些场景下，不能使用 `&mut self` ，Trait、API、设计约束已经固定成了 `&self` 。例如：

展开

自动换行

trait Messenger {

fn send(&self, msg: &str);

}

这个Trait规定死了：方法签名就是 `&self` ，不是 `&mut self` 。

假如在实现它的时候，想顺手记录调用了几次，那么就麻烦了：

展开

自动换行

struct MyMessenger {

count: usize,

}

impl Messenger for MyMessenger {

fn send(&self, msg: &str) {

self.count += 1; // 错误

println!("{}", msg);

}

}

因为 `send` 只有 `&self` 。

这时候 `Cell` 就派上用场了：

展开

自动换行

use std::cell::Cell;

trait Messender {

fn send(&self, msg: &str);

}

struct MyMessenser {

count: Cell<usize>,

}

impl Messender for MyMessenser {

fn send(&self, msg: &str) {

self.count.set(self.count.get() + 1);

println!("send {} times: {}", self.count.get(), msg);

}

}

说明：

整个 `MyMessenger` 对外仍然是“不可变借用”

但 `count` 这个字段允许内部修改

### （2）常用API

`Cell::new()` ： 创建一个 `Cell<T>`

`get()` ：取出内部值，要求 `T: Copy`

`set(v)` ：直接覆盖内部值

`replace(v)` ：替换新值，并返回旧值

`into_inner()` ：消费掉 `Cell<T>` ，拿出内部值

### （3）使用场景

`Cell<T>` 适合简单类型，如整型、布尔类型、字符型等其他 `Copy` 类型。核心特征是值比较小，改动方式主要是整体替换。

因为 `Cell<T>` 最舒服的模式是：

展开

自动换行

let old = cell.get();

cell.set(f(old));

但 `Cell<T>` 不适合复杂类型，特别是没有实现 `Copy` 的类型，例如：

展开

自动换行

use std::cell::Cell;

fn main() {

let c = Cell::new(vec!\[1, 2, 3\]);

}

`Cell` 中存放了一个 `Vec<String>` ，但无法通过 `c.get()` 获取，更不能写类似 `c.get().push(4)` 的代码，因为 `Vec<T>` 不是 `Copy` ， `Cell<T>` 也不会把 `&mut Vec<_>` 借出来。

如果你想 `push` 、 `insert` 、遍历、改某个元素，一般应该用 `RefCell<T>` 。

## 3、RefCell<T>

### （1）RefCell<T>与Cell<T>的区别

`RefCell<T>` 也是 Rust 里实现 **内部可变性** 的工具，它解决的问题是：当手里只有 `&self` / `&T` 时，仍然想修改内部数据，而且这个数据 **不是简单值** ，而是需要“ **借引用操作** ”的复杂数据。

它和 `Cell<T>` 的核心区别是：

`Cell<T>` 不借引用，靠整体取值和整体替换

`RefCell<T>` 可以借引用，但借用规则从编译期检查改到运行时检查

也就是说， `RefCell<T>` 更像是：普通借用规则还在，只是从编译期检查，变成了运行时检查。

### （2）RefCell<T>核心机制

`RefCell<T>` 允许你：

`borrow()` ：拿不可变借用

`borrow_mut()` ：拿可变借用

但它会在运行时检查借用是否合法。

规则与普通借用本质一样：

可以有多个不可变借用或者一个可变借用

不能同时有多个可变借用

不能同时既有不可变借用又有可变借用

如果违反，程序会panic。

### （3）常用API

`RefCell::new()` ：创建一个新的 `RefCell<T>`

`borrow()` ：不可变借用，返回的类型不是普通 `&T` ，而是 `Ref<T>` ，但是用起来很像不可变引用。

`borrow_mut()` ：可变借用，返回的类型不是普通 `&mut T` ，而是 `RefMut<T>` ，但像 `&mut T` 一样使用。

`into_inner()` ：消费 `RefCell<T>` ，直接拿到内部值

示例：

展开

自动换行

use std::cell::RefCell;

struct Logger {

logs: RefCell<Vec<String>>,

}

impl Logger {

fn log(&self, msg: String) {

self.logs.borrow\_mut().push(msg);

}

fn show(&self) {

for log in self.logs.borrow().iter() {

println!("{}", log);

}

}

}

### （4）Rc + RefCell的组合使用

在Rust中，一个非常常见的组合是：

`Rc<T>` ：允许同一个值有多个所有者

`RefCell` ：提供内部可变性，外部值不可变，但其内部字段可变。

将它们组合在一起就是：多个地方共同拥有同一份数据，且允许在运行时对值进行修改。

示例：

展开

自动换行

use std::cell::RefCell;

use std::rc::Rc;

fn main() {

let s = Rc::new(RefCell::new(String::from("Hello")));

let s2 = Rc::clone(&s);

s.borrow\_mut().push\_str(", world");

println!("{}", s.borrow());

println!("{}", s2.borrow());

}
