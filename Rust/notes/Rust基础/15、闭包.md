## 一、问题引入

如果想对一个整数向量排序，那么可以直接使用 `sort()` 方法：

展开

自动换行

let mut numbers = vec!\[3, 1, 4, 2\];

numbers.sort();

但是，如果想对一个自定义类型排序，就不能直接使用 `sort()` 方法了：

展开

自动换行

struct Person {

name: String,

age: u8,

}

fn main() {

let mut people = vec!\[

Person { name: "Tom".to\_string(), age: 25 },

Person { name: "Bob".to\_string(), age: 30 },

Person { name: "Alice".to\_string(), age: 27 }

\];

people.sort();

}

报错：

![[Image 68.png]]

原因很简单，因为 `Person` 没有实现 `Ord` 特征，因为 `sort` 方法存在限界： `T: Ord` 。

展开

自动换行

impl<T> \[T\]

pub fn sort(&mut self)

where

T: Ord

一种方法是使用派生属性宏为 `Person` 自动实现 `Ord` ：

展开

自动换行

#\[derive(Debug, Ord, PartialOrd, Eq, PartialEq)\]

struct Person {

name: String,

age: u8,

}

**但自动实现会按照结构体字段的声明顺序进行字典序比较，即先比较name的大小，如果name完全一致，再通过age比较大小** 。

如果只是想根据age字段排序，那么上述方法就不可行了。

此时可以通过 `sort_by_key` 方法，其思想是把元素映射成一个可以排序的key，然后按key排序，其原型为：

展开

自动换行

impl<T> \[T\]

pub sort\_by\_key<K, F>(&mut self, mut f: F)

where

F: FnMut(&T) -> K,

K: Ord,

示例：

展开

自动换行

fn age\_descending(p: &Person) -> i8 {

\-p.age

}

#\[derive(Debug)\]

struct Person {

name: String,

age: i8,

}

fn main() {

let mut people = vec!\[

Person { name: "Tom".to\_string(), age: 25 },

Person { name: "Bob".to\_string(), age: 30 },

Person { name: "Alice".to\_string(), age: 27 }

\];

people.sort\_by\_key(age\_descending);

println!("{:#?}", people);

}

这里定义的 `age_descending` 函数是按照 `age` 字段排序，通过取负来实现降序排序。

这个辅助函数固然可行，但可以更加简洁：

展开

自动换行

people.sort\_by\_key(|p| -p.age );

这里的 `|p| -p.age` 就是闭包，它是一种匿名函数表达式，接收一个参数p并返回-p.age。

## 二、闭包的定义

在Rust中， **闭包（Closure）是一种匿名函数** ，它类似于C++中的lambda表达式。闭包可以赋值给变量，也可以作为参数传递给函数。

**与普通函数不同的是，闭包可以捕获其定义环境中的变量（即捕获调用者作用域中的值）** ，因此闭包能够访问外部作用域中的数据。

闭包使用 `||` 定义参数列表， `{}` 中定义函数体。

基本语法：

展开

自动换行

|param1, param2,..| {

语句1;

语句2;

返回表达式

}

如果只有一个表达式，可以省略 `{}` ，简写为：

展开

自动换行

|param1, param2,..| 表达式

示例：

展开

自动换行

// 将闭包赋值给变量后，变量可以像函数那样使用

let closure = |x| x + 1;

assert\_eq!(closure(2), 3);

说明：

`|x|` 表示闭包参数

`x + 1` 表示闭包体

`closure(2)` 的调用方式与普通函数类似

**在Rust中，函数的参数类型和返回值类型需要显式注解** ，这是因为函数往往作为API暴露给其他代码使用，明确的类型有助于用户理解和使用函数。

而 **闭包通常用于局部场景，代码较短且上下文明确** ，因此Rust编译期往往能根据使用情况自动推断参数和返回值的类型，所以 **闭包不强制要求类型注解** 。

但也可以显式为闭包增加类型注解。

示例：

展开

自动换行

fn add\_one\_v1 (x: i32) -> i32 { x + 1 }

let add\_one\_v2 = |x: i32| -> i32 { x + 1 }

let add\_one\_v3 = | x | { x + 1 }

let add\_one\_v4 = | x | x + 1

说明：

`add_one_v1` 是普通函数，需要显式类型

`add_one_v2` 是带完整类型标注的闭包

`add_one_v3` 省略参数类型和返回类型

`add_one_v4` 同时省略 `{}` 和类型

尽管定义闭包是可以不写类型注解，但闭包的类型必须能够推断出来。如果闭包没有被调用，编译期就无法推断类型，会产生编译错误。

此外， **闭包的类型参数一旦被推断出来，就会固定下来，之后不能再用其他类型调用该闭包** 。

示例：

展开

自动换行

let closure = |x| x;

// 第一次调用时，编译器推断 x 的类型为 String

let s = closure(String::from("Hello"));

// 错误：x 的类型已经被推断为 String

let n = closure(1);

## 三、闭包捕获环境

## 1、捕获环境值

**闭包和普通函数的一个重大区别就在于：闭包可以捕获其定义所在作用域中的值。**

示例：

展开

自动换行

let x = 4;

let equal\_to\_x = |z| z == x;

let y = 4;

assert!(equal\_to\_x(y))

上述代码中， `x` 并不是闭包的参数，但是闭包内部仍然可以使用 `x` ，这是因为闭包定义时可以访问其所在作用域的变量。

与普通函数相比，闭包之所以能记住其环境，是因为闭包在底层会保存它所捕获的环境数据。 **因此，闭包通常会比普通函数多一些额外的存储开销** 。不过，Rust编译器会尽量只捕获真正需要的内容，而不是整个作用域。

## 2、捕获环境的三种方式

闭包可以通过三种方式捕获环境中的值，分别是：

**不可变借用**

**可变借用**

**获取所有权**

这与函数的参数传递方式有些类似。 **编译器会根据闭包体中如何使用被捕获的变量，自动决定采用哪一种方式捕获** ：

能用不可变借用，就不会用可变借用

能用借用，就不会直接按值移动获取所有权

## 3、不可变借用

示例：

展开

自动换行

fn main() {

let s = String::from("hello");

println!("Before defining closure: {}", s);

let closure = || {

println!("In closure: {}", s);

};

println!("Before calling closure: {}", s);

closure();

println!("After calling closure: {}", s);

}

结果：

![[Image 69.png]]

分析：

闭包体中只是读取并打印 `s` ，没有修改它，也没有消耗它的所有权，因此编译器会让闭包以不可变借用的方式捕获 `s` 。所以，闭包可以访问 `s` ，在定义闭包之后，仍然可以使用 `s` 。

在底层，可以近似理解为编译器生成了一个匿名类型的值，这个值中保存了它所需要捕获的环境数据。在这个例子中，就是 `s` 的不可变引用。

正因为只是不可变借用，所以这句代码仍然合法：

展开

自动换行

println!("Before calling closure: {}", s);

因此它只是再次对 `s` 进行不可变借用，而Rust允许同时存在多个不可变引用。

## 4、可变借用

示例：

展开

自动换行

fn main() {

let mut s = String::from("hello");

println!("Before defining closure: {}", s);

let mut closure = || { // closure 需要声明为 mut

s.push\_str(" world");

println!("In closure: {}", s);

};

// println!("Before calling closure: {}", s); // 报错

closure();

println!("After calling closure: {}", s);

}

结果：

![[Image 70.png]]

注意： **如果闭包体内要修改捕获到的环境值，那么闭包通常以可变借用的方式捕获该变量，此时用来绑定闭包的变量也要声明为** `mut` **，因为调用该闭包可能会改变闭包内部状态** 。

分析：

`s.push_str(" world");`说明闭包需要对 `s` 进行可变访问，因此编译器会让闭包以 **可变借用** 的方式捕获 `s` 。

因此定义闭包后，其底层持有了对s的可变借用，根据借用规则，同一时间不允许存在其他借用，所以在调用闭包前，下面的代码会出错：

展开

自动换行

println!("Before calling closure: {}", s);

不过，在闭包调用结束之后，如果可变借用的作用范围已经结束，就可以继续使用s，所以下面代码不会报错：

展开

自动换行

println!("After calling closure: {}", s);

## 5、获取所有权

示例：

展开

自动换行

fn main() {

let s = String::from("hello");

println!("Before defining closure: {}", s);

let closure = || {

println!("In closure: {}", s);

drop(s); // 释放 s

};

// println!("Before calling closure: {}", s); // 报错

closure();

// println!("After calling closure: {}", s); // 报错

}

分析：

闭包体中调用了 `drop(s)` ，这会消耗 `s` 的所有权。因此，编译器必须让闭包以获取所有权的方式捕获 `s` 。一旦闭包获取了 `s` 的所有权，那么从闭包定义开始，外部的就 `s` 就已经不能再使用了，所以这两句会报错：

展开

自动换行

println!("Before calling closure: {}", s);

println!("After calling closure: {}", s);

## 6、move关键字

使用 `move` 关键字可以强制闭包获取其环境中所使用变量的所有权。具体语法就是在 `||` 前加上 `move` 。

示例：

展开

自动换行

fn main() {

let s = String::from("hello");

println!("Before defining closure: {}", s);

// move强制移动所有权

let closure = move || {

println!("In closure: {}", s);

};

// println!("Before calling closure: {}", s); // 报错

closure();

// println!("After calling closure: {}", s); // 报错

}

分析：

在 `move` 闭包中， `s` 会被移动进闭包，因此在闭包定义之后，外部变量 `s` 就不能再使用了。

## 7、闭包底层原理

当定义一个闭包时，编译器会为其生成一个匿名结构体类型，而闭包本身是该类型的一个实例。其中包括两部分内容：

用于保存闭包捕获的环境值

用于执行闭包逻辑的调用能力

可以近似理解为这个匿名类型会保存被捕获的外部变量，编译器同时为该类型实现调用相关的trait，这样它就能像函数那样调用。

编译器可能为其实现以下trait：

`FnOnce` ： **闭包会消耗捕获的值，因此该类型闭包只能调用一次**

`FnMut` ： **闭包不会消耗捕获的值，但会修改捕获的值，因此调用时需要可变访问**

`Fn` ： **闭包既不消耗值，也不修改捕获值，因此可以多次调用且不需要可变访问**

需要注意的是：

所有闭包都至少实现了 `FnOnce`

如果闭包没有移动走捕获值，会进一步实现 `FnMut`

如果闭包既不修改也不移动捕获值，会进一步实现 `Fn`

因此， `Fn` 是限制最少、能力最强的闭包类型，而 `FnOnce` 是最基础的调用形式。

## 8、疑难释义

闭包 **以何种方式捕获变量** 与 **闭包实现了什么trait** 是相关但不完全相同的两个问题。

例如，一个 `move` 闭包是按所有权捕获变量，但它不一定只能调用一次，是否只能调用一次，要看闭包体是否把捕获的值消耗掉了。

示例：

展开

自动换行

fn main() {

let s = String::from("hello");

let closure = move || {

println!("{}", s);

};

closure();

closure();

}

这段代码是可以通过编译的，闭包可以多次调用。因为虽然 `s` 被 move 进了闭包，但闭包体只是借用它打印，并没有再次将其move出去。

VS Code IDE可以显式出该闭包实现了什么Trait：

![[Image 71.png]]

但如果在闭包体末尾加了 `drop(s)` ，那么s又被move进了 `drop` ，此时闭包只能调用一次，实现的 `FnOnce` 特征：

![[Image 72.png]]

## 四、闭包的类型

## 1、函数与闭包的类型

在Rust中， **函数和闭包都可以像普通值一样使用** ，因此它们有着各自的类型。

例如，先看一个简单的普通函数：

展开

自动换行

fn add\_one(x: i32) -> i32 {

x + 1

}

这个函数接收一个 `i32` 参数，返回一个 `i32` 。因此，其类型为：

展开

自动换行

fn(i32) -> i32

函数可以赋值给变量：

展开

自动换行

fn add\_one(x: i32) -> i32 {

x + 1

}

fn main() {

let f = add\_one;

let res = f(1);

println!("{}", res);

}

这里的 `f` 是一个变量，类型就是 `f(i32) - > i32` 。

函数可以作为参数传递给另一个函数：

展开

自动换行

fn apply(f: fn(i32) -> i32, arg: i32) -> i32 {

f(arg)

}

fn add\_one(x: i32) -> i32 {

x + 1

}

fn main() {

// 将\`add\_one\`作为参数传递给函数

let res = apply(add\_one, 5);

println!("{}", res);

}

对于闭包来说，虽然它看起来和函数很像，也确实可以调用：

展开

自动换行

let add\_one = |x| x + 1;

let res = add\_one(1);

println!("{}", res);

但需要注意 `add_one` 的类型不是 `fn(i32) -> i32` ，而是由一个 **编译器自动生成、独一无二的匿名类型** ，这个匿名类型通常不能直接写出来。

不是所有的闭包都可以传给以 `fn` 作为参数的函数：

没有捕获环境值的闭包可以适配 `fn` 参数，它会自动退化/强制转换为 `fn`

捕获环境值的闭包，不能直接当作 `fn` 传

示例1：

展开

自动换行

fn apply(f: fn(i32) -> i32, arg: i32) -> i32 {

f(arg)

}

fn main() {

let add\_one = |x| x + 1;

let res = apply(add\_one, 1); // 闭包没有捕获环境值，强制转换为fn(i32) -> i32

println!("{}", res);

}

示例2：

展开

自动换行

fn apply(f: fn(i32) -> i32, arg: i32) -> i32 {

f(arg)

}

fn main() {

let base = 1;

let add\_one = |x| x + base;

let res = apply(add\_one, 1); // 闭包捕获了环境值，不呢当作fn(i32) -> i32类型

println!("{}", res);

}

尽管编译器为闭包生成的匿名类型无法直接显示，但是由于每个闭包都实现了对应的 `FnOnce` 、 `FnMut` 、 `Fn` 特征，如果想让一个函数既能接收普通函数，也能接收闭包，通常写成泛型：

展开

自动换行

fn apply<F>(f: F, arg: i32) -> i32

where

F: Fn(i32) -> i32

{

f(arg)

}

或者：

展开

自动换行

fn apply(f: impl Fn(i32) -> i32, arg: i32) -> i32

{

f(arg)

}

## 2、闭包实现的三种特征

闭包除了接受参数，返回结果外，还可以捕获其定义位置周围环境的值。 **闭包最终会实现哪些调用Trait** ，不应只看“它是如何捕获值的”，更准确来说，应看“ **闭包体在调用时如何使用捕获到的这些值** ”。

**编译器会根据闭包体对环境值的使用方式，自动、渐进地为闭包实现一个、两个甚至全部三个调用Trait（** `FnOnce` **、** `FnMut` **、** `Fn` **）** ：

所有闭包都至少实现 `FnOnce` 特征

如果闭包在调用时不会把捕获到的值从闭包内部移走，那么它还可以实现 `FnMut`

如果闭包在调用时既不移走捕获值，也不修改捕获的值，那么它还可以进一步实现 `Fn`

三者的关系如下：

实现了 `Fn` Trait的闭包，也一定实现了 `FnMut` 和 `FnOnce` 特征

实现了 `FnMut` Trait的闭包，也一定实现了 `FnOnce` 特征

反过来则不成立

因此， `Fn` 的要求最严格， `FnOnce` 的要求最宽松。

一个容易混淆的点是： `move` 关键字只影响捕获方式，并不直接决定闭包一定只是 `FnOnce` 。一个按值捕获环境的闭包，如果闭包内不移动值的所有权，它仍然吧可以实现 `FnMut` 或 `Fn` 。

### （1）FnOnce Trait

所有闭包都实现 `FnOnce` 特征，因此 `FnOnce` 表示这个闭包至少可以被调用一次。

但是，如果在闭包调用时会把捕获的值从闭包内部移走，那么这个闭包就只能调用一次，因此它只实现 `FnOnce` 特好着呢个，而不能实现 `FnMut` 和 `Fn` 。

`FnOnce` 的对应的调用方式可以理解为： **调用时会消耗闭包本身** 。

示例：

展开

自动换行

fn fnonce\_test<F>(f: F)

where

F: FnOnce()

{

f();

// f(); 错误，因为f在第一次调用时已经被消耗

}

fn main() {

// 可以传入实现\`FnOnce\`的闭包

let s1 = String::from("hello");

fnonce\_test(move || {

println!("{}", s1);

drop(s1);

});

// 可以传入实现\`FnMut\`的闭包

let mut s2 = String::from("hello");

fnonce\_test(|| {

println!("{}", s2);

s2.push\_str(", world");

});

// 可以传入实现\`Fn\`的闭包

fnonce\_test(|| {

println!("{}", s2);

});

}

以 `FnOnce` 特征为参数的函数可以传入实现了 `FnOnce` 特征、实现了 `FnMut` 特征或者实现了 `Fn` 特征的闭包。

### （2）FnMut Trait

实现 `FnMut` 特征的闭包在调用时需要可变的使用自己，也就是说，闭包在执行过程中可能会修改它内部持有的状态。

很多时候，这表现为闭包以可变借用的方式捕获环境中的值；但也可能是闭包按值捕获了某个值，然后在调用时修改这个值。因此，“FnMut”强调的是“调用时会发生可变使用”，而不只是“捕获时一定是可变借用”

示例：

展开

自动换行

fn fnmut\_test<F>(mut f: F)

where

F: FnMut()

{

f();

f(); // 可以多次调用

}

fn main() {

// 可以传入实现\`FnMut\`的闭包

let mut s1 = String::from("hello");

fnmut\_test(|| {

s1.push\_str(", world");

println!("{}", s1);

});

// 可以传入实现\`Fn\`的闭包

let s2 = String::from("hello");

fnmut\_test(|| {

println!("{}", s2);

});

}

对于函数参数为实现了 `FnMut` Trait 的闭包，该函数可以接收实现 `Fn` 或 `FnMut` Trait 的闭包，但不能接收只实现 `FnOnce` Trait 的闭包。本质上是因为实现了 `Fn` Trait 的闭包也实现了 `FnMut` Trait。

### （3）Fn Trait

实现 `Fn` 特征的闭包在调用时只需要不可变地使用自己，也就是说，闭包在执行过程中不会修改其内部状态，也不会把捕获到的值从闭包内部移出。

很多时候，这表现为闭包以不可变借用的方式捕获环境中的值；但也要注意，这并不是绝对的。一个闭包即使按值捕获了环境值，只要调用时只是只读使用这些值，它仍然可以实现 Fn Trait。

示例：

展开

自动换行

fn fn\_test<F>(f: F)

where

F: Fn()

{

f();

f(); // 可以多次调用

}

fn main() {

// 可以传入实现\`Fn\`的闭包

let s1 = String::from("hello");

fn\_test(|| println!("{}",s1))

}

对于函数参数为实现了 Fn Trait 的闭包，该函数只能接收实现 Fn Trait 的闭包。因为 Fn 的要求最严格，它要求闭包在调用时既不修改内部状态，也不移出捕获值。

## 五、对闭包的Copy和Clone

闭包是表示包含它们捕获的值（对于 `move` 闭包）或者对值的引用（对于非 `move` 闭包）的结构体。

因此，闭包的 `Copy` 和 `Clone` 规则与常规结构体的规则一样：

一个不修改变量的非 `move` 闭包只持有共享引用，这些引用既能 `Clone` 也能 `Copy` ，所以闭包也能 `Clone` 和 `Copy` 。

展开

自动换行

let x = 1;

let add\_x = |y| y + x;

let add\_x\_copy = add\_x;

let z = add\_x\_copy(add\_x(1));

println!("{}", z);

一个会修改值的非 `move` 闭包持有可变引用，可变引用既不能 `Copy` 也不能 `Clone` ，所以闭包也不能 `Copy` 和 `Clone` 。

展开

自动换行

let mut x = 1;

let add\_to\_x = |y| {

x += y;

x

};

let mut add\_to\_x\_copy = add\_to\_x;

// let z = add\_to\_x(1); 错误，add\_to\_x已经发生了移动

let z = add\_to\_x\_copy(1);

println!("{}", z);

println!("{}", x);

注意：由于add\_to\_x定义时修改了x，因此按可变引用捕获了， `let mut add_to_x_copy = add_to_x;`发生了所有权的注意， `add_to_x` 失效。

对于 `move` 闭包，如果其捕获的内容都能 `Copy` ，那么闭包就能 `Copy` ；如果其捕获的内容都能 `Clone` ，那么闭包就能 `Clone` 。

展开

自动换行

let mut greeting = String::from("hello ");

let greet = move |name| {

greeting.push\_str(name);

println!("{}", greeting)

};

greet.clone()("Tom");

greet.clone()("Jerry");

`.clone()(...) ` 语法有点儿奇怪，其实它只是表示克隆此闭包并调用其克隆体。

## 六、回调

很多库都会在其API中使用回调函数。 **所谓回调，就是由调用者提供一段函数或闭包，库先把它存起来，等到合适的时机再调用** 。

下面这个函数接收一个回调，并在内部调用它：

展开

自动换行

fn run\_task<F>(name: &str, callback: F) -> String

where

F: Fn(&str) -> String

{

callback(name)

}

fn main() {

let greeting = run\_task("Tom", |name| format!("Hello, {name}"));

println!("{greeting}");

}

这里可以这样理解：

`run_task` 并不知道 `format!("Hello, {name}"` 的逻辑

这段逻辑是调用者以闭包的形式传递进来的

`run_task` 只是负责在合适的时候调用 `callback(name)`

这就是回调最基本的样子。

上面的例子里，回调是 **传进来立即调用** 的。但很多库并不是立即调用，而是先保存起来，等以后再调用。

例如下面的一个命令分发器：

展开

自动换行

use std::collections::HashMap;

// 命令分发器

commands: HashMap<String, C>,

}

impl<C> CommandDispatcher<C>

where C: Fn(&str) -> String

{

// 创建一个空的分发器

CommandDispatcher {

commands: HashMap::new(),

}

}

// 增加一条命令

有了增加命令的方法后，还需要一个执行命令的方法：

展开

自动换行

use std::collections::HashMap;

type BoxedCallback = Box<dyn Fn(&str) -> String>;

// 命令分发器

struct CommandDispatcher {

commands: HashMap<String, BoxedCallback>,

}

impl CommandDispatcher

{

// 创建一个空的分发器

fn new() -> CommandDispatcher {

CommandDispatcher {

commands: HashMap::new(),

}

}

// 增加一条命令

fn add\_command<C>(&mut self, command: &str, callback: C)

where

C: Fn(&str) -> String + 'static

{

self.commands.insert(command.to\_string(), Box::new(callback));

}

fn run(&self, command: &str, arg: &str) -> String {

match self.commands.get(command) {

Some(callback) => callback(arg),

None => format!("未知的命令: {}", command),

}

}

}

到这里，回调的整个流程就完整了：

调用者注册回调

分发器把回调保存起来

稍后通过命令的名字找到回调

最终调用它

由于这里使用了 `Box` 以及特征对象，因此存在堆分配和动态调用，存在一定的开销。对于这里的不捕获环境的闭包，还可以进一步使用函数指针的形式： **因为不捕获环境的闭包可以自动退化为函数指针** 。
