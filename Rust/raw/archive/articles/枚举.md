
枚举（enum）是一种自定义数据类型，用于表示**一组有限的可能状态**。

在 Rust 中，使用`enum`关键字定义枚举。

枚举中的每一种可能状态称为一个**变体**（variant）。

示例：

```rust
enum Color {
	Red,
	Green,
	Blue,
}
```

这里定义了一个`Color`类型，它有三个可能的值：

- `Color::Red`
- `Color::Green`
- `Color::Blue`

创建枚举值时，使用`枚举名::变体名`形式。

示例：

```rust
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    let c = Color::Red;
}
```

枚举适合表达*一个值只能是几种情况之一*的场景。例如：

- 颜色只能是红、绿、蓝之一
- 网络连接状态只能是连接、断开、超时之一
- 函数结果只能是成功或失败之一
- 一个值可能存在，也可能不存在

相比C语言中的枚举，Rust的枚举更强大。**Rust枚举的变体不仅可以表示状态，还可以携带数据**。因此，可以先把 Rust 枚举分为两类理解：

- C风格枚举
- 带数据枚举

# 二、C风格枚举

## 1、定义C风格枚举

C风格枚举的变体不携带额外数据，只表示几个固定状态。

示例：

```rust
#[derive(Debug)]
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    let c = Color::Red;

    println!("{:?}", c);
}
```

输出：

```shell
Red
```

这里 `Color` 有三个变体：`Red`、`Green`、`Blue`。

每个变体本身就是一个可能的枚举值。

为了能够使用 `{:?}` 打印枚举值，这里使用了 `#[derive(Debug)]`。`Debug` 主要用于开发调试输出。

再例如标准库中的`std::cmp::Ordering`枚举：

```rust
enum Ordering {
	Less = -1,
	Equal = 0,
	Greater = 1,
}
```

对于没有携带数据的枚举变体，Rust允许显式指定判别值，如这里的`Less = -1`等。如果不显式指定，默认从`0`开始递增。例如前面的`Color`默认相当于：

```rust
enum Color {
    Red = 0,
    Green = 1,
    Blue = 2,
}
```

可以将它们转换成整数：

```rust
enum Color {  
    Red,  
    Green,  
    Blue  
}  
  
fn main() {  
    println!("{}", Color::Red as usize);  
    println!("{}", Color::Green as usize);  
    println!("{}", Color::Blue as usize);  
}
```

输出：

```shell
0
1
2
```

但是，Rust不允许直接把任意整数转换成枚举值。原因是：一个整数不一定对应某个合法的枚举变体。

例如 `Color` 只有 `0`、`1`、`2` 三种有效情况，但 `100` 并不是合法的 `Color`。

如果需要从整数转换为枚举，通常应该手动检查并返回`Option`：

```rust
#[derive(Debug, PartialEq, Eq)]
enum Color {
    Red,
    Green,
    Blue,
}

fn color_from_u8(x: u8) -> Option<Color> {
    match x {
        0 => Some(Color::Red),
        1 => Some(Color::Green),
        2 => Some(Color::Blue),
        _ => None,
    }
}

fn main() {
    assert_eq!(color_from_u8(1), Some(Color::Green));

    assert_eq!(color_from_u8(4), None);
}
```

这里用到了 `match` 模式匹配，[[8、模式匹配|模式匹配]]会单独介绍。

当前只需要知道：从整数转换为枚举时，应该处理“不合法整数”的情况。


## 2、枚举变体的使用

枚举变体通常使用`枚举名::变体名`的形式访问。

示例：

```rust
use std::cmp::Ordering;  
  
  
fn compare(n: i32, m: i32) -> Ordering {  
    if n < m {  
        Ordering::Less  
    } else if n > m {  
        Ordering::Greater  
    } else {  
        Ordering::Equal  
    }  
}
```

如果想直接使用变体名，可以将枚举的各个变体一并导入（关于导入详见[[9、crate与模块|crate与模块]]）。


```rust
use std::cmp::Ordering::{self, *};  
  
  
fn compare(n: i32, m: i32) -> Ordering {  
    if n < m {  
        Less  
    } else if n > m {  
        Greater  
    } else {  
        Equal  
    }  
}
```

但这样写语义不明确，通常认为不导入变体可读性更好，除非是常用的几种枚举变体（下面即将介绍的`Option`和`Result`变体在预导入模块中，可直接使用变体名）。


## 3、枚举的所有权规则

结构体和枚举默认不是`Copy`类型（关于Copy详见[[3、所有权|所有权]]）。

如果直接赋值，默认会发生所有权移动。

示例：

```rust
#[derive(Debug)]
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    let c1 = Color::Red;

    let c2 = c1;

    // println!("{:?}", c1); // 错误，c1已经被移动

    println!("{:?}", c2);
}
```

这里 `c1` 被移动给了 `c2`，所以后面不能再使用 `c1`。

如果枚举的所有变体都不包含非`Copy`数据，可以通过`#[derive(Copy,Clone)]`让它成为`Copy`类型：

```rust
#[derive(Debug, Copy, Clone)]
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    let c1 = Color::Red;

    let c2 = c1;

    println!("{:?}", c1);

    println!("{:?}", c2);
}
```



**和结构体类似，枚举也可以有方法** 。

示例：
```rust
#[derive(Debug, Copy, Clone)]

enum Color {

Red,

Green,

Blue,

}

impl Color {

fn as_str(&self) -> &'static str {

match self {

Color::Red => "red",

Color::Green => "green",

Color::Blue => "blue"

}

}

}

fn main() {

let c = Color::Red;

println!("{:?} -> {}", c, c.as_str());

}
```
这里用到的 **match 模式匹配** 将在下一节中介绍。`match self` 自动对 `self` 进行了解引用，因为 `self` 是 `&Color` 类型。


# 三、带数据的枚举
## 1、带数据枚举的基本概念

Rust 枚举不仅可以表示状态，还可以携带数据。

示例：

```rust
#[derive(Debug)]
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    Color(i32, i32, i32),
}

fn main() {
    let m1 = Message::Quit;

    let m2 = Message::Move { x: 10, y: 20 };

    let m3 = Message::Write(String::from("hello"));

    let m4 = Message::Color(255, 255, 255);

    println!("{:?}", m1);

    println!("{:?}", m2);

    println!("{:?}", m3);

    println!("{:?}", m4);
}
```

这里 `Message` 有四个变体：

- `Quit` 是单元型变体，不携带数据
- `Move { x: i32, y: i32 }` 是结构体型变体
- `Write(String)` 是元组型变体
- `Color(i32, i32, i32)` 也是元组型变体

带数据枚举的重点在于：**不同变体可以携带不同类型、不同数量的数据**。

这也是 Rust 枚举比 C 风格枚举更强大的地方。


## 2、单元型变体

单元型变体也就是前面讨论的C风格枚举的变体，它不携带任何数据，只表示一种状态。

示例：

```rust
enum NetworkEvent {
    Timeout,
}
```

这里 `Timeout` 只表示“超时”这个状态，不需要额外数据。

创建时直接写：

```rust
let event = NetworkEvent::Timeout;
```


## 3、元组型变体

元组型变体携带的数据没有字段名，只有位置。

示例：

```rust
enum Message {  
	Write(String),  
	Color(i32, i32, i32),  
}
```

`Write(String)` 表示这个变体携带一个 `String`。`Color(i32, i32, i32)` 表示这个变体携带三个 `i32`。元组型变体适合数据含义比较简单，或者通过变体名已经能表达清楚的场景。


## 4、结构体型变体

结构体型变体携带的数据有字段名。

示例：

```rust
enum Message {  
	Move { x: i32, y: i32 },  
}
```

`Move { x: i32, y: i32 }` 携带两个字段：`x` 和 `y`。

结构体型变体适合字段含义需要明确表达的场景。


## 5、变体携带枚举

枚举变体携带的数据也可以是另一个枚举类型。

示例：

```rust
#[derive(Debug)]
enum IpAddrKind {
    V4,
    V6,
}

#[derive(Debug)]
enum NetworkEvent {
    Connect(IpAddrKind, String),
    Disconnect(IpAddrKind),
    Timeout,
}

fn main() {
    let event1 = NetworkEvent::Connect(
        IpAddrKind::V4,
        String::from("127.0.0.1"),
    );

    let event2 = NetworkEvent::Timeout;

    println!("{:?}", event1);

    println!("{:?}", event2);
}
```

这里 `NetworkEvent::Connect` 携带了两个数据：

- 一个 `IpAddrKind`
- 一个 `String`

这种写法可以把多个相关状态组合成更准确的业务模型。

# 四、枚举方法

## 1、定义方法

和[[6、结构体|结构体]]一样，枚举也可以通过`impl`块定义方法。

示例：

```rust
#[derive(Debug, Copy, Clone)]
enum Color {
    Red,
    Green,
    Blue,
}

impl Color {
    fn as_str(&self) -> &'static str {
        match self {
            Color::Red => "red",
            Color::Green => "green",
            Color::Blue => "blue",
        }
    }
}

fn main() {
    let c = Color::Red;

    println!("{:?} -> {}", c, c.as_str());
}
```

这里 `as_str` 是 `Color` 的方法。

它接收 `&self`，表示只读取当前枚举值，不取得所有权，也不修改它。

`match` 会根据当前枚举值是哪一个变体，返回不同的字符串。`match` 模式匹配会在下一篇笔记中详细介绍，这里只需要先知道：它常用于处理枚举的不同变体。

> `match self` 自动解引用，等价于 `match *self`

## 2、方法中的self参数

枚举方法和结构体方法一样，也可以使用 `self`、`&self`、`&mut self`：

- `self` 表示取得当前枚举值的所有权
- `&self`表示共享借用当前枚举值
- `&mut self`表示可变借用当前枚举值

示例：

```rust
#[derive(Debug)]
enum State {
    Open,
    Closed,
}

impl State {
    fn close(&mut self) {
        *self = State::Closed;
    }
}

fn main() {
    let mut state = State::Open;

    state.close();

    println!("{:?}", state);
}
```

这里 `close(&mut self)` 通过可变引用修改当前枚举值，把 `State::Open` 改成 `State::Closed`。

# 五、Option和Result枚举

Rust标准库中有两个非常常用的枚举：`Option<T>`和`Result<T,E>`。它们在 Rust 代码中几乎无处不在。

这里的`T`和`E`是泛型参数，有关泛型参考[[10、泛型和特征|泛型和特征]]，当前只需要先把它们理解为“某个具体类型的占位符”。

由于`Option`和`Result`过于常用，它们的变体通常可以直接写：

- `Some(...)`
- `None`
- `Ok(...)`
- `Err(...)`

不需要写成 `Option::Some(...)` 或 `Result::Ok(...)`。

## 1、Option

`Option<T>`用来表示**可能有值，也可能无值**的情况。

它的定义大致如下：

```rust
enum Option<T> {
	Some(T),
	None,
}
```

含义是：

- `Some(v)`：有值，值的类型是 `T`
- `None`：没有值

Rust 没有 `null`。如果一个值可能不存在，通常使用 `Option<T>` 表示。

示例：

```rust
fn main() {
    let x = Some(1);

    let y: Option<i32> = None;

    println!("{:?}", x);

    println!("{:?}", y);
}
```

这里 `x` 是 `Some(1)`，表示有一个 `i32` 值。`y` 是 `None`，表示没有 `i32` 值。

**需要注意的是，如果直接写`None`，编译器可能无法推断出`T`是什么类型，所以通常需要显式标注类型**。

## 2、Result

`Result<T, E>` 用来表示**操作可能成功，也可能失败**的情况。

它的定义大致如下：

```rust
enum Result<T, E> {
	Ok(T),
	Err(E),
}
```

含义：

- `Ok(v)`：操作成功，成功值的类型是 `T`
- `Err(e)`：操作失败，错误值的类型是 `E`

`Result<T, E>` 常作为**函数返回类型**，用于错误处理。

示例：

```rust
fn divide(n: i32, m: i32) -> Result<i32, String> {
    if m == 0 {
        Err(String::from("cannot divide by zero"))
    } else {
        Ok(n / m)
    }
}

fn main() {
    let r1 = divide(10, 2);

    let r2 = divide(10, 0);

    println!("{:?}", r1);

    println!("{:?}", r2);
}
```

这里：

`divide(10, 2)` 返回 `Ok(5)`。

`divide(10, 0)` 返回 `Err(...)`。

后续学习[[11、错误处理|错误处理]]时，会更系统地介绍 `Result<T, E>`。

# 六、用枚举表示富数据结构

枚举非常适合表示**有限种类，但每种种类携带不同数据**的结构。

例如 JSON 值只有几种固定类型：

- null
- boolean
- number
- string
- array
- object

可以使用 Rust 枚举来建模。

示例：

```rust
use std::collections::HashMap;

enum Json {
	Null,
	Boolean(bool),
	Number(f64),
	String(String),
	Array(Vec<Json>),
	Object(HashMap<String, Json>),
}
```

这里每个变体对应一种 JSON 值：

- `Json::Null` 表示空值
- `Json::Boolean(bool)` 表示布尔值
- `Json::Number(f64)` 表示数字
- `Json::String(String)` 表示字符串
- `Json::Array(Vec<Json>)` 表示数组
- `Json::Object(HashMap<String, Json>)` 表示对象

这个例子展示了枚举的一个重要用途：**用类型系统表达业务数据的所有可能形态**。

JSON 数组中的每个元素仍然是 `Json`，所以可以嵌套。

JSON 对象中的每个值也仍然是 `Json`，所以也可以嵌套。

例如，一个 JSON 数组可以包含数字、字符串、布尔值，甚至另一个数组或对象。

需要注意的是，递归数据结构在 Rust 中有更细的内存大小问题，比如某些情况下需要使用 `Box`。这个问题和智能指针有关，后续学习[[10、智能指针|智能指针]]时再展开。



# 七、枚举的内存表示
## 1、C风格枚举的内存表示

C风格枚举在内存中通常会用一个**整数标签**表示当前是哪一个变体。

示例：

```rust
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    println!("{}", std::mem::size_of::<Color>());
}
```

输出结果：

```shell
1
```

对于这种只有三个变体的枚举，编译器通常只需要很小的空间就能表示它。默认情况下，Rust会使用**可以容纳它们的最小内置整数类型**来存储C风格枚举，最适合的就是1字节。

如果需要指定底层整数表示，可以使用`#[repr(...)]`。

示例：

```rust
#[repr(u32)]
enum Color {
    Red,
    Green,
    Blue,
}

fn main() {
    println!("{}", std::mem::size_of::<Color>());
}
```

输出结果：

```shell
4
```

这里 `#[repr(u32)]` 表示让这个枚举使用 `u32` 作为底层表示。

普通 Rust 代码里，一般不需要关心这个。只有在 FFI、二进制格式、底层内存交互等场景中，才需要明确控制表示方式。


## 2、带数据枚举的内存表示

带数据枚举可以理解为由两部分组成：

- 一个标签，用来记录当前是哪一个变体
- 一块数据区域，用来存储当前变体携带的数据

例如：

```rust
enum Message {
    Quit,
    Write(String),
    Color(i32, i32, i32),
}
```

对于一个 `Message` 值，Rust 需要知道它当前到底是：

- `Message::Quit`
- `Message::Write(...)`
- `Message::Color(...)`

所以通常需要某种标签来区分变体。

同时，枚举还要有足够空间存放变体携带的数据。

不过，Rust 默认不承诺枚举在内存中的具体布局。编译器可能会根据类型特点进行优化。

因此，普通代码不要依赖枚举的具体内存布局。


## 3、空值优化

Rust会对某些枚举做内存优化。

典型例子是`Option<&T>`。

从逻辑上看，`Option<&T>` 有两种情况：

- `Some(&value)`：有引用
- `None`：没有引用

引用本身不能是空指针，因此 Rust 可以用空指针表示 `None`，用非空指针表示 `Some(...)`。

这样 `Option<&T>` 通常可以和 `&T` 一样大。


示例：

```rust
fn main() {
    println!("{}", std::mem::size_of::<&i32>());

    println!("{}", std::mem::size_of::<Option<&i32>>());
}
```

这类优化说明：枚举并不一定总是“标签 + 数据”那么简单。编译器会尽量利用类型中不可能出现的值来节省空间。

当前阶段只需要知道：Rust 枚举既有清晰的类型语义，也可能有很好的内存表示。
