# 一、模式匹配的基本概念

模式匹配是 Rust 中非常核心的控制流能力。它不仅可以判断一个值属于哪一种情况，还可以在匹配成功时把值内部的数据取出来，绑定到变量上继续使用。

最常见的模式匹配形式是`match`：

```rust
enum Msg {
    Text(String),
    Ping,
    Move { x: i32, y: i32 },
}

fn describe(msg: Msg) -> String {
    match msg {
        Msg::Text(text) => format!("text: {}", text),
        Msg::Ping => "ping".to_string(),
        Msg::Move { x, y } => format!("move to {}, {}", x, y),
    }
}
```

`=>`左边是模式，右边是匹配成功后执行的表达式。

在上述例子中：

- `Msg::Ping` 匹配不带数据的枚举变体
- `Msg::Text(text)` 匹配带一个数据的变体，并将内部`String`绑定到`text`
- `Msg::Move { x, y }` 匹配结构体风格的枚举变体，并把字段绑定到`x`和`y`

很多模式看起来像创建值的语法，这是 Rust 有意设计的：**表达式用来创建值，模式用来拆开值、检查形状并提取数据**。

`match`会从上到下依次尝试每个分支，第一个匹配成功的分支会被执行，后面的分支不再检查。因此，分支顺序很重要，**更具体的模式要放在更前面**。

示例：

```rust
let x = 10;

match x {
    n => println!("{}", n),
    1 => println!("one"),
}
```

这里的 `n` 是变量模式，可以匹配任何值，所以第二个分支永远不会被执行。编译器会给出警告 `unreachable pattern`，表示该模式不可达。


# 二、match表达式

## 1、match的基本形式

`match`的基本形式如下：

```rust
match value {
  pattern1 => expression1,
  pattern2 => expression2,
  pattern3 => expression3,
}
```

每个分支由一个模式和一个表达式组成。分支表达式较短时，通常直接写在`=>`后面，并以逗号`,`结尾。

示例：

```rust
fn describe_number(n: i32) -> &'static str {
	match n {
		0 => "zero",
		1 => "one",
		_ => "other",
	}
}
```

如果分支中需要执行多条语句，可以使用代码块：

```rust
fn print_number(n: i32) {
    match n {
        0 => println!("zero"),
        x => {
            println!("number:");
            println!("{}", x);
        }
    }
}
```

`match` 本身是表达式，所以各个分支的返回类型必须一致。

示例：

```rust
fn is_zero(n: i32) -> bool {
    match n {
        0 => true,
        _ => false,
    }
}
```


## 2、match的穷尽性检查

Rust 要求 `match` 必须覆盖所有可能情况，这称为穷尽性检查。

```rust
fn describe(n: i32) -> &'static str {
    match n {
        0 => "zero",
        1 => "one",
    }
}
```

这段代码无法通过编译，因为 `i32` 还有很多其他可能值。通常可以使用 `_` 作为兜底分支：

```rust
fn describe(n: i32) -> &'static str {
    match n {
        0 => "zero",
        1 => "one",
        _ => "other",
    }
}
```

`_`是通配符模式，表示匹配任何值，但不绑定变量。

穷尽性检查是 Rust 模式匹配的重要安全机制。它能减少遗漏分支造成的逻辑错误，尤其适合处理枚举。


# 三、常见模式

## 1、字面量模式、变量模式和通配符模式
### （1）字面量模式

字面量模式用于精确匹配某个值 。

示例：

```rust
fn traffic_light(color: &str) -> &'static str {
    match color {
        "red" => "stop",
        "green" => "go",
        "yellow" => "slow down",
        _ => "unknown",
    }
}
```

### （2）变量模式

变量模式用于匹配任何值，并把值绑定到变量。

示例：

```rust
fn show(n: i32) {
    match n {
        0 => println!("zero"),
        x => println!("non-zero: {}", x),
    }
}
```

这里的 `x` 会匹配所有没有被前面分支处理的值，并把值绑定到 `x`。


### （3）通配符模式

通配符模式 `_` 也能匹配任何值，但不会绑定变量。

示例：

```rust
fn maybe_print(n: i32) {
    match n {
        0 => println!("zero"),
        _ => {}
    }
}
```

变量模式和 `_` 的区别在于：变量模式会拿到匹配到的值，`_` 只是忽略它。

## 2、元组模式

元组模式适合一次匹配多个位置。

示例：

```rust
fn describe_point(x: i32, y: i32) -> &'static str {
    match (x, y) {
        (0, 0) => "origin",
        (_, 0) => "on the x axis",
        (0, _) => "on the y axis",
        (_, _) => "somewhere else",
    }
}
```

`(_, 0)` 表示第一个位置不关心，第二个位置必须是 `0`。

元组模式常用于**将多个值组合起来一起判断**。

```rust
fn compare(a: i32, b: i32) -> &'static str {
    match (a > 0, b > 0) {
        (true, true) => "both positive",
        (true, false) => "only a is positive",
        (false, true) => "only b is positive",
        (false, false) => "neither is positive",
    }
}
```


## 3、结构体模式

结构体模式按字段名匹配和绑定。

示例：

```rust
struct Point {
    x: i32,
    y: i32,
}

fn describe(point: Point) -> String {
    match point {
        Point { x: 0, y } => format!("on y-axis, y = {}", y),
        Point { x, y } => format!("at ({}, {})", x, y),
    }
}
```

`Point { x: 0, y }` 表示 `x` 必须等于 `0`，同时把 `y` 字段绑定到变量 `y`。

字段名和变量名不一样时，可以写成：

```rust
Point { x: a, y: b }
```

如果变量名和字段名相同，可以简写为：

```rust
Point { x, y }
```

当结构体字段很多，而模式只关心其中一部分时，可以使用 `..` 忽略其他字段。

示例：

```rust
struct User {
    name: String,
    age: u8,
    city: String,
}

fn show_name(user: User) {
    match user {
        User { name, .. } => println!("name = {}", name),
    }
}
```

`User { name, .. }` 表示绑定 `name` 字段，忽略 `age` 和 `city`。

需要注意，如果被绑定的字段不是`Copy`类型，例如`String`，那么这里会把字段移动出来。移动后，原来的结构体会发生部分移动，不能再作为完整值继续使用。

## 4、枚举模式

枚举模式是 Rust 模式匹配最常见的使用场景之一。

```rust
enum Command {
    Quit,
    Echo(String),
    Move { x: i32, y: i32 },
}

fn run(command: Command) {
    match command {
        Command::Quit => println!("quit"),
        Command::Echo(text) => println!("{}", text),
        Command::Move { x, y } => println!("move to {}, {}", x, y),
    }
}
```

不同枚举变体可以携带不同形状的数据，模式匹配可以根据变体种类分别处理，并取出内部数据。

`Option<T>`和`Result<T,E>`本质也是枚举，所以他们也经常和模式匹配一起使用。

```rust
fn print_value(value: Option<i32>) {
    match value {
        Some(n) => println!("{}", n),
        None => println!("no value"),
    }
}
```

```rust
fn parse_number(text: &str) -> i32 {
    match text.parse::<i32>() {
        Ok(n) => n,
        Err(_) => 0,
    }
}
```

`match` 非常适合处理这种“有几种明确情况，每种情况处理方式不同”的逻辑。


## 5、数组模式和切片模式

数组和切片都可以使用`[...]`形式进行模式匹配。

数组长度固定，所以**数组模式按固定位置匹配**。

示例：

```rust
fn hsl_to_rgb(hsl: [u8; 3]) -> [u8; 3]  {
	match hsl {
	  [_, _, 0] => [0, 0, 0],
	  [_, _, 255] => [255, 255, 255],
	  ...
	}
}
```

将HSL（色相、饱和度、亮度）颜色值转换为RGB（红色、绿色、蓝色）颜色值时，具有零亮度或全亮度的颜色只会是黑色或白色。

切片长度可变，可以配合`..`匹配剩余元素。

示例：

```rust
fn greet(names: &[&str]) {
    match names {
        [] => println!("Hello, nobody."),
        [one] => println!("Hello, {}.", one),
        [first, second] => println!("Hello, {} and {}.", first, second),
        [first, .., last] => println!("Hello, {} ... {}.", first, last),
    }
}
```

这里`names`类型是`&[&str]`，所以`one`、`first`、`last` 的类型是 `&&str`。这是因为**匹配的是一个切片引用**，模式中绑定到的是切片元素的引用。

切片模式适合处理“开头、结尾、长度不同”的数据结构，例如命令参数、路径片段、Token 列表等。

# 四、模式中的绑定与借用

## 1、默认绑定可能发生移动

**模式匹配不只是判断，还会绑定变量**。绑定变量时，Rust会遵守所有权规则。

如果匹配的是一个拥有所有权的值，并且绑定了其中的非`Copy`字段，那么该字段会被移动出来。

示例：

```rust
struct User {
    name: String,
    lang: String,
    age: u8,
}

fn demo(user: User) {
    match user {
        User { name, lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

这里的 `name` 和 `lang` 是 `String`，不是 `Copy` 类型，因此会被移动到分支内部的局部变量中。

如果只移动了结构体的一部分字段，原结构体会发生部分移动。之后不能再把它作为完整值使用。


## 2、ref和ref mut模式

如果只是想借用字段，而不是把字段移动出来，可以使用`ref`。

示例：

```rust
struct User {
    name: String,
    lang: String,
    age: u8,
}

fn demo(user: User) {
    match user {
        User { ref name, ref lang, .. } => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

`ref name` 表示不要把 `name` 字段移动出来，而是借用它。此时 `name` 的类型是 `&String`。

如果需要可变借用字段，可以使用 `ref mut`。

```rust
#[derive(Debug)]
struct User {
    name: String,
    lang: String,
    age: u8,
}

fn demo(mut user: User) {
    match user {
        User { ref name, ref mut lang, .. } => {
            lang.clear();
            lang.push_str("Rust");

            println!("name: {}, lang: {}", name, lang);
        }
    }

    println!("{:?}", user);
}
```

要使用 `ref mut`，被匹配的值本身必须允许可变借用，所以这里的参数写成了 `mut user`。

`ref` 和 `ref mut` 出现在模式中，它们不是创建引用的表达式，而是改变绑定方式：从“移动绑定”变成“借用绑定”。

## 3、引用模式

**`&`出现在表达式中表示创建引用，出现在模式中表示匹配引用**。

- 表达式： `&v`产生`&T`
- 模式：`&pat`只匹配`&T`类型的值。匹配成功后，Rust 会把这个引用“解开一层”，然后再拿引用里面的值去匹配 `pat`。

示例：

```rust
let x = 10;
let r = &x;

match r {
	&n => println!("{}", n),
}
```

这里 `r` 的类型是 `&i32`，模式 `&n` 会匹配这个引用，并把引用指向的 `i32` 复制到 `n` 中。因为 `i32` 是 `Copy` 类型，所以这样写没问题。

结构体也可以使用引用模式：

```rust
struct Point {
    x: i32,
    y: i32,
}

fn show(point: &Point) {
    match point {
        &Point { x, y } => {
            println!("({}, {})", x, y);
        }
    }
}
```

这里 `x` 和 `y` 是 `i32`，所以可以从共享引用中复制出来。

但如果**字段不是 `Copy` 类型，就不能通过共享引用把字段移动出来**。

示例：

```rust
#[derive(Debug)]  
struct User {  
    name: String,  
    lang: String,  
    age: u8,  
}  
  
  
fn demo(user: &User) {  
  
    match user {  
        &User { name, lang, .. } => {  
            println!("name: {}, lang: {}", name, lang)  
        }  
    }  
  
}
```

上述代码会报错：

```shell
error: cannot move out of `user.name` which is behind a shared reference
```

可以这样解决：

```rust
struct User {
    name: String,
    lang: String,
    age: u8,  
}

fn show(user: &User) {
    match user {
        &User { ref name, ref lang, ..} => {
            println!("{} uses {}", name, lang);
        }
    }
}
```

这里必须用 `ref` 借用 `name` 和 `lang`，不能直接移动它们。

**实际开发中，如果匹配的是引用，Rust的人体工学通常会自动帮你把绑定变成引用**，所以很多时候可以写得更简洁：

```rust
struct User {  
    name: String,  
    lang: String,  
    age: u8  
}  
  
  
fn show(user: &User) {  
  
    match user {  
        User { name, lang, ..} => {  
            println!("{} uses {}", name, lang);  
        }  
    }  
      
}
```

这里虽然模式里没有显式写 `&User` 或 `ref`，但因为被匹配的值是 `&User`，所以 `name` 和 `lang` 会被绑定为引用。

这一点在匹配`Option<&T>`、迭代器元素、切片引用时很常见。

示例：

```rust
fn demo(next: Option<&char>) {
    match next {
        Some(&c) => println!("next char = {}", c),
        None => println!("end"),
    }
}
```

`char`是`Copy`类型，所以`Some(&c)`可以直接把字符复制出来。

如果`T`不是`Copy`类型，例如`String`，通常直接保持引用：

```rust
fn demo(value: Option<&String>) {
    match value {
        Some(text) => println!("{}", text),
        None => println!("none"),
    }
}
```


# 五、组合模式

## 1、或模式

或模式用`|`表示一个分支可以匹配多种情况。

示例：

```rust
fn is_vowel(c: char) -> bool {
	match c {
		'a' | 'e' | 'i' | 'o' | 'u' => true,
		_ => false
	}
}
```

或模式**适合合并多个处理逻辑相同的分支**。

```rust
fn at_line_end(ch: Option<&char>) -> bool {  
    match ch {  
        Some(&'\n') | Some(&'\r') | None => true,  
        _ => false,  
    }  
}
```


## 2、范围模式

范围模式用于匹配某个区间内的值。

示例：

```rust
fn classify(c: char) -> &'static str {
	
	match c {
		'0'..='9' => "digit",
		'a'..='z' | 'A'..='Z' => "letter",
		' ' | '\n' | '\t' | '\r' => "whitespace",
		_ => "other"
	}
	
}
```

`start..=end` 表示包含`end`端点的范围，`start..end`表示不包含`end`端点的范围

示例：

```rust
fn level(score: u32) -> &'static str {
	match score {
		0..=59 => "fail",
		60..=89 => "pass",
		90.. => "excellent"
	}
}
```

`90..` 表示从 `90` 到该类型最大值的范围。

范围模式常用于数字和字符分类。需要注意，模式中的范围和表达式中的范围迭代器不是一回事；这里的范围是“匹配条件”。


## 3、匹配守卫

匹配守卫是在模式后面加一个`if`条件，用来表达更细的判断。

例如，之前判断空白字符的例子还不完善，空白字符包含空格、换行、Tab、回车，以及一些Unicode空白字符：

```rust
fn classify(c: char) -> &'static str {
	
	match c {
		'0'..='9' => "digit",
		'a'..='z' | 'A'..='Z' => "letter",
		c if c.is_whitespace() => "whitespace",
		_ => "other"
	}
	
}
```

`is_whitespace()` 覆盖的范围更广。这里的 `c if c.is_whitespace()` 是带有匹配守卫的模式，表示先把当前字符绑定到变量 `c`，然后再判断 `c.is_whitespace()` 是否为 `true`。只有这个条件成立时，才会进入该分支。

匹配守卫适合处理**结构匹配成功后，还要额外判断值**的场景。



有时候模式本身匹配成功还不够，还需要额外条件为真，这个额外条件就称为匹配守卫（match guard）。

**匹配守卫的写法就是在模式和=>之间加上if条件：**

```rust
PATTERN if CONDITION => EXPR
```
匹配流程：

先判断值是否匹配 PATTERN；

若匹配成功，再计算 CONDITION；

只有条件为真才执行该分支；否则该分支视为不匹配，继续尝试下一个分支

示例：
```rust
fn choose_page(current: u32, clicked: Option<u32>) -> Result<u32, &'static str> {

match clicked {

None => Err("not on a page"),

Some(p) => {

if p == current { Err("you clicked the current page") }

else { Ok(p) }

}

}

}
```
用匹配守卫，可改写为：
```rust
fn choose_page(current: u32, clicked: Option<u32>) -> Result<u32, &'static str> {

match clicked {

None => Err("not on a page"),

Some(p) if p == current => Err("you clicked the current page"),

Some(p) => Ok(p),

}

}
```

 
## 4、@绑定

`@`模式可以在匹配某个模式的同时，把匹配到的整个值绑定到变量上。

```rust
x @ pattern
```

示例：

```rust
fn read_token(c: char) -> String {  
    match c {  
        digit @ '0'..='9' => format!("digit: {}", digit),  
        _ => "other".to_string(),  
    }  
}
```

这里 `digit @ '0'..='9'` 表示：先判断 `c` 是否是数字字符；如果是，就把这个字符整体绑定到 `digit`。

`@`适合**既要检查模式，又要保留匹配的值**的场景。

示例：

```rust
#[derive(Debug)]
enum Msg {
    Text(String),
    Ping,
}

fn handle(msg: &Msg) {
    println!("{:?}", msg);
}

fn demo(msg: Msg) {
    match msg {
        whole @ Msg::Text(_) => handle(&whole),
        Msg::Ping => {}
    }
}
```

这里 `whole @ Msg::Text(_)` 会匹配 `Msg::Text(_)`，同时把整个 `Msg::Text(...)` 绑定到 `whole`。

如果被绑定的值不是 `Copy` 类型，`@` 绑定也会遵守所有权规则，可能发生移动。


# 六、简洁控制流

## 1、if let

当只关心一种模式时，完整的`match`会显得有些啰嗦。`if let`用来处理这种场景。

基本形式：

```rust
if let PATTERN = EXPR {

	// 匹配成功：这里可以使用PATTERN绑定的变量

}

```

- `EXPR`匹配`PATTERN`：进入代码块，并完成变量绑定
- `EXPR`不匹配`PATTERN`：什么也不做


示例：

```rust
let value = Some(10);

if let Some(n) = value {
	println!("{}", n);
}
```

它等价于：

```rust
let value = Some(10);

match value {
	Some(n) => println!("{}", n),
	None => {}
}

```

`if let` 可以理解为**只写一个关心的匹配分支，其他情况自动忽略**。

如果也想处理失败的情况，可以加`else`：

```rust
let value = Some(10);

if let Some(n) = value {
	println!("{}", n);
} else {
	println!("no value");
}
```

`if let` 适合只关心某一种成功情况的代码，例如只处理 `Some`、只处理 `Ok`、只处理某个枚举变体。

## 2、let else

`let else`适合**匹配失败就提前退出**的场景。

基本语法：

```rust
let PATTERN = EXPR else {

// 匹配失败时执行

// 这里必须提前退出：return / break / continue / panic!

};
```

- 若`EXPR`匹配`PATTERN`：把绑定写入变量，并且这个变量**在后面的外层作用域可用**
- 若`EXPR`不匹配`PATTERN`：执行else块，并且必须在块内提前退出（编译器会强制这一点）

示例：

```rust
fn process(value: Option<i32>) {  
    let Some(n) = value else {  
        return;  
    };  
    println!("{}", n);  
}
```

如果`value`是`Some(n)`，程序继续向下执行，并且后面可以使用 `n`。

如果 `value` 不是 `Some(n)`，就执行 `else` 块。`else` 块必须提前退出，例如 `return`、`break`、`continue` 或 `panic!`。

`let else`可以减少嵌套，让主流程更清楚。

```rust
fn parse(text: &str) -> i32 {
    let Ok(n) = text.parse::<i32>() else {
        return 0;
    };

    n
}
```

## 3、while let

`while let`把循环和模式匹配结合起来：只要匹配成功，就继续循环；一旦匹配失败，就退出循环。

基本语法：

```rust
while let PATTERN = EXPR {

// 循环体：可以使用PATTERN绑定的变量

}
```

示例：

```rust
fn main() {
    let mut values = vec![1, 2, 3];

    while let Some(n) = values.pop() {
        println!("{}", n);
    }
}
```

输出：

```text
3
2
1
```

`Vec::pop()`返回`Option<T>`。当还有元素时返回`Some(n)`，循环继续；当没有元素时返回 `None`，循环结束。

它等价于一个带 `match` 的循环：

```rust
fn main() {
    let mut values = vec![1, 2, 3];
	
	loop {
		match values.pop() {
			Some(n) => println!("{}", n),
			None => break,
		}
	}
}
```

`while let` 适合不断获取下一个值，直到没有值为止的流程。


# 七、模式出现的位置

模式不止出现在`match`中，也会出现在许多Rust语法里。

变量绑定中的左侧就是模式：

```rust
let (x, y) = (10, 20);
```

函数参数中也可以使用模式：

```rust
fn print_point((x, y): (i32, i32)) {
    println!("({}, {})", x, y);
}
```

`for` 循环中也常用模式拆开元素：

```rust
let pairs = vec![(1, "one"), (2, "two")];

for (number, text) in pairs {
    println!("{} = {}", number, text);
}
```

这些写法本质上都是在用模式拆解值。`match` 是最完整的模式匹配形式，其他语法则是在特定场景下使用模式。
