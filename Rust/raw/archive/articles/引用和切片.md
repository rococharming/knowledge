# 一、问题引入

在上一节[[3、所有权|所有权]]中已经知道，对于非`Copy`类型，按值传参会发生所有权转移。

假设有一个哈希表（Rust标准库数据结构为`HashMap`），键是类别名，值是该类别下的一些例子。

示例：

```rust
use std::collections::HashMap;

type Table = HashMap<String, Vec<String>>;   // 别名，用 Table 指代 HashMap<String, Vec<String>>

fn show_table(table: Table) {
    for (tag, examples) in table {
        println!("{}:", tag);

        for example in examples {
            println!("\t{}", example);
        }
    }
}

fn main() {
    let mut table = Table::new();

    table.insert(
        "fruit".to_string(),
        vec![
            "apple".to_string(),
            "orange".to_string(),
            "banana".to_string(),
        ],
    );

    table.insert(
        "animal".to_string(),
        vec![
            "dog".to_string(),
            "cat".to_string(),
            "rabbit".to_string(),
        ],
    );

    table.insert(
        "color".to_string(),
        vec![
            "red".to_string(),
            "green".to_string(),
            "blue".to_string(),
        ],
    );

    show_table(table);

    // println!("{:?}", table); // 错误，table已经被移动
}
```

结果：

![[Image 38.png|300]]

上述代码本身没有问题。`show_table(table)` 会把 `table` 按值传入函数，因此 `table` 的所有权会转移进 `show_table` 函数中。

当 `show_table` 执行结束后，函数参数 `table` 离开作用域，整个哈希表也会被释放。此时，`main` 函数中的 `table` 已经失效，不能再继续使用。

但很多时候，函数只是想临时读取一个值，并不想取得它的所有权。为了解决这个问题，Rust提供了 **引用（Reference）** 。

# 二、引用

## 1、引用的概念

引用是**非拥有型指针**。

**它允许代码临时访问某个值，但不取得这个值的所有权**。引用离开作用域时，不会释放被引用的数据。被引用的数据什么时候释放，仍然由它的所有者决定。

Rust把创建引用的过程称为**借用（borrow）**。

示例：

```rust
fn main() {
    let s = String::from("hello");

    let r = &s;  // r 是 s 的引用

    println!("{}", r);

    println!("{}", s);
}
```

这里 `r` 是 `s` 的引用。`r` 只是临时借用了 `s`，并没有取得 `String` 的所有权。

因此，使用 `r` 之后，`s` 仍然可以继续使用。

引用可分为**共享引用**和**可变引用**。


## 2、共享引用

共享引用用`&T`表示。

表达式`&s`会创建一个指向`s`的共享引用。若`s`的类型为`T`，则`&s`的类型就是`&T`。

**共享引用只能读取数据，不能修改数据**。

示例：

```rust
fn main() {
    let s = String::from("Hello");

    let r1 = &s;  // r1 是对 s 的共享引用(&String)

    let r2 = &s;  // r2也是对 s 的共享引用(&String)

    println!("r1: {}", r1);

    println!("r2: {}", r2);

    println!("s: {}", s);
}
```

![[Image 39.png|400]]

这里 `r1` 和 `r2` 都是 `s` 的共享引用。它们只是读取 `s`，不会取得 `s` 的所有权。

所以在创建共享引用之后，`s` 本身仍然有效。

内存关系可以这样理解：

![[Image 40.png|400]]

共享引用不拥有堆区字符串数据，因此不会负责释放它。

由于可以有多个共享引用，所以共享引用是`Copy`类型。

## 3、可变引用

可变引用使用`&mut T`表示。它也称为**独占引用**。

表达式 `&mut s` 会创建一个指向 `s` 的可变引用。若`s`的类型为`T`，则`&mut s`的类型就是`&mut T`。

**如果要创建可变引用，被引用的数据本身也要声明为`mut`。**

示例：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r1 = &mut s;  // r 是 s 的可变引用

    r1.push_str(", World");

    println!("{}", r1);
}
```
结果：

![[Image 41.png|400]]

这里 `r1` 是 `s` 的可变引用。通过 `r1` 可以修改 `s` 指向的字符串数据。

如果 `s` 没有声明为 `mut`，就不能创建 `&mut s`。只有变量本身允许修改，才能借出可变引用。

需要注意的是，`mut s` 和 `&mut s` 不是同一个概念：

- `mut s` 表示变量 `s` 本身允许被修改。
- `&mut s` 表示创建一个指向 `s` 的可变引用。


## 4、共享引用与可变引用的借用规则

共享引用与可变引用不能随意混用。Rust 对它们有严格的借用规则。

规则可以概括为两条：

1. **同一时间，可以有多个共享引用**
2. **同一时间，只能有一个可变引用，并且不能同时存在正在使用的共享引用**。

先看多个可变引用的例子：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r1 = &mut s;

    let r2 = &mut s; // 错误

    println!("{}", r1);

    println!("{}", r2);
}
```

这段代码会报错：

```text
cannot borrow `s` as mutable more than once at a time
```

原因是：**如果同一时间存在多个可变引用，就可能同时修改同一个值。Rust为了避免数据混乱，要求可变引用必须是独占的**。

再看共享引用和可变引用同时存在的例子：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r1 = &s;

    let r2 = &mut s; // 错误

    println!("{}", r1);

    println!("{}", r2);
}
```

这段代码会报错：

```text
cannot borrow `s` as mutable because it is also borrowed as immutable
```

原因是：`r1` 正在以共享引用的方式读取 `s`，而 `r2` 又想以可变引用的方式修改 `s`。如果允许这种行为，就可能出现“读的时候数据被改掉”的问题。

因此，记住如下的规则：

- 只要一个值正在被共享引用使用，就不能同时通过可变引用修改它。
- 只要一个值正在被可变引用使用，就不能再通过其他引用访问它。


## 5、所有者与引用的关系

**引用存在时，所有者本身也要遵守借用规则**。

示例：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r = &mut s;  // r 是 s 的可变引用

    println!("{}", s); // 错误

    println!("{}", r);
}
```

这段代码会报错。

原因是：`r` 是 `s` 的可变引用，在 `r` 最后一次使用之前，它独占了对 `s` 的访问权。此时即使是所有者 `s` 本身，也不能绕过 `r` 去访问这个值。

再看一个共享引用的例子：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r = &s;   // r 是 s 的共享引用

    s.push_str(", World"); // 错误

    println!("{}", r);
}
```

这段代码也会报错。

原因是：`r` 正在共享借用 `s`。在这个共享引用仍然要被使用的情况下，不能修改 `s`。

共享引用表示可以读取，不允许修改；可变引用表示可以修改，但必须独占。

## 6、引用的实际使用范围

Rust判断引用是否还存在，看的不是变量声明的位置，而是**最后一次使用的位置**。

示例：

```rust
fn main() {
    let mut s = String::from("Hello");

    let r1 = &s;

    let r2 = &s;

    println!("{} {}", r1, r2);

    let r3 = &mut s;

    r3.push_str(", World");

    println!("{}", r3);
}
```

这段代码可以编译。

虽然 `r1` 和 `r2` 在语法上仍然处于同一个作用域中，但它们在 `println!("{} {}", r1, r2);` 之后就不再被使用了。

因此，编译器允许后面再创建 `r3` 这个可变引用。

这种能力叫做 **非词法生命周期** 。这里不需要展开生命周期，只需要知道：Rust会根据引用的实际使用位置来判断借用是否结束。

# 三、使用引用改造函数

## 1、使用共享引用读取数据

前面的问题中，`show_table(table)` 会移动整个哈希表。如果只是想打印表格，就不应该让函数取得所有权，而应该传入共享引用。

示例：

```rust
use std::collections::HashMap;  
  
type Table = HashMap<String, Vec<String>>;  
  
  
fn show_table(table: &Table)  
{  
    for (tag, examples) in table {  
        println!("{}:", tag);  
  
        for example in examples {  
            println!("\t{}", example);  
        }  
    }  
}  
  
  
  
fn main() {  
  
    let mut table = Table::new();  
  
    table.insert(  
        "fruit".to_string(),  
        vec![  
            "apple".to_string(),  
            "orange".to_string(),  
            "banana".to_string(),  
        ]  
    );  
  
    table.insert(  
        "animal".to_string(),  
        vec![  
            "dog".to_string(),  
            "cat".to_string(),  
            "rabbit".to_string(),  
        ],  
    );  
  
    table.insert(  
        "color".to_string(),  
        vec![  
            "red".to_string(),  
            "green".to_string(),  
            "blue".to_string(),  
        ],  
    );  
  
    show_table(&table);  
  
    println!("{:?}", table); 
  
}
```

这里 `show_table` 的参数类型从 `Table` 改成了 `&Table`。

调用时写成：

```rust
show_table(&table);
```

这表示把 `table` 的共享引用传给函数。函数可以读取哈希表内容，但不会拥有哈希表本身。

因此 `show_table` 调用结束后，`main` 函数中的 `table` 仍然可用。

## 2、使用可变引用修改数据

如果函数需要修改外部传入的数据，可以使用可变引用。

下面实现一个 `sort_table` 函数，对每个类别下的例子进行排序。

示例：

```rust
use std::collections::HashMap;

type Table = HashMap<String, Vec<String>>;

fn show_table(table: &Table) {
    for (tag, examples) in table {
        println!("{}:", tag);

        for example in examples {
            println!("\t{}", example);
        }
    }
}

fn sort_table(table: &mut Table) {
    for (_tag, examples) in table {
        examples.sort();
    }
}

fn main() {
    let mut table = Table::new();

    table.insert(
        "fruit".to_string(),
        vec![
            "orange".to_string(),
            "banana".to_string(),
            "apple".to_string(),
        ],
    );

    table.insert(
        "animal".to_string(),
        vec![
            "dog".to_string(),
            "rabbit".to_string(),
            "cat".to_string(),
        ],
    );

    table.insert(
        "color".to_string(),
        vec![
            "red".to_string(),
            "blue".to_string(),
            "green".to_string(),
        ],
    );

    sort_table(&mut table);

    show_table(&table);
}
```

这里 `sort_table` 的参数类型是：

```rust
&mut Table
```

调用时写成：

```rust
sort_table(&mut table);
```

这表示把 `table` 的可变引用传给函数。函数没有取得 `table` 的所有权，但可以通过可变引用修改它。

`sort_table` 调用结束后，可变借用结束，后面又可以通过 `show_table(&table)` 创建共享引用来读取它。

# 四、解引用

## 1、显式解引用

引用指向某个值，如果想通过引用拿到底层值，可以使用`*`进行解引用。

示例：

```rust
fn main() {
    let x = 10;
    
    let r = &x;

    assert_eq!(*r, 10);
}
```

> `assert_eq!`是一个断言宏，断言两个值是否相等，若不想等程序会panic

这里 `r` 的类型是 `&i32`，表示它是一个指向 `i32` 的引用。

`*r` 表示沿着引用找到它指向的值，所以结果是 `10`。

## 2、通过可变引用修改值

可变引用也可以通过 `*` 解引用，然后修改底层值。

示例：

```rust
fn main() {
    let mut x = 10;

    let r = &mut x;

    *r += 5;

    println!("{}", r);
}
```

这里 `*r += 5` 表示修改 `r` 指向的那个值。执行之后，`x` 的值变成 `15`。


## 3、自动解引用

在很多场景下，Rust会自动帮我们解引用，因此不需要总是手写`*`。

例如通过引用访问结构体字段时，Rust会自动解引用，由于这里还没学习[[结构体|结构体]]，因此可以先留个印象。

示例：

```rust
struct Person {
    name: String,
    age: u32,
}

fn main() {
    let p = Person {
        name: "Tom".to_string(),
        age: 20,
    };

    let p_ref = &p;

    println!("{}", p_ref.name);

    println!("{}", p_ref.age);
}
```

这里`p_ref`是`&Person`类型，但仍然可以直接写：

```rust
p_ref.name
```

它等价于：

```rust
(*p_ref).name
```

Rust会根据上下文自动完成解引用，让代码更简洁。

Rust支持**对引用再进行引用**，此时`.`运算符会尽可能追踪多层次的引用，解引用到最终的目标：

```rust
struct Person {  
    name: String,  
    age: u32  
}  
  
fn main() {  
  
    let p = Person {  
        name: "Tom".to_string(),  
        age: 20  
    };  
  
    let r: &Person = &p;  
  
    let rr: &&Person = &r;  
  
    let rrr: &&&Person = &rr;  
  
    assert_eq!(rrr.age, 20);  
}
```

`rrr.age`会根据`rrr`的类型指引遍历3层引用取到最终的age字段。

## 4、方法调用中的自动借用

**调用方法时，Rust不仅会自动解引用，还会根据方法参数自动借用**。关于方法，详见[[结构体|结构体]]。

例如`Vec::sort`方法需要的是`&mut self`，也就是可变引用。

示例：

```rust
fn main() {
    let mut v = vec![20, 10, 30];

    v.sort();

    println!("{:?}", v);
}
```

这里直接写：

```rust
v.sort();
```

Rust自动理解为：

```rust
(&mut v).sort();
```

所以方法调用经常比普通函数调用看起来更简洁。


## 5、比较引用

除了`.`运算符外，**比较运算符**也可以自动进行解引用。

示例：

```rust
let x = 10;
let y = 10;

let rx = &x;
let ry = &y;

let rrx = &rx;
let rry = &ry;

assert!(rrx == rry);
```

`==`运算符会追踪所有的引用并对最终目标`x`和`y`进行比较。

需要注意的是，比较运算符左右两边的类型必须完全一致，下面这种写法就是错误的：

```rust
assert!(rrx == ry); // 错误，rrx 是 &&i32类型，ry 则是 &i32 类型，类型不匹配
```

比较运算符默认会对引用进行解引用，比较它们引用的底层的目标的值是否一样。

但如果想要知道**两个引用是否指向同一目标**，需要使用`std::ptr::eq`，它会比较地址：

```rust
assert!(!std::ptr::eq(rx, ry));
```


# 五、对引用变量赋值

引用本身也可以绑定到变量上。

如果引用变量声明为 `mut`，那么这个引用变量可以改为指向另一个目标。

示例：

```rust
fn main() {
    let x = 10;

    let y = 20;

    let mut r = &x;

    println!("{}", r);

    r = &y;

    println!("{}", r);
}
```

这和C++的引用有很大区别，C++的引用一旦完成初始化就不能改变。

这里需要区分两个概念：

`mut r` 表示变量 `r` 本身可以重新赋值。

`&mut T` 表示可以通过引用修改它指向的值。

它们不是一回事。

# 六、悬垂引用

## 1、悬垂引用的概念

悬垂引用是指：**引用指向的数据已经失效，但引用仍然存在**。

在C/C++中，悬垂指针是一类常见错误。好消息是，Rust会在编译期间阻止这种情况。

示例：

```rust
fn dangle() -> &'static String {
    let s = String::from("Hello");
    &s
}
```

这段代码是错误的，Rust会在编译期指出问题：

![[Image 44.png|600]]

原因是：`s` 是函数内部创建的局部变量。当函数执行结束时，`s` 会离开作用域并被释放。如果允许返回 `&s`，那么返回的引用就会指向一块已经失效的内存。

> 这里的 `'static` 属于生命周期相关内容，后续再展开。当前只需要关注一点：Rust不允许返回指向局部变量的引用。

正确写法是直接返回值本身：

```rust
fn no_dangle() -> String {
    let s = String::from("Hello");
    s
}
```

这里不是返回 `s` 的引用，而是把 `s` 的所有权移动给调用者。

因此，函数结束后数据不会被释放，而是由调用者继续拥有。


# 七、借用临时值

**在Rust中，引用不仅可以指向已经存在的变量，也可以指向一个临时计算出来的值**。

示例：

```rust
fn main() {
    let r = &String::from("hello");  

    println!("{}", r);
}
```

这里的 `String::from("hello")` 会创建一个新的 `String` 值。但是这个值没有绑定到任何变量上，也就是说我们没有写：

```rust
let s = String::from("hello");
```

所以它是一个临时值。

当我们写：

```rust
let r = &String::from("hello");
```

可以近似理解为：

```rust
let temp = String::from("hello");
let r = &temp;
```

只不过这里的 `temp` 是编译器生成的临时内存位置，我们无法直接使用它的名字。

也就是说，Rust 会先计算 `String::from("hello")`，然后把结果保存到一个临时位置中，再让引用 `r` 指向这个临时值。

通常情况下，临时值会在当前语句结束时被释放。但对于这种`let r = &String::from("hello")`特定语法，Rust会延长临时值的生命周期。

下面这种写法就是错误的：

```rust
fn main() {
    let r;

    r = &String::from("hello");  // 错误

    println!("{}", r);
}
```

报错：

```text
temporary value dropped while borrowed
```

原因是这里的`r`是先声明变量，再进行普通赋值。Rust 的临时值生命周期延长规则主要发生在特定的 `let` 初始化语句中，不会对这种普通赋值语句生效。

实际上上述错误代码可以等价于：

```rust
fn main() {
    let r;
	
	{
		let temp = String::from("hello");
		r = &temp;  // 错误
	}
    
    println!("{}", r);
}
```

当 temp 离开作用域，值就被释放了，此时`r`指向的就是一个已经失效的值，成为悬垂引用。Rust 为了保证内存安全，会直接拒绝这段代码。

# 八、切片

## 1、切片的基本概念

在介绍切片（slice）之前，需要区分**切片**和**切片引用**的概念。

>很多资料会把切片引用简称为切片，但从类型角度来看，它们不是一个东西

**切片**：表示**集合中的一段连续元素**，类型为`[T]`。由于切片 `[T]` 可以表示任意长度的一段元素，因此它属于动态大小类型（Dynamically Sized Type，DST）。DST 类型的值在编译时无法确定大小，所以不能直接存储在普通变量中，也不能直接作为函数参数按值传递。

**切片引用**：表示**对切片的引用**，类型为`&[T]`或`&mut [T]`。切片引用和普通引用不同，它本质上是一个 **非拥有型胖指针**，包含两部分信息：指向第一个元素的指针，以及切片的长度。

日常开发中，为了方便，通常会把切片引用 `&[T]` 或 `&mut [T]` 简称为切片。后文如果没有特别说明，提到“切片”时，通常指的就是切片引用。


## 2、创建切片

可以对数组或`Vec`使用**范围语法**创建切片。

示例：

```rust
fn main() {
    let arr = [10, 20, 30, 40, 50];

    let slice = &arr[1..4];

    println!("{:?}", slice);
}
```

输出结果：

```text
[20, 30, 40]
```

这里的`&arr[1..4]`表示从下标 `1` 开始，到下标 `4` 之前结束（不包含下标 `4` 对应的 `50`）。

需要注意的是，`arr[1..4]` 表示数组中的一段区域，类型是 `[i32]`；但我们实际绑定到变量上的是 `&arr[1..4]`，也就是 `&[i32]` 类型的切片引用。

## 3、常见范围写法

切片范围有几种常见写法：

```rust
fn main() {
    let arr = [10, 20, 30, 40, 50];

    let s1 = &arr[1..4]; // [20, 30, 40]

    let s2 = &arr[..3]; // [10, 20, 30]

    let s3 = &arr[2..]; // [30, 40, 50]

    let s4 = &arr[..]; // [10, 20, 30, 40, 50]

    println!("{:?}", s1);

    println!("{:?}", s2);

    println!("{:?}", s3);

    println!("{:?}", s4);
}

```

说明：

- `start..end` 表示从 `start` 到 `end` 之前。

- `..end` 表示从开头到 `end` 之前。

- `start..` 表示从 `start` 到结尾。

- `..` 表示整个范围。

> 如果切片范围越界，程序会 panic。


## 4、可变切片

如果想通过切片修改原集合中的元素，可以创建可变切片。

示例：

```rust
fn main() {
    let mut arr = [1, 2, 3];

    let slice = &mut arr[..];

    slice[1] = 20;

    println!("{:?}", slice);

    println!("{:?}", arr);
}
```

输出结果是：

```
[1, 20, 3]
[1, 20, 3]
```

这里 `slice` 是 `arr` 的可变切片。通过 `slice[1] = 20` 修改的是原数组中的元素。

可变切片同样遵守可变引用的规则：当可变切片存在并被使用时，不能同时使用原数组或创建其他引用。

例如下面这样会报错：

```rust
fn main() {
    let mut arr = [1, 2, 3];

    let slice = &mut arr[..];

    println!("{:?}", arr); // 错误

    slice[1] = 20;
}
```

原因是：`slice` 是对 `arr` 的可变借用，在 `slice` 最后一次使用之前，不能再通过 `arr` 访问原数组。


## 5、数组切片和Vec切片

数组和 `Vec` 都可以创建切片。

```rust
fn main() {
    let arr = [10, 20, 30, 40, 50];

    let v = vec![10, 20, 30, 40, 50];

    let arr_slice = &arr[1..4];

    let vec_slice = &v[1..4];

    println!("{:?}", arr_slice);

    println!("{:?}", vec_slice);
}
```

数组切片和 `Vec` 切片的类型都是`&[T]`，**因此，函数如果只需要读取一段连续元素，参数就可以写成`&[T]`。这样既可以接收数组切片，也可以接收 `Vec` 切片**。

示例：

```rust
fn print_numbers(numbers: &[i32]) {
    for number in numbers {
        println!("{}", number);
    }
}

fn main() {
    let arr = [1, 2, 3];

    let v = vec![4, 5, 6];

    print_numbers(&arr);

    print_numbers(&v);

    print_numbers(&arr[1..]);

    print_numbers(&v[..2]);
}
```

这里 `print_numbers` 的参数是 `&[i32]`，因此它不关心数据来自数组还是 `Vec`，只关心“这一段连续的 `i32` 元素”。

这种写法比直接接收 `&Vec<i32>` 更通用。

## 6、切片引用的本质

切片引用是一个 **胖指针** 。

普通引用通常只保存一个地址。

切片引用除了保存起始地址，还要保存切片长度。

所以一个 `&[T]` 大致包含两部分信息：

- 指向第一个元素的指针
- 切片包含的元素数量

## 7、切片的常用方法

切片可以使用一些常见方法。

示例：

```rust
fn main() {
    let arr = [10, 20, 30, 40, 50];

    let slice = &arr[..];

    println!("{}", slice.len());       // 返回切片长度

    println!("{}", slice.is_empty());  // 判断切片是否为空

    println!("{:?}", slice.get(0));    // 安全访问指定位置的元素。

    println!("{:?}", slice.get(100));
}
```

- `len()` 返回切片长度。

- `is_empty()` 判断切片是否为空。

- `get(index)` 安全访问指定位置的元素。

	- 如果下标存在，`get` 返回 `Some(&元素)`。

	- 如果下标越界，`get` 返回 `None`。

`get` 不会因为越界而 panic。相比直接使用`slice[index]`要安全。
